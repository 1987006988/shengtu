# 03 关键角色类别草案

## 1. 文档用途

本文档用于给 `07_characters` 建立关键角色类别的第一轮草案。

当前阶段重点不是写具体人物，而是定义“哪些类别的角色要出现、分别承担什么设定功能、适合从哪些来源模型中长出来”。

## 2. 主要角色类别

### 主角

- 设定功能：承担玩家主要视角、连接宇宙广度与古裂残天现实、承受主线推动力。
- 更适合的来源：高含金量飞升者、本土关键接口人物，或两者兼具的稀缺来源。
- 写作边界：当前阶段只定来源框架与接口，不写完整生平、对白与剧情线。

### 同伴

- 设定功能：扩展世界接口、补齐模板覆盖、提供不同社会层与阵营层视角。
- 更适合的来源：飞升者、本土修士、地方势力人物、边缘结构接口人物。
- 写作边界：不能全部同质，也不能全部围着主角服务而缺乏自身世界接口。

### 阵营代表

- 设定功能：把中心大势力、地方势力、反抗网络、旧势力、新势力等结构人格化。
- 更适合的来源：中心体系人物、地方强权人物、新势力中坚、反抗网络节点人物。
- 写作边界：当前阶段只确定其类别功能，不写具体阵营戏码。

### 地方关键人物

- 设定功能：承接地方区、群陆外海区、资源区、商路与地方共同体的现实结构。
- 更适合的来源：地方势力来源、本土修士来源、地方凡俗国家或基层共同体接口来源。
- 写作边界：不能被简单写成“地方版阵营代表”，其重点是社会落地功能。

### 历史承载型角色

- 设定功能：承接旧势力遗绪、双极旧秩序余波、大战后残存认知与历史断层。
- 更适合的来源：旧势力后裔、旧法统残脉、隐势力承载者、极少数高位背景人物。
- 写作边界：当前阶段只确定其设定功能，不写成历史解说器或纯 exposition 角色。

### 反抗网络人物

- 设定功能：承接暗线层、地下网络、地方渗透结构与裂伤后的秩序反面。
- 更适合的来源：旧敌残余、不认同现秩序者、暗线势力中坚、地方节点型人物。
- 写作边界：不得写成突然冒出来的地下组织化身，必须承接长期渗透网络来源。

### 中心体系人物

- 设定功能：承接名义中心、中心合法性、中心秩序的现实边界与执行力衰减问题。
- 更适合的来源：中心法统内部人物、核心区制度人物、中心秩序的执行或承接人物。
- 写作边界：不得一律写成中心绝对代言人，也不能都写成僵硬反派。

### 背景大人物

- 设定功能：提供远景压力、世界上层秩序感、阵营重量与历史纵深。
- 更适合的来源：旧势力高层残脉、中心高位人物、新秩序顶层人物、隐势力真正核心承载者。
- 写作边界：当前阶段只保留类别，不写成具体人物设定卡。

## 3. 每类角色承担的设定功能摘要

- 主角 / 同伴：承担玩家长期视角与世界接口覆盖。
- 阵营代表 / 中心体系人物：承担秩序与权力结构的人格化出口。
- 地方关键人物：承担地方社会、商路、资源区、地方共同体等现实结构落地。
- 历史承载型角色：承担大战前后、旧势力与历史断层的余波。
- 反抗网络人物：承担地下结构、渗透网络与秩序裂缝。
- 背景大人物：承担世界重量感，而不是日常陪跑。

## 4. 每类角色最适合从哪些来源模型中生长出来

- 主角：飞升者来源 / 本土关键接口来源
- 同伴：飞升者来源 / 本土修士来源 / 地方势力来源 / 边缘接口来源
- 阵营代表：中心体系来源 / 新势力来源 / 反抗网络来源 / 地方强权来源
- 地方关键人物：地方势力来源 / 本土修士来源 / 地方共同体接口来源
- 历史承载型角色：旧势力来源 / 隐势力来源 / 旧法统残脉来源
- 反抗网络人物：旧敌残余来源 / 不认同者来源 / 暗线势力来源
- 中心体系人物：中心法统来源 / 核心区制度来源
- 背景大人物：旧势力高层来源 / 中心高位来源 / 隐势力真正核心来源

## 5. 后续写具体角色时的边界

- 不得直接跳过来源模型与类别功能，直接写人物小传。
- 不得让所有关键角色都来自同一社会层、同一模板或同一势力来源。
- 不得让历史承载型角色和背景大人物过度抢占主角团功能。
- 不得让角色类别直接替代剧情结构；类别只负责“世界接口”和“设定功能”。
