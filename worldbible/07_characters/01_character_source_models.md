# 01 角色来源模型总论

## 1. 文档用途

本文档用于给 `07_characters` 提供统一的角色来源模型语言。

当前阶段重点不是“谁是谁”，而是“什么样的人会从这个宇宙与古裂残天现实格局中长出来，并值得被写进故事”。

## 2. 角色来源模型总论

在本项目里，值得进入故事的角色，不应只因为“战力高”或“身份大”，而应至少满足以下一种或数种接口价值：

- 能承接宇宙广度：体现小世界模板差异、飞升制度、九主世界差异或世界群结构。
- 能承接古裂残天现实：体现中心大势力、地方势力、反抗网络、商路 / 资源区 / 空间入口带等现实结构。
- 能承接历史层：体现旧势力遗绪、新势力成长、隐势力潜伏或大战后修复期的裂痕。
- 能承接游戏聚焦层：具备进入主角团、阵营人物、关键支线人物或背景大人物的位置。

因此，角色来源模型不是身份清单，而是“世界接口清单”。

## 3. 飞升者来源模型

飞升者最适合承接的功能包括：

- 体现宇宙广度与 `3000` 小世界的道路差异。
- 体现不同小世界模板如何塑造人格、修行习惯、法的组织方式与规则接近路径。
- 体现“在不完整环境中触规”的含金量与进入主世界后的不适应。
- 体现古裂残天如何接纳、筛选、利用或误判外来高含金量人物。

飞升者内部还可继续细分为：

- 宗门体系飞升者
- 家族 / 势力经营型飞升者
- 剑器 / 本命承载型飞升者
- 斗战 / 体魄升阶型飞升者
- 香火 / 神道 / 佛国型飞升者
- 王朝 / 江湖 / 探案型飞升者
- 诡异 / 污染环境中成长的飞升者
- 游历 / 志怪 / 公路仙侠型飞升者

飞升者最适合作为：

- 主角团核心来源之一
- 重要同伴来源
- 少数高含金量阵营人物来源

但不应让所有关键角色都来自飞升者，否则会削弱古裂残天本土结构的存在感。

## 4. 本土修士来源模型

本土修士最适合承接的功能包括：

- 体现古裂残天内部真实社会如何运转。
- 体现中心秩序、地方秩序、群陆外海网络与入口带秩序的差异。
- 体现“高位主世界仍在运转，但执行力分层衰减”的现实。
- 为主角团提供真正的本地认知、制度接口与风险判断能力。

本土修士内部可继续细分为：

- 中心体系内成长的修士
- 地方区强宗 / 地方共同体出身的修士
- 群陆外海节点型修士
- 空间入口带周边成长的高风险环境修士

本土修士最适合作为：

- 主角团中的本地接口人物
- 阵营代表人物
- 地方关键人物
- 中心体系人物

## 5. 地方势力来源模型

地方势力人物是古裂残天现实结构最重要的角色来源之一，因为他们最能体现：

- 依附坐大为主、区域自立为辅的地方格局。
- 地方凡俗国家、商路、资源区与地方共同体如何在大势力阴影下运转。
- 为什么古裂残天不是中心全包，也不是完全碎裂。

这类人物最适合作为：

- 主角团中的地方接口人物
- 地方关键人物
- 支线核心人物
- 资源区、商路区、节点带的重要承接者

## 6. 旧势力 / 新势力 / 隐势力 / 反抗势力的人物来源模型

### 旧势力来源

- 适合承接古裂残天历史厚度、旧法统残片与双极旧秩序的余波。
- 更适合作为历史承载型角色、背景大人物或高层阵营人物。

### 新势力来源

- 适合承接修复期后成长起来的新秩序现实。
- 更适合作为中坚人物、上升型人物、地方或区域强权的人物来源。

### 隐势力来源

- 适合承接附属空间层、封存结构、旧日残脉与长期未公开显形的力量。
- 更适合作为关键支线人物、历史承载型人物或中后段揭示型人物来源。

### 反抗势力来源

- 已确认其来源是旧敌残余、不认同当前统治风格者，以及借裂伤成长的暗线力量。
- 这类人物更适合承接秩序裂缝、地下网络、商路渗透与地方共生关系。
- 不应写成突然出现的外来角色群。

## 7. 哪些来源更适合主角团，哪些更适合配角与阵营人物

### 更适合主角团 / 重要同伴的来源

- 飞升者来源：负责宇宙广度与模板差异。
- 本土修士来源：负责现实接口与古裂残天本地认知。
- 地方势力来源：负责地方社会、资源区、商路与地方共同体接口。
- 少量反抗网络或隐势力来源：负责连接地下结构与隐藏真相，但不宜全部如此。

### 更适合阵营人物 / 关键支线人物 / 背景大人物的来源

- 中心体系人物：更适合作为阵营代表与秩序接口人物。
- 旧势力来源：更适合作为历史承载型角色或背景大人物。
- 新势力中坚：更适合作为区域格局、地方上升秩序或竞争阵营代表。
- 隐势力承载者：更适合承担局部揭示、历史残片或关键边缘支线功能。

## 8. 本文件对后续写作的约束

- 后续写具体角色时，必须先说明其属于哪一种来源模型，再写人物细节。
- 不得让所有重要角色都来自飞升者。
- 不得让所有重要角色都绑定古裂残天中心大势力。
- 不得把来源模型直接写成完整人物传记、剧情线或角色对白草稿。
- 角色模块不得反向改写宇宙通则、主世界层与社会层口径。
