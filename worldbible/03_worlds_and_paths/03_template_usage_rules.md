# 03 模板使用规则

## 文档定位

本文件用于规定“小世界道路模板库”的使用边界，避免模板化工具在后续扩写中失控、互相污染或被误当作具体设定。

## 1. 模板不是具体世界

模板只是一种抽象框架，用来描述某类小世界在修行生态、道路源流与文明重心上的共性。

因此：

- 模板不能直接替代具体小世界设定
- 模板不能直接充当世界名
- 模板不能代替环境、历史、秩序与地方差异的具体写作

## 2. 模板不是具体职业

模板描述的是“小世界整体道路风格”，而不是单个修行者的职业树或技能树。

因此：

- 一个模板中可以容纳多种职业与多条法门
- 模板不能被理解为单一职业分类
- 模板也不能直接对应游戏玩法构成

## 3. 模板不是具体小说映射

模板可以承接未来对某类小说气质的参考，但模板本身不是具体小说映射。

因此：

- 不应把某一部作品直接等同于某一个模板
- 不应把模板正文写成某部作品的改写摘要
- 可在后续创作讨论中使用“凡人流、家族流、剑道流、诡仙流”等泛称作为参考方向，但不写入正典模板正文为具体作品对应表

## 4. 后续如何从模板发展出具体小世界设定

后续扩写具体小世界时，建议遵循以下顺序：

1. 先确定一个主模板。
2. 再确定一个副模板。
3. 再补充该世界的天地偏性、规则显化偏向、文明结构与历史秩序遗产。
4. 最后再进入具体势力、地区、人物、剧情与事件设计。

这样做的目的，是让“模板 -> 世界 -> 势力 / 人物 / 剧情”的展开顺序保持稳定。

## 5. 如何避免模板之间互相污染或无限膨胀

为避免模板体系失控，应遵守以下规则：

- 八个主模板当前固定，不得私自增加第九类。
- 单个小世界原则上只采用“一个主模板 + 一个副模板”。
- 若某个世界已经需要三个以上模板同时才能成立，优先反查是否模板定义过于模糊。
- 若某种风格只在个别世界中成立，应优先写成该世界特例，而不是上升为新模板。
- 若模板之间边界不清，应先修模板定义，不要直接堆叠概念。

## 6. 本模块对后续设定的约束

- 不得把模板卡直接拿来当作世界正文。
- 不得在模板文件中混入具体世界名、势力名、人物名、地名与具体剧情。
- 不得把模板写成玩法树、职业树或技能成长图。
- 模板只负责提供分类框架与语言，不负责替代具体创作。

## 自检

- 是否把模板误写成具体世界：否
- 是否引入了具体小说名或具体剧情：否
- 是否把模板写成职业技能树：否
- 是否保持内部正式设定文风：是
