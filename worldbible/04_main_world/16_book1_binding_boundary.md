# 第一部强绑定设定边界文件（白鹿原 / 古裂残天接口）

## 1. 文件定位

本文件专门回答一件事：

> 第一部《封绝之地》可以从古裂残天总层中调用什么，不可以调用什么。

它的作用是防止后续写作出现两个常见错误：

1. 把白鹿原写成整个古裂残天；
2. 把第一部写成整个系列的终局预演。

## 2. 第一部可调用的古裂残天层级

### 2.1 可直接调用
- 古裂残天是受损但仍在运行的主世界；
- 古战让其产生多处伤区；
- 白鹿原是其中一处被推离常规秩序带的伤区；
- 守衡司、地方共同体、行栈、灰线、誓约见证力量等都属于古裂残天总层逻辑的地方化显现；
- 古裂残天的路权、修复权、签字权冲突可在白鹿原局部具体化。

### 2.2 只能边角感知，不可展开
- 古裂残天其他伤区具体情况；
- 主世界中枢区、外海碎界带、旧大战最深区的完整格局；
- 古裂残天对其他主世界的系统关系；
- 主世界级主势力的全量谱系。

### 2.3 第一部绝对禁止
- 把白鹿原问题上升为整个古裂残天唯一核心问题；
- 把主角写成能直接改变古裂残天总体命运的人；
- 把白鹿之灵写成古裂残天总意志；
- 把接引法宝写成整个宇宙一切谜题的钥匙。

## 3. 白鹿之灵边界

### 3.1 白鹿之灵能做什么
- 在白鹿原局部范围显化；
- 反馈白鹿原结构承压；
- 比较局部未来代价；
- 对与白鹿原直接相关的回引、节点、缓冲失稳做出回应。

### 3.2 白鹿之灵不能做什么
- 直接解释九主世界总局势；
- 替角色做伦理判断；
- 代替作者做世界说明；
- 随时随地出现；
- 作为万用保命装置。

### 3.3 白鹿之灵在第一部中的递进
- 前段：只给半句、只给方向、只给压力；
- 中段：可以指出代价正在被谁推高；
- 后段：在鹿眠内环才可说整句，但依旧只说与白鹿原未来有关的内容。

## 4. 接引法宝边界

### 4.1 第一部必须明确的部分
- 它与白鹿原为什么没有彻底崩塌有关；
- 它与主角回引能力有关；
- 它不是简单能源，而是受损的未来比较装置；
- 它要求人为未来签字，而不是自动给出正确答案。

### 4.2 第一部不能讲满的部分
- 它在古战前的完整建制；
- 它与整个飞升体系的完整关系；
- 它与更高层宇宙秩序的完整关系；
- 它在其他主世界或其他伤区中的对应系统。

## 5. 主角边界

### 5.1 第一部可成立的主角定位
- 小世界飞升者；
- 被异常坠入白鹿原；
- 具备高适配度的替代锚点潜质；
- 其价值在于被写入地方未来，而非立刻站上主世界顶层。

### 5.2 第一部禁止的主角定位
- 古裂残天命定救世主；
- 九主世界关键预言中的终极主角；
- 白鹿之灵选中的唯一神谕者；
- 可以自由进出所有秩序层级的万能通行者。

## 6. 历史边界

### 第一部必须揭示
- 白鹿原被主动推离的事实；
- 紧急治理延续至今的事实；
- 旧誓约对今天仍有影响的事实；
- 主角的回返代价与未来比较机制。

### 第一部必须后置
- 古战双方全部动机；
- 古裂残天大战全过程；
- 古裂残天所有伤区因果链；
- 九主世界在大战中的全局位置。

### 7.2 揭示层级调用规则

1. 任何“历史解释”都必须先挂在六区的现实承压面上，再抬到旧誓约、旧法统或更大秩序压力。
2. 白鹿之灵与受损法宝只能在现实压力已经成立后，帮助比较代价、放大后果，不能反过来替现实压力开场。
3. `50–58` 后续蓝图层继承本表时，应先按“前段看到什么 / 中段才升级什么 / 后段才进入什么”排揭示顺序，不得一开始把后段信息讲满。
4. 这张表服务第一部强绑定边界，不把白鹿原升格为整个古裂残天的总压力清单。

### 7.5 地区锚点绑定规则

1. 前段若没有同时落地鹿门集、回声渠与断引栈道三者的现实代价，后续任何关于白鹿原“为何必须现在处理”的说法都不能算成立。
2. 中段若没有把黑砂外缘的伤亡成本与伏誓碑林的调证成本同时压实，就不允许把旧签字、旧法统或灰缄线争议抬成主要冲突。
3. 后段进入鹿眠内环之前，必须先让鹿门集的承压名单与伏誓碑林的旧证边界已经成立；否则内环只能算设定越界，不算合法推进。
4. 任何“有限重开”都必须同时带着断引栈道的路权代价、伏誓碑林的旧债追索与鹿眠内环的余量排序出现，缺一项都只能算局部波动，不算第一部合法终局推进。
5. `21 / 22 / 50–58` 继承本组表时，只能在这里规定的地区组合内细化角色职责、揭示顺序与状态变化，不能反过来删掉某个区域，让白鹿原退回单线舞台。
