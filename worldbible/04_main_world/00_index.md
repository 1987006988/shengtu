# 04 主世界索引

## 1. 模块定位

`04_main_world` 负责主世界承载层。

当前默认工作重点是：

1. 把 `古裂残天` 从概念主世界升级为长期承重主世界
2. 把 `白鹿原` 从功能地图升级为地区圣经
3. 明确第一部《升途：封绝之地》在 `古裂残天` 中的绑定边界

本模块不能退化成只服务旧任务文本包的下游设定目录。

## 2. 当前默认入口

进入本模块后，默认先读：

1. `14_gulie_cantian_rebuild_bible.md`
2. `15_bailuyuan_region_bible.md`
3. `16_book1_binding_boundary.md`
4. `17_nine_main_world_longterm_usage_matrix.md`

## 3. 文件口径

### 3.1 当前主入口文件

- `14_gulie_cantian_rebuild_bible.md`：当前 `古裂残天` 总层母本
- `15_bailuyuan_region_bible.md`：当前 `白鹿原` 地区圣经母本
- `16_book1_binding_boundary.md`：第一部绑定边界与揭示边界
- `17_nine_main_world_longterm_usage_matrix.md`：九主世界长期使用矩阵

### 3.2 仍可保留的上游正式文件

- `01_main_world_foundations.md`
- `05_story_main_world_draft.md`
- `06_main_world_comparison.md`
- `07_nine_main_worlds_difference_matrix.md`
- `12_gulie_cantian_macro_structure.md`

这些文件仍具上游参考价值，但若与 `14–17` 冲突，以 `14–17` 为准。

### 3.3 已被新母本覆盖的旧入口

- `11_bailuyuan_faction_landscape.md`
- `13_bailuyuan_functional_map.md`

这两份文件保留为白鹿原地区圣经重建前的历史草图，只可回收局部结构与功能关系，不再作为当前主入口真相。

## 4. 当前允许做什么

- 继续补强 `古裂残天` 的长期承载能力
- 继续补强 `白鹿原` 的地区圣经密度
- 继续明确第一部绑定与不绑定的范围

## 5. 当前明确禁止

- 用旧功能地图直接替代地区圣经
- 让任务层、对白层或系统文本层反向决定主世界层
- 把 `白鹿原` 上浮成整个 `古裂残天`
