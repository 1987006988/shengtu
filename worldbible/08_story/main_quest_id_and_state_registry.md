# 第一款游戏主线任务 ID 与状态注册表

## 1. 文档定位

本文档对应 `B05：主线任务架构与 ID 体系` 的另一份主输出。

它负责把 `32_main_quest_architecture.md` 的总层任务树落成可逐条调用的注册表，供后续主线正文、Journal、失败反馈、同伴插话与 review 直接索引。

本文档不是正文，也不是只列 ID 的空壳表。每个 `MQ` 必须至少携带：

- 来源场景卡
- 进入条件
- 玩家目标
- 关键选择
- 成功写回
- 失败回流
- Journal 锚
- 白鹿之灵接口

## 2. 注册表总规则

### 2.1 编号规则

- `MQ01–MQ48`：主线任务单元
- `CH01–CH17`：章节单元
- `CL01–CL14`：任务簇
- `JR-MQ01–JR-MQ48`：MQ 级 Journal 更新锚
- `JR-CL01–JR-CL14`：任务簇汇总锚

### 2.2 六项状态缩写

- `TRUST`：地方信任
- `ORDER`：残秩序合法性
- `GRAY`：灰线接触
- `GATE`：入口带稳定
- `RELIC`：法宝承压
- `PARTY`：同伴立场压力

### 2.3 写回规则

- `成功写回` 至少包含 `1` 项状态变化。
- `失败回流` 至少包含 `1` 项状态恶化或证据残缺。
- `Journal 锚` 只能写当前已成立的问题，不得提前泄露后段真相。

## 4. MQ 注册表

#### MQ01：坠落失衡

- 章节 / 任务簇：`CH01` / `CL01`
- 源场景卡：`SC01`、`SC03`
- 进入条件：飞升完成但接引落点异常，主角团未接入正常秩序带。
- 玩家目标：在高位失衡环境中站稳并确认异常不是幻觉。
- 关键选择：先稳自身位置还是先搜索失散席位。
- 成功写回：`RELIC` 初值建立；`PARTY` 开启低阶观察。
- 失败回流：高位压迫后遗加重，后续证词口气更尖锐。
- Journal 锚：`JR-MQ01` 记录“飞升落点偏移”“高位环境失衡”。
- 白鹿之灵接口：只允许识别异常，不解释成因。
