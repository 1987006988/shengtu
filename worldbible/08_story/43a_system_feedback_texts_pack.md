# 43A System 文本与最终整合准备包

## 1. 文档定位

本文档对应拆分后子批次 `B15B：System 文本与最终整合准备 B` 的唯一主输出。

它负责把 `42a_items_relics_and_documents_pack.md` 已锁定的物件 / 文书调用层，继续压成可供 UI、系统播报、交互检查、失败回流、阶段广播与最终验收前联调使用的 system feedback 文本资产，并显式生成一份只能服务“进入最终验收前检查”的缺口清单。

本文档承担的结构任务是：

- 承接 `42a` 的法宝 / 任务物件 / 灰线物件 / 文书底盘，不回头重写条目正文。
- 承接 `40a` 的 Codex 解锁层与 `41a` 的 Journal / 更新层，补上“玩家当下看见什么、系统当下怎么提示”的即时反馈层。
- 承接 `35b`、`37a`、`38a` 与 `17` 已锁定的终局代价、支线余压、灰线缺口与同伴条件支持，把这些内容压成可直接调用的系统反馈，不让系统层越权替代正文判断。
- 在不进入最终验收轮的前提下，明确列出进入 `END` 前必须完成的 final readiness 缺口检查项。

本文档不做：

- 不重写 `42a` 的物件 / 文书正文。
- 不把 `40a`、`41a` 的 Codex / Journal 正文再压写一遍。
- 不新增世界级、主世界级或 `古裂残天` 总层设定。
- 不让 system feedback 代替主线、支线、区域或同伴正文。
- 不直接产出最终验收结论；本文件只负责“验收前缺口层”，不负责“验收通过”。

## 2. 使用规则

### 2.1 system 层允许承担的功能

本包中的 system 文本只允许承担以下功能：

- 提示玩家“发生了什么变化”。
- 提示玩家“当前可查看 / 可复查 / 可回收 / 可归档的是什么”。
- 提示玩家“当前风险提升在哪里、哪一项状态正在承压”。
- 提示玩家“失败没有被抹掉，而是被转写成新的调查、回流或延迟处理节点”。
- 提示玩家“某位同伴、某个区域、某条支线现在为什么会以不同语气回看同一件事”。

本包中的 system 文本不得承担以下功能：

- 宣布哪条路线天然正确。
- 替玩家对承担者比较做道德裁决。
- 用白鹿之灵直接解释完整真相。
- 用 UI 提示替代正文冲突。
- 提前把 `END` 批次的最终验收语气写成“项目已完成”。

### 2.2 状态锚点统一口径

本包只使用现有六项主状态作为锚点，不创建平行状态体系：

- `TRUST`：地方承接与彼此信任相关反馈。
- `ORDER`：资格、排布、执行面秩序相关反馈。
- `GRAY`：灰线、反证、外缘渗透与不透明交换相关反馈。
- `GATE`：区域边界、入口残带、放行 / 回撤相关反馈。
- `RELIC`：法宝承压、回返余痕、阈值示警与事故后处理相关反馈。
- `PARTY`：同伴支持条件、组合态、尾声余压与队内感受差异相关反馈。

### 2.3 窗口缩写

- `W1`：`MQ01–MQ12`
- `W2`：`MQ13–MQ24`
- `W3`：`MQ25–MQ38`
- `W4`：`MQ39–MQ48`
- `EP`：尾声后置留存窗口

### 2.4 白鹿之灵边界

凡涉及白鹿之灵的 system 提示，只允许表现为：

- 低强度回响
- 阈值示警
- 残忆式提示
- 边界接近提醒
- 回返代价未清零提示

不得表现为：

- 世界意志发言
- 路线裁判
- 绝对安全承诺
- 无代价复生许可
- “你现在该怎么选”的直接代答

## 3. 即时反馈文本包

### 3.1 获得 / 解锁 / 录入反馈

#### SYS-ACQ-01：法宝余痕录入

- `触发条件`：首次获得 `ITM-REL-01 回返余痕片` 或同类承压留痕物件。
- `状态锚点`：`RELIC`
- `短提示`：`已记录：回返余痕正在累积，相关说明已收入档案。`
- `展开提示`：`你拿到的不是单纯残片，而是一笔仍在继续生效的回返留痕。物件说明、关联 Journal 与后续阈值提示已开放查看。`
- `高压变体`：`记录完成：回返余痕密度升高，后续相关提示将更频繁出现。`
- `调用边界`：只用于提示“承压已被系统看见”，不得直接替代复生机制说明正文。

#### SYS-ACQ-02：资格物件录入

- `触发条件`：首次获得救急牌、资格改列签、放行牌等排布类物件。
- `状态锚点`：`TRUST + ORDER`
- `短提示`：`已录入：一项“谁能先被看见”的排布凭据。`
- `展开提示`：`该物件不会替你解决承压，只会让原本被藏起来的排序逻辑变得可见。相关区域、支线或争议窗口已追加注记。`
- `低信任变体`：`已录入：此类凭据会改变他人对队伍立场的判断。`
- `调用边界`：不得把录入提示写成“资源问题已解决”。

#### SYS-ACQ-03：灰线反证录入

- `触发条件`：首次获得风风险票据、暗债折记、反证抄片、借道记录等灰线物件。
- `状态锚点`：`GRAY`
- `短提示`：`已录入：灰线证据进入可复查状态。`
- `展开提示`：`这类记录不是旁证装饰，而是某种交换、转嫁或隐藏后果留下的硬痕。相关灰线节点、对话口风与失败回流已开放联动。`
- `强灰线变体`：`已录入：新的反证会扩大对既有说法的冲击范围。`
- `调用边界`：不得把“证据已录入”写成“真相已经全部判定”。

#### SYS-ACQ-04：文书归档开启

- `触发条件`：首次获得可归档文书、抄件、值守记录、封存条、共监留痕环对应文书。
- `状态锚点`：`ORDER + RELIC`
- `短提示`：`已归档：文书类记录可在后续节点复查。`
- `展开提示`：`这份文书的价值不在于替谁洗白，而在于把“谁知道、谁签过、谁承担过”固定下来。归档后可在相关结算或追责节点再次调取。`
- `尾声变体`：`已归档：该记录将在尾声阶段以留痕方式继续生效。`
- `调用边界`：不得写成“归档后争议自动结束”。

#### SYS-ACQ-05：Codex 联动解锁提示

- `触发条件`：物件获取同时触发 `40a` 中相关词条开放。
- `状态锚点`：`TRUST / ORDER / GRAY / GATE / RELIC`
- `短提示`：`相关条目已开放：可在 Codex 查看背景与边界。`
- `展开提示`：`系统已把你刚接触到的物件与既有条目关联。Codex 会补充“它为何存在”，但不会替代你眼前这场冲突的现实代价。`
- `调用边界`：不得暗示 Codex 提供超阶段真相。

#### SYS-ACQ-06：Journal 追踪锚点建立

- `触发条件`：物件 / 文书获取同时在 `41a` 中生成或推进 Journal 锚点。
- `状态锚点`：`TRUST + ORDER + GRAY + GATE + RELIC`
- `短提示`：`日志已更新：新线索进入追踪。`
- `展开提示`：`这次获取不会单独闭环，它只把一条原本分散的线索重新拽到可追踪范围内。Journal 已记录下一步要验证的对象。`
- `调用边界`：不得把追踪提示写成明确剧透。

### 3.2 调查 / 复查 / 查验反馈

#### SYS-INV-01：二次查看成功

- `触发条件`：玩家对 `42a` 中已录入物件执行二次查看。
- `状态锚点`：沿用该物件原状态锚点
- `短提示`：`已复查：更多细部痕迹被确认。`
- `展开提示`：`你不是第一次看见它，所以这次系统不再重复基础说明，而是把之前容易被忽略的细部、口风或留痕重新压到前台。`
- `调用边界`：只用于“复查后增量反馈”，不得把复查写成全新掉落。

#### SYS-INV-02：来源链校验通过

- `触发条件`：物件来源、Journal 锚点与区域节点之间的调用链匹配。
- `状态锚点`：`ORDER`
- `短提示`：`来源链已校验：调用关系成立。`
- `展开提示`：`该条目与既有任务、区域或文书链条一致，后续可以稳定用于结算、播报或复盘，不需要额外补建平行说明。`
- `调用边界`：面向内部联调或后台状态确认，不直接在强剧情时刻频繁弹出。

#### SYS-INV-03：来源链缺项提醒

- `触发条件`：物件已存在，但缺少对应 Journal、Codex 或区域侧反馈挂接。
- `状态锚点`：`ORDER + GATE`
- `短提示`：`发现缺项：该条目的调用链尚未闭合。`
- `展开提示`：`物件本身已入库，但它还没有完整接上应有的日志、条目或区域回显。此提示只能作为整合准备提醒，不得直接对玩家表现为断裂文本。`
- `调用边界`：主要用于 final readiness 联调，不作为常规玩家提示。

#### SYS-INV-04：争议记录对照

- `触发条件`：同一事件存在主线说法与灰线反证说法并列时。
- `状态锚点`：`GRAY + ORDER`
- `短提示`：`记录对照开启：同一事件存在不一致说法。`
- `展开提示`：`系统已确认这不是重复文本，而是现实立场差异留下的并行记录。你可以继续追查，但系统不会替你先行裁决哪一边更干净。`
- `调用边界`：不得把系统写成真相仲裁者。

#### SYS-INV-05：承压物件异常复查

- `触发条件`：承压类物件在 `W3/W4` 出现异常增长、变色、封存升级或共监留痕追加。
- `状态锚点`：`RELIC`
- `短提示`：`异常复查：承压痕迹出现新变化。`
- `展开提示`：`这不是新物件，而是旧物件在更重的现实压力下露出了新的后果面。相关事故、阈值与后续处置提示已同步刷新。`
- `调用边界`：不得写成“法宝已自行修复”。

### 3.3 区域 / 边界 / 阈值预警反馈

- `触发条件`：入口残带相关 `GATE` 状态过低或放行牌被争议调用。

### 3.5 同伴 / 路线敏感反馈

- `触发条件`：不同承担方案开始明显牵动同伴对承担者比较的语气。
- `状态锚点`：`PARTY + RELIC + ORDER`
- `短提示`：`路线压力显影：队内对“谁先承担”出现更明确感受。`
- `展开提示`：`系统只确认差异正在变明显，不替任何一路背书。后续差异主要体现在对话、尾声评语与条件支持变化。`
- `调用边界`：不得把 system 提示写成路线推荐。

#### SYS-PTY-05：组合态回响

- `触发条件`：特定同伴组合在关键节点触发组合态反应。
- `状态锚点`：`PARTY`
- `短提示`：`组合回响生效：当前同行组合会改变反馈口风。`
- `展开提示`：`这说明不是单人态度变化，而是某种队内关系结构正在影响同一事件的观看方式。系统只负责告诉你“它存在”，具体锋利处仍留给正文。`
- `调用边界`：不得代替 banter 或营地正文。

### 3.6 失败回流 / 延迟处理反馈

#### SYS-FAL-01：失败未清零

- `触发条件`：主线、支线或区域节点失败，但未导致流程终止。
- `状态锚点`：对应失败节点原状态
- `短提示`：`失败已写回：后续流程继续，但痕迹保留。`
- `展开提示`：`系统不会把这次失手吞掉。它已被转写为新的追查、补救、追责或更冷的回应，后续文本会继续记得它。`
- `调用边界`：不得把失败写成无成本 retry。

#### SYS-FAL-02：撤离但未脱账

- `触发条件`：玩家从高压区域主动撤离或被迫回撤。
- `状态锚点`：`GATE + RELIC`
- `短提示`：`已撤离：离开不等于这笔账不存在。`
- `展开提示`：`你暂时脱离了现场，但相关边界、封存、追责或阈值提示会继续跟着这次撤离行动。`
- `调用边界`：不得把回撤写成干净脱身。

#### SYS-FAL-03：延迟处理成立

- `触发条件`：某问题被临时封存、转后处理、挂账到尾声。
- `状态锚点`：`ORDER + RELIC + TRUST`
- `短提示`：`处理被延后：问题被按住，不是被解决。`
- `展开提示`：`这是一种现实中的勉强托底，而不是乐观收束。后续条目会以“留存问题”的形式继续存在。`
- `调用边界`：不得写成温和版成功结算。

#### SYS-FAL-04：证据不足

- `触发条件`：灰线或追责节点证据未达阈值。
- `状态锚点`：`GRAY + ORDER`
- `短提示`：`证据不足：现有记录不足以支撑更重结论。`
- `展开提示`：`系统确认你已经摸到问题边缘，但还不能把它钉死。后续仍可继续追查、比对或在尾声留存。`
- `调用边界`：不得写成“玩家走错路线”的惩罚提示。

#### SYS-FAL-05：承接失败的地方反馈

- `触发条件`：地方承接层无法继续托底或公开表示疲惫 / 冷却。
- `状态锚点`：`TRUST`
- `短提示`：`地方反馈转冷：这次没有更多余量替你们兜底。`
- `展开提示`：`这并不一定是敌意，而是某种现实疲劳终于说出口。求助、救急与排序类反馈会明显变得更硬。`
- `调用边界`：不得把地方群体写成突然反派化。

#### SYS-FAL-06：尾声缺口转入留存

- `触发条件`：某失败 / 未完事项在尾声中被转为缺口留存而非正文继续处理。
- `状态锚点`：`PARTY + TRUST + RELIC`
- `短提示`：`缺口已留存：它不会在本轮被补平。`
- `展开提示`：`系统确认该问题仍在未来里继续生效。它不会再占用主流程目标，但会作为尾声余压、区域背景或同伴评语继续存在。`
- `调用边界`：不得写成“未完成内容已无关紧要”。

### FR-01：主线尾段 system 挂接完整性

- `检查对象`：`MQ35–MQ48` 的系统反馈、Journal 提示、物件录入、失败回流提示。

- `失败征兆`：只有主线正文，没有对应系统提示或结算播报。
- `关联资产`：`35b`、`41a`、本文件 `SYS-PROG-* / SYS-FAL-*`。

### FR-02：支线正文与系统反馈闭环

- `检查对象`：`SQ01–SQ12` 与区域 / Journal / 系统提示联动。
- `通过条件`：每条支线完成、失败、延迟处理至少各有一条可调用反馈。
- `失败征兆`：支线正文已写，但系统层对失败或尾声留存无反馈。
- `关联资产`：`37a`、`38a`、`41a`、本文件 `SYS-PROG-02 / SYS-FAL-01 / SYS-FAL-06`。

### FR-03：物件二次查看与重估链

- `检查对象`：`42a` 中高频物件的二次查看、异常复查、尾声重估提示。
- `通过条件`：重点物件至少具备首次录入与后续重估两层反馈。
- `失败征兆`：物件只在掉落时出现一次，后续完全失语。
- `关联资产`：`42a`、本文件 `SYS-INV-01 / SYS-INV-05 / SYS-PROG-04`。

### FR-04：Codex / Journal / system 三层分工

- `检查对象`：背景说明、当下追踪、即时提示三层是否混淆。
- `通过条件`：Codex 讲背景边界，Journal 讲追踪目标，system 讲当下变化。
- `失败征兆`：system 文本承担大段百科说明，或 Journal 直接承担 UI 播报。
- `关联资产`：`40a`、`41a`、本文件 `SYS-ACQ-05 / SYS-ACQ-06`。

### FR-05：白鹿之灵边界强度

- `检查对象`：全部涉及白鹿之灵的 system 提示。
- `通过条件`：只停留在示警、回响、边界提醒与代价未清零提示。
- `失败征兆`：出现解释真相、裁判路线、宣布安全或代答选择的句子。
- `关联资产`：`canon_rules.md`、`40a`、`42a`、本文件 `SYS-WARN-02`。

### FR-06：白鹿原尺度锁定

- `检查对象`：system 播报的地理与叙事尺度。
- `通过条件`：全部锁在 `白鹿原` 隔绝地域与第一部游戏范围。
- `失败征兆`：把一次放行、事故或尾声提示上浮为整个 `古裂残天` 总命运线。
- `关联资产`：`canon_rules.md`、`35b`、本文件全局规则。

### FR-07：失败回流不是 retry 占位

- `检查对象`：全部失败 / 回撤 / 延迟处理提示。
- `通过条件`：每条失败提示都明确说明“痕迹保留、后果转写”。
- `失败征兆`：出现“请重新尝试即可”“失败不影响后续”等空提示。
- `关联资产`：本文件 `SYS-FAL-*`。

### FR-08：地方承接层现实感

- `检查对象`：`TRUST` 相关提示与区域口风变化。
- `通过条件`：地方反馈呈现疲劳、托底、收紧、保留帮助等现实差异。
- `失败征兆`：所有地方反馈只分“欢迎 / 敌对”两档。
- `关联资产`：`39`、`37a`、本文件 `SYS-WARN-05 / SYS-PROG-03`。

### FR-09：灰线反证保持“不判死”

- `检查对象`：灰线证据、反证抄片、暗债记录相关提示。
- `通过条件`：system 只确认矛盾存在，不替玩家先判定唯一真相。
- `失败征兆`：一条系统提示直接宣布灰线问题盖棺。
- `关联资产`：`38a`、`42a`、本文件 `SYS-ACQ-03 / SYS-INV-04 / SYS-FAL-04`。

### FR-10：法宝承压与事故后处理分层

- `检查对象`：承压示警、封存、共监、异常复查提示。
- `通过条件`：示警、事故、封存、追责四层用语清楚，不相互代替。
- `失败征兆`：把封存写成修复，把留痕写成免责，把共监写成立即清算。
- `关联资产`：`42a`、本文件 `SYS-WARN-02 / SYS-WARN-03 / SYS-WARN-06 / SYS-INV-05`。

### FR-11：同伴差异未被系统层抹平

- `检查对象`：支持条件、保留意见、低信任协作、组合态回响。
- `通过条件`：system 只确认差异存在，不把队内复杂度压成单一立场。
- `失败征兆`：所有同伴提示都变成“全队赞同”或“全队反对”。
- `关联资产`：`17`、`20a`、本文件 `SYS-PTY-*`。
