# 32 第一款游戏主线任务架构

## 1. 文档定位

本文档对应 `B05：主线任务架构与 ID 体系` 的主输出之一。

它负责把 `25_volume1_core_conflict.md`、`26_volume1_revelation_ladder.md`、`27_volume1_plot_spine.md`、`28_volume1_chapter_outline.md`、`29a_volume1_scene_cards_act1_act2.md`、`29b_volume1_scene_cards_act3_epilogue.md` 与 `29_volume1_scene_cards.md` 统一压成可供后续主线正文、Journal、同伴插话与系统反馈直接调用的主线任务架构。

本文档不是主线正文，不是小说章节回顾，也不是只给几个示例的任务提案。它必须回答以下问题：

- 《升途：封绝之地》第一款游戏的主线任务树如何分层。
- `MQ01–MQ48` 如何按 Act、章节与任务簇组织成可生产的任务单元。
- 每组任务单元的现实目标、结构问题、关键选择、状态写回与 Journal 更新位如何绑定。
- 白鹿之灵在哪些节点允许出现，哪些节点必须完全由现实证据、地方承压、灰线行动或法宝后果承担。

本文档服务 `白鹿原` 第一款游戏尺度。所有任务目标都只收束到 `白鹿原` 的存续、承压、路径比较与受限未来，不得上浮为整个 `古裂残天` 的命运线。

## 3. 主线任务树总览

### 3.1 总层级

本作主线任务树固定采用四层：

1. `Act 层`
2. `章节层`
3. `任务簇层`
4. `MQ 单元层`

调用规则：

- `Act 层` 负责定义本幕核心压力与揭示目标。
- `章节层` 负责把同一幕中的推进节拍压成可生产的章节边界。
- `任务簇层` 负责把若干场景卡整合为可调用的任务功能单元。
- `MQ 单元层` 是后续正文、Journal、失败反馈与系统调用的最小主线任务骨架。

### 3.2 总量

- Act：`4` 幕 + `1` 个 Epilogue
- 章节：`17`
- 任务簇：`14`
- MQ 单元：`48`
- 场景卡：`80`

### 3.4 任务树主原则

- 任务簇不是章节复述，而是 `现实目标 + 结构问题 + 状态写回` 的组合单元。
- 每个任务簇都必须带有可失败、可变体、可写回的结果。
- 每个 MQ 单元都必须能回指到具体场景卡。
- 若某个任务单元无法说明“失败后哪项状态变坏”，则它还不是合格的游戏任务骨架。

## 4. ID 体系与调用约束

### 4.1 主 ID

- `ACT01–ACT04`：主线 Act 编号
- `EP01`：尾声编号
- `CH01–CH17`：章节编号
- `CL01–CL14`：任务簇编号
- `MQ01–MQ48`：主线任务单元编号

### 4.2 辅助调用 ID

- `JR-MQ01` 至 `JR-MQ48`：对应 MQ 的 Journal 更新锚
- `JR-CL01` 至 `JR-CL14`：对应任务簇完成后的阶段性 Journal 汇总锚
- `SC01–SC80`：场景卡来源锚，不可省略

### 4.3 状态写回缩写

为后续 UI、Journal、系统反馈与 review 对齐，本批统一采用以下六项状态写回缩写：

- `TRUST`：`地方信任`
- `ORDER`：`残秩序合法性`
- `GRAY`：`灰线接触`
- `GATE`：`入口带稳定`
- `RELIC`：`法宝承压`
- `PARTY`：`同伴立场压力`

这些缩写只服务工程调用，不替代正式中文术语。

### 4.4 任务单元最小字段

后续任何正文、系统或 Journal 调用 `MQ` 时，至少必须带出：

- `源场景卡`
- `进入条件`
- `节点目标`
- `关键选择`
- `成功写回`
- `失败回流`
- `Journal 锚`
- `白鹿之灵接口`

少于这八项，即视为重新把游戏任务压回叙事摘要。

## 5. 各任务簇架构

### 5.1 `CL01` 坠落求生簇

- 章节 / 范围：`CH01`；`MQ01–MQ03`
- 源场景卡：`SC01–SC05`
- 簇目标：让玩家从“飞升成功预期”坠入“高位失衡求生现实”，并第一次确认回返有代价。
- 结构任务：
  - 证明白鹿原不是普通废墟。
  - 证明主角团已被异常接引写入此地。
  - 把“继续求救”改写成“先在失衡里活下来”。
- 关键选择：
  - 先回收失散席位，还是先稳住高位压迫点。
  - 第一次回返后，是把异常当工具，还是当高风险绑定。
- 状态写回重点：
  - `RELIC` 必开。
  - `PARTY` 进入低阶观察。
- Journal 规则：
  - 只能记录“落点偏移”“残秩序仍在”“回返有代价”三项事实。
  - 不得提前写出法宝完整用途。

### 5.2 `CL02` 生存托底簇

- 章节 / 范围：`CH02`；`MQ04–MQ07`
- 源场景卡：`SC06–SC11`
- 簇目标：让玩家确认白鹿原有人活着，且这种活法依赖地方承压与法宝托底双重结构。
- 结构任务：
  - 把聚落拒识写成现实防御，不写成新手村礼节。
  - 把地方承压写成行动事实，不写成设定说明。
  - 把法宝托底写成结构支点，不写成万能异常源。
- 关键选择：
  - 优先处理现实承压，还是优先追查法宝异常。
  - 如何在代价换路与缓冲圈试探间分配风险。
- 状态写回重点：
  - `TRUST` 首次进入显性升降。
  - `ORDER` 从模糊印象变成“有用但不无害”。
  - `RELIC` 从个人异常扩展到地域压力。
- Journal 规则：
  - 必须记录“这里有人活着”“代价由谁在扛”“法宝也许在撑着这里，但继续逼近会出后果”。

### 5.3 `CL03` 封绝定性簇

- 章节 / 范围：`CH03`；`MQ08–MQ10`
- 源场景卡：`SC12–SC16`
- 簇目标：把白鹿原从“坏掉的接引落点”定性为“被推离常规秩序带的隔绝地域”。
- 结构任务：
  - 证明入口残带是结构阀门，不是出口。
  - 证明外界不会直接收束本作问题。
  - 让第一幕以尺度锁定收束，而不是以战斗胜利收束。
- 关键选择：
  - 强行验证入口，还是保存通路稳定度。
  - 接受白鹿原尺度锁定，还是继续把外界求援当主目标。
- 状态写回重点：
  - `GATE` 首次显性记录。
  - `TRUST` 与 `ORDER` 的解释基础被更新。
  - `PARTY` 对“留下还是找出口”产生第一轮明确分野。
- Journal 规则：
  - 必须从“寻找离开方法”改成“理解白鹿原为何仍能存在”。

### 5.4 `CL04` 地方运转簇

- 章节 / 范围：`CH04`；`MQ11–MQ12`
- 源场景卡：`SC17–SC21`
- 簇目标：把主聚落与凡俗承重面拉上桌，防止第二幕退化成地理跑图。
- 结构任务：
  - 把地方共同体写成持续承压单位。
  - 把事故救援、分配取舍与营地现实争议写成同一条任务链。
- 关键选择：
  - 先稳现实运转，还是先追踪异常源头。
  - 资源分配是按救急、按资格还是按结构风险排序。
- 状态写回重点：
  - `TRUST` 进入中段曲线。
  - `PARTY` 围绕“现实优先还是深因优先”拉开。
- Journal 规则：
  - 不得只写“调查主聚落”；必须写清哪类问题被解决、哪类问题只是被延后。

### 5.5 `CL05` 地方坐大簇

- 章节 / 范围：`CH05`；`MQ13–MQ14`、`MQ20`
- 源场景卡：`SC22–SC26`
- 簇目标：说明地方势力为何依附坐大，并把风险处置权写成结构权力。
- 结构任务：
  - 把交换带、风险估值与处置权争夺绑为一个完整弧线。
  - 把地方依附残序的现实理由写深，而不是写成逐利标签。
- 关键选择：
  - 借用地方既有秩序止损，还是拆开其处置权链条。
  - 允许地方继续依附坐大，还是用更高代价换来短期纠偏。
- 状态写回重点：
  - `ORDER` 进入现实重估。
  - `TRUST` 与 `ORDER` 开始互相拉扯。
  - `PARTY` 围绕“维持可运行结构”还是“先拆坏结构”产生差异。
- Journal 规则：
  - 必须记录“风险定价”“处置权”“依附与保护并存”三层事实。

### 5.6 `CL06` 流动托底簇

- 章节 / 范围：`CH06`；`MQ15–MQ17`
- 源场景卡：`SC27–SC31`
- 簇目标：把通路、节点与入口残带写成白鹿原续命阀门。
- 结构任务：
  - 把保通与封口做成真正的结构选择。
  - 让入口带稳定度成为中段核心状态，而不是环境背景值。
- 关键选择：
  - 优先保通、优先控险，还是做高代价折中。
  - 如何处理资格盘查对地方与边缘群体的不同后果。
- 状态写回重点：
  - `GATE` 成为中段核心状态。
  - `TRUST` 与 `GATE` 绑定。
  - `GRAY` 打开前夜窗口。
- Journal 规则：
  - 必须把“保通不是便利，是保命”写成任务事实。

### 5.7 `CL07` 托底翻转簇

- 章节 / 范围：`CH07`；`MQ18–MQ24`
- 源场景卡：`SC32–SC40`
- 簇目标：把飞升差异、异质接口、灰线风闻、托底对账与深因追问压成第二幕终点。
- 结构任务：
  - 让三类飞升差异真正影响任务理解与路线偏向。
  - 让灰线从无到有，但不提前洗白。
  - 把第二幕收束在“问题为何会变成这样”，而不是“局势暂时稳住了”。
- 关键选择：
  - 如何理解异质接口，是当成阴影、变量还是新论据。
  - 面对灰线风闻时，是提前压制、谨慎接触还是顺线追查。
- 状态写回重点：
  - `GRAY` 正式开启。
  - `PARTY` 从观察分歧进入中高位。
  - `RELIC`、`TRUST`、`GATE` 之间的关系被看见。
- Journal 规则：
  - 必须把“托底结构是谁在承担”与“为什么会变成这样”拆成两组未解问题。

### 5.8 `CL08` 灰线显形簇

- 章节 / 范围：`CH08–CH09`；`MQ25–MQ27`
- 源场景卡：`SC41–SC48`
- 簇目标：让灰线从第二幕传闻正式升级为第三幕结构证据。
- 结构任务：
  - 把灰线活动与节点、裂缝、入口边界相连。
  - 把“被裹挟者”拉上桌，拆掉纯敌我视角。
  - 把营地裂口作为团队内部认知震荡的第一阶段。
- 关键选择：
  - 面对灰线时是先追缉、先追证，还是先处理被裹挟者。
  - 面对地方恐慌时是压消息、控边界，还是让真相先露头。
- 状态写回重点：
  - `GRAY` 由风险线转入认知线。
  - `GATE` 与 `GRAY` 联动恶化或稳住。
  - `PARTY` 因认知裂口而抬升。
- Journal 规则：
  - 只能写“灰线不再只是传闻，它沿节点、裂缝与边界代价持续活动”。

### 5.9 `CL09` 法宝翻面簇

- 章节 / 范围：`CH10–CH11`；`MQ28–MQ33`
- 源场景卡：`SC49–SC56`
- 簇目标：把异质承认、法宝承认链、接引残忆、推离成因、双方抓手与复生来源压成同一段翻面结构。
- 结构任务：
  - 证明法宝涉及承认链，而不只是力量或通道。
  - 证明白鹿原与法宝是一同被推离常规秩序带的残存支点。
  - 把复生来源从机制说明变成结构代价。
- 关键选择：
  - 承认链争执中，优先保旧制稳定还是优先承认异质证据。
  - 残忆读取要多深，愿意承受多少 `RELIC` 与认知代价。
- 状态写回重点：
  - `RELIC` 进入第三幕中心层。
  - `ORDER` 与 `GRAY` 同步重估。
  - `PARTY` 明显抬升。
- Journal 规则：
  - 只允许记录“法宝牵涉承认链”“白鹿原与法宝一同被推离”“回返不是免费”。

### 5.10 `CL10` 绑定回返簇

- 章节 / 范围：`CH12–CH13`；`MQ34–MQ38`
- 源场景卡：`SC57–SC64`
- 簇目标：把主角团从“卷入白鹿原问题”推进到“已经被写入白鹿原未来”。
- 结构任务：
  - 把绑定回灼与留离反证做成不能回避的现实结果。
  - 把反派框架拆面与修复代价预示接到同一条线。
  - 让第三幕在“知道更多，不等于答案更轻”处收束。
- 关键选择：
  - 是否继续高频调用法宝或回返机制。
  - 如何处理团队分歧，是压制、协调还是带条件前进。
- 状态写回重点：
  - `PARTY` 进入终局前夜版本。
  - `RELIC` 逼近高压阈值。
  - `TRUST` 受主角团资格争议影响。
- Journal 规则：
  - 必须明确记录“主角团已被写入白鹿原未来，离场本身也是一种改写”。

### 5.11 `CL11` 代价聚焦簇

- 章节 / 范围：`CH14`；`MQ39–MQ40`
- 源场景卡：`SC65–SC68`
- 簇目标：把第四幕起手重新压回法宝核心与现实承压面，不给“终局前准备”留空泛空间。
- 结构任务：
  - 让法宝回收线、承压面列席、状态对照与阈值示警彼此扣合。
  - 让所有终局路径都先面对现实合法性，而不是先做口号判断。
- 关键选择：
  - 先稳承压面，还是先占法宝线。
  - 面对阈值示警时，是收束调用频率，还是冒险换取路径信息。
- 状态写回重点：
  - `RELIC` 锁入终局高压区。
  - `TRUST`、`GATE` 与 `RELIC` 同台对账。
- Journal 规则：
  - 必须写成“第四幕一开始就在比较代价，而不是准备选项”。

### 5.12 `CL12` 立场压缩簇

- 章节 / 范围：`CH15`；`MQ41–MQ42`
- 源场景卡：`SC69–SC72`
- 簇目标：把路线镜像与七席位明牌压在同一簇里，使团队分歧本身变成路线条件。
- 结构任务：
  - 让条件支持与分歧回压成为任务推进条件，而不是营地闲聊。
  - 让同伴反应真正改变后续合法性与风险版本。
- 关键选择：
  - 是强压统一口径，还是接受条件支持的分裂推进。
  - 是让现实承压主导路线，还是让结构风险主导路线。
- 状态写回重点：
  - `PARTY` 进入终局高段。
  - `RELIC` 与 `PARTY` 出现互相放大效应。
- Journal 规则：
  - 只能记录“团队分歧本身就是路线条件的一部分”。

### 5.13 `CL13` 路径比较簇

- 章节 / 范围：`CH16`；`MQ43–MQ46`
- 源场景卡：`SC73–SC76`
- 簇目标：完成重连、封闭、重启、重组四类路径的承担者比较。
- 结构任务：
  - 把路线从抽象名词压回承压者与后果。
  - 不允许逃避“谁来承担第一责任与首轮伤害”。
- 关键选择：
  - 更偏向重连、继续封闭、重启旧结构还是重组新结构。
  - 面对承压者比较时，是接受有限更坏版本，还是强求零代价答案。
- 状态写回重点：
  - `TRUST`、`ORDER`、`GRAY`、`GATE`、`RELIC`、`PARTY` 全部进入终局判定。
- Journal 规则：
  - 必须写“真正被比较的是承压者和后果，而不是路线名词”。

### 5.14 `CL14` 受限未来簇

- 章节 / 范围：`CH17`；`MQ47–MQ48`
- 源场景卡：`SC77–SC80`
- 簇目标：把终局路线落成白鹿原的受限未来，并回收地方后果、同伴尾声与白鹿回响。
- 结构任务：
  - 明确白鹿原得到了什么、失去了什么、保留了什么风险。
  - 让尾声显示代价被接住或被延后，而不是被清零。
- 关键选择：
  - 不再新增路线，只决定以哪种代价版本落地。
  - 是否允许部分未解风险留作长线压力。
- 状态写回重点：
  - `TRUST`、`GATE` 与地方后果落地。
  - `PARTY` 与 `RELIC` 进入尾声版本记录。
- Journal 规则：
  - 必须写“白鹿原获得受限未来，但代价不会被清零”。

## 6. MQ 单元设计规则

### 6.1 MQ 不是剧情摘要

每个 `MQ` 必须至少包含：

- 一个玩家要处理的现实问题。
- 一个会改变认知或承压结构的任务结果。
- 一个可见的状态写回位。
- 一个失败或变体版本。

以下写法都不合格：

- “玩家得知某真相。”
- “玩家前往某处并继续调查。”
- “玩家与某势力交锋后进入下一章。”

正确的 MQ 写法必须是：

- “玩家为了让某条现实链路继续运转，被迫接触哪层结构代价。”
- “玩家为了验证哪类解释，必须承担哪项状态恶化风险。”
- “玩家完成此单元后，哪些状态、证据、同伴态度或 Journal 目标被改写。”

### 6.2 MQ 与场景卡的关系

- 一个 `MQ` 至少回指 `1` 张核心场景卡。
- 大多数 `MQ` 实际应回指 `2–4` 张场景卡。
- 若某个 `MQ` 无法回指场景卡，说明它是从章纲空降出来的伪任务。
- 若某张“不可删卡”没有被任何 `MQ` 调用，说明后续生产偷删了结构骨架。

### 6.3 MQ 与章节的关系

- 章节是节拍单元，`MQ` 是执行单元。
- 同一章节可容纳多个 `MQ`，但这些 `MQ` 必须围绕同一章的现实压力与结构问题组织。
- 不允许把一个章节切成互不关联的任务碎片。

## 7. 状态写回与 Journal 架构

### 7.1 六项状态的任务化绑定

| 状态 | 前段主写回区 | 中段主写回区 | 后段主写回区 | 终局主写回区 |
| --- | --- | --- | --- | --- |
| `TRUST` | `CL02`、`CL03` | `CL04`、`CL05` | `CL10` | `CL11`、`CL13`、`CL14` |
| `ORDER` | `CL02` | `CL05`、`CL07` | `CL09` | `CL13` |
| `GRAY` | `CL07` 前夜 | `CL08` | `CL09` | `CL13`、`CL14` |
| `GATE` | `CL03` | `CL06` | `CL08` | `CL11`、`CL14` |
| `RELIC` | `CL01`、`CL02` | `CL07` | `CL09`、`CL10` | `CL11–CL14` |
| `PARTY` | `CL01`、`CL02` | `CL04`、`CL07` | `CL08–CL10` | `CL12–CL14` |

### 7.2 Journal 更新位

Journal 在本作只能承担三种功能：

- 记录当前可行动问题。
- 记录已经成立但仍未解的结构判断。
- 记录状态恶化、失败回流与条件支持。

不允许承担的功能：

- 提前公布第三幕或第四幕真相。
- 用一句总结抹掉玩家此前的选择差异。
- 替代主线任务本身的推进。

### 7.3 任务簇级 Journal 更新

每个任务簇至少需要一个簇级 Journal 汇总位：

- `JR-CL01`：飞升落点偏移、残秩序试压、回返代价成立。
- `JR-CL02`：地方承压与法宝托底双重结构成立。
- `JR-CL03`：白鹿原尺度锁定，外界不会直接收束。
- `JR-CL04`：现实承重与地方运转成为主线正题。
- `JR-CL05`：风险处置权与地方依附逻辑成立。
- `JR-CL06`：保通 / 封口成为结构选择。
- `JR-CL07`：灰线与深因追问窗口被正式打开。
- `JR-CL08`：灰线成为结构证据，不再只是传闻。
- `JR-CL09`：法宝牵涉承认链、推离成因与回返成本。
- `JR-CL10`：主角团已被写入白鹿原未来。
- `JR-CL11`：第四幕从承压面比较起步。
- `JR-CL12`：团队分歧成为路线条件。
- `JR-CL13`：真正被比较的是承担者与后果。
- `JR-CL14`：白鹿原获得受限未来，代价不清零。

## 8. 白鹿之灵硬边界

### 8.1 允许出现的任务簇

白鹿之灵只允许在以下任务簇里有限出现：

- `CL01`：绑定异常与回返代价
- `CL02`：法宝托底短触
- `CL09`：承认链显影、残忆拼片、回返成本
- `CL10`：绑定回灼与承压上限
- `CL11`：阈值示警
- `CL14`：尾声余响

### 8.2 原则上不得承担主推进的任务簇

- `CL04` 地方运转簇
- `CL05` 地方坐大簇
- `CL06` 流动托底簇
- `CL08` 灰线显形簇
- `CL12` 立场压缩簇
- `CL13` 路径比较簇

这些部分必须由地方现实、结构压力、灰线证据、团队分歧与法宝后果推动，而不是由白鹿之灵替代现实逻辑。

### 8.3 不合格写法

- 用白鹿之灵讲完整大战真相。
- 用白鹿之灵给路线背书。
- 用白鹿之灵替玩家赢得地方信任。
- 用白鹿之灵解释灰线动机。
- 用白鹿之灵让回返无代价。

## 9. 对 B06 的直接接口

### 9.1 B06 的实际展开边界

`B06` 只允许扩写：

- `MQ01–MQ12`
- 对应 `CL01–CL04`
- 对应 `CH01–CH04`

不得提前进入：

- `MQ13+`
- `CL05+`
- 第二幕中后段、第三幕、第四幕或尾声正文

### 9.2 B06 必须继承的任务层要求

- 每个 `MQ` 必须保留状态差异，而不是只写一版标准对话。
- 每个 `MQ` 必须保留失败回流，而不是只允许成功。
- 每个 `MQ` 的 Journal 更新必须只写当下玩家知道的事。
- 白鹿之灵只允许承担 `CL01–CL02` 中有限的异常反馈。

### 9.3 返工信号

若 B06 出现以下情况，应判定为返工：

- `MQ01–MQ03` 没有把回返代价写实。
- `MQ04–MQ07` 把地方现实压成背景说明。
- `MQ08–MQ10` 把入口残带写成坏掉的出口。
- `MQ11–MQ12` 把主聚落写成普通任务板。
- 白鹿之灵解释了太多。
- 失败与变体不写回六项状态。
