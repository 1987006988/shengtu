# 42A Item / Document 文本包：法宝 / 任务物件 / 文献底盘

## 1. 文档定位

本文档对应拆分后子批次 `B15A：Item / Document 文本 A` 的唯一主输出。

它负责把《升途：封绝之地》已经在主线、支线、Codex 与 Journal 中锁定的关键物件、残片、凭据、抄件与缺口记录，压成可供任务实现、交互检查、物品栏说明、调查节点、区域反馈与尾声调用直接使用的文本资产。

本包承担的结构任务是：

- 把 `40a` 已确认的区域、法宝、白鹿之灵、受限未来条目，压成可拾取、可查看、可调用的物件层入口。
- 把 `41a` 已确认的 `CL01–CL14`、`SQ01–SQ12`、六大区域更新与尾声余压，落实成能被任务节点实际引用的凭据与文书。
- 把 `35b`、`37a`、`38a` 与 `17` 已锁定的承担者、资格裂口、后路收缩、灰线反证、法宝余压与尾声不整齐，写成能直接回扣玩法与叙事实现的物件文本。
- 为后续 `43a_system_feedback_texts_pack.md` 提供稳定调用面，但不反向替 `43a` 预写 UI 提示、系统播报或最终验收清单。

本包不做：

- 不回头重做 `40a` 的 Codex 条目或 `41a` 的 Journal 文本。
- 不新增正式势力名、地名、人物名、小世界名或世界级规则。
- 不把物件层写成设定百科，也不把文书层写成尾声总结。
- 不让白鹿之灵通过物件说明越权解释灰线、路线或终局正确性。
- 不提前进入 `43a` 的系统反馈、即时提示、失败播报与 final readiness 缺口清单。

## 2. 使用规则

### 2.1 条目统一字段

本包每个条目统一包含：

- `首次获得`
- `关联窗口`
- `来源接口`
- `物件 / 文献正文`
- `使用 / 调用边界`
- `尾声留存`

解释：

- `首次获得` 只写首次应被玩家看见或正式入账的窗口。
- `关联窗口` 只允许使用已落地的 `W1 / W2 / W3 / W4 / EP` 与 `CL / SQ` 链接。
- `来源接口` 必须回扣到已存在的主线、支线、区域、Codex 或 Journal 链，不得空降。
- `物件 / 文献正文` 必须说明它为什么存在、被谁使用、会把哪一层现实关系带进场。
- `使用 / 调用边界` 必须写清物件能做什么、不能偷渡什么。
- `尾声留存` 必须保留受限未来与未清零代价，不得把尾声写成“该物件已圆满退场”。

### 2.2 窗口缩写

- `W1`：`MQ01–MQ12`
- `W2`：`MQ13–MQ24`
- `W3`：`MQ25–MQ38`
- `W4`：`MQ39–MQ48`
- `EP`：尾声后置留存

### 2.3 物件层边界

本包中的物件与文献只允许承担以下功能：

- 作为主支线任务目标、调查凭据、交互道具、区域承压证据与尾声回声载体。
- 证明某项结构判断已经在现实层留痕，而不是只停留在角色口头判断里。
- 为 `40a / 41a / 43a` 提供一致调用面。

不允许承担以下功能：

- 用单件物品解释完整大战真相或完整灰线历史。
- 用单件物品宣布某条路线天然正确。
- 用文献替代主线 / 支线正文推进。
- 用白鹿回响类物件把法宝代价写轻。

### 2.4 白鹿之灵边界

本包中凡涉及白鹿之灵、法宝承压与回返相关条目，只允许体现：

- 接引法宝之灵的低强度回响接口
- 阈值示警、残忆碎片、承压留痕、边界提醒
- 与回返、承认链、共监与余压事故相关的有限痕迹

不允许体现：

- 世界意志
- 路线裁判
- 万能解释器
- 无代价复生许可
- 对灰线、地方或同伴作价值判决

## 3. 法宝与承压物件

### ITM-REL-01：回返余痕片

- `首次获得`：`W1`
- `关联窗口`：`W1–W4`
- `来源接口`：`CL01`、`CL02`、`JR-CL01`、`JR-CL02`

`物件 / 文献正文`

> 这不是能被拿来炫耀的残片，而是一块每次回返后都会更难忽视的留痕。它看起来像细薄剥落物，实际记录的却不是“回来过”，而是“回来时把哪一层账继续写深了”。越靠近主角团，余痕越密；越把回返当补救，它越像一页不会自己翻过去的旧账。

`使用 / 调用边界`

- 可用于回返机制说明、调查对白、营地复盘与阈值抬升提示。
- 不得被写成无限复生凭证。
- 不得单独解释白鹿原为何仍能存在。

`尾声留存`

> 尾声里这类余痕不会消失。它只会更安静，也更像一种被正式承认的提醒：主角团被接回来的每一次，都仍在未来里继续生效。

### ITM-REL-02：承压刻片

- `首次获得`：`W2`
- `关联窗口`：`W2–EP`
- `来源接口`：`CL07`、`CL09`、`SQ07`、`CDX-REL-04`

`物件 / 文献正文`

> 承压刻片通常并不完整，边缘常伴有被高压啃噬过的细裂。它的价值不在于能补足多少力量，而在于它总带着一段“是谁先替这份结构承压”的物证语气。地方、外缘回收队与中心执行面会以不同方式看待它：前者把它看成还能不能继续撑命的补丁，后者把它看成继续运行的高风险凭据。

`使用 / 调用边界`

- 可用于法宝承压、外缘回收、事故善后与共监面节点。
- 不得被写成万能修复材料。
- 不得在获得时直接附带完整法宝真相。

`尾声留存`

> 到尾声时，承压刻片更像是一类仍要继续被盯住的东西。它证明白鹿原还没彻底断，也证明任何“还能撑”的说法都不是没代价的。

### ITM-REL-03：阈值示警签

- `首次获得`：`W3`
- `关联窗口`：`W3–EP`
- `来源接口`：`CL09`、`CL10`、`MQ39–MQ40`、`SQ11`

`物件 / 文献正文`

> 这类示警签很轻，轻到初看像没有分量。可一旦法宝余压升高、绑定关系被拉紧，最先变色的往往就是它。它不回答“还能不能继续”，只回答“继续之后先裂的是哪里”。因此它在主角团手里像警示，在地方眼里却更像一张永远在追问第一责任归属的薄片。

`使用 / 调用边界`

- 可用于阈值检查、余压前兆、节点封存与法宝相关区交互。
- 不得被写成系统意义上的绝对安全指示器。
- 不得替玩家判定哪条路线更正确。

`尾声留存`

> 尾声里的示警签不会归零。颜色可能变浅，意义却更重，因为所有人都已经知道“还在用，就还在负”。

### ITM-REL-04：承认链残印

- `首次获得`：`W3`
- `关联窗口`：`W3–W4`
- `来源接口`：`CL09`、`MQ43–MQ46`、`CDX-REL-01`

`物件 / 文献正文`

> 承认链残印不是许可牌，也不是神秘认证。它更像某种被磨损后仍未完全消失的判断痕迹，提醒玩家与白鹿原里的每一方：谁被接住、谁被记入、谁被排在边外，原本就有一套残余逻辑在悄悄运作。残印越清楚，越说明旧资格线还在；残印越破，越说明所有方案都在拿新的代价与旧的残值硬碰。

`使用 / 调用边界`

- 可用于承认链调查、旧链残值判断与终局条件对照。
- 不得直接当作路线背书道具。
- 不得让玩家仅凭一枚残印跳过承担者比较。

`尾声留存`

> 尾声中承认链残印最重要的意义，不是证明旧链条还活着，而是证明旧链条的残值和旧链条的排除后果都没有被自动结清。

### ITM-REL-05：共监留痕环

- `首次获得`：`W4`
- `关联窗口`：`W4–EP`
- `来源接口`：`MQ39`、`SQ11`、`CDX-ACT-05`

`物件 / 文献正文`

> 共监留痕环本身不提供安全，只提供“谁曾在场、谁看见了什么、谁同意把这份风险写进共同记录”的最低限度证据。它的作用从来不是降低法宝余压，而是逼着主角团、地方与执行面不能再把“继续使用的决定”藏在门后面做完。

`使用 / 调用边界`

- 可用于共监面建立、事故后追责、区域交互与尾声回查。
- 不得写成自动化免责道具。
- 不得让它替任何一方洗白。

`尾声留存`

> 尾声里的留痕环如果还在场，就说明白鹿原至少有一部分人不再愿意用“不知道”来换暂时的安稳。

### ITM-REL-06：余压封存条

- `首次获得`：`W4`
- `关联窗口`：`W4–EP`
- `来源接口`：`MQ40`、`SQ11`、`JR-REG-06`

`物件 / 文献正文`

> 余压封存条是白鹿原最容易被误会成“已经处理完”的物件之一。它真正完成的只是暂时隔离、暂时标记、暂时把伤口别再继续往外崩。每一枚封存条都在提醒人：法宝事故最危险的不是刚炸开的瞬间，而是后来所有人都希望尽快把那一幕忘掉的时候。

`使用 / 调用边界`

- 可用于封点、事故后区域切换、延迟处理说明。
- 不得写成彻底解决余压的封印。
- 不得与白鹿之灵回响混同。

`尾声留存`

> 封存条留到尾声时，会显得格外刺眼，因为它证明有些东西不是“没问题了”，只是被诚实地承认为还不能乱动。

## 4. 主聚落 / 交换带 / 入口 / 节点物件

### ITM-OPS-01：主聚落救急牌

- `首次获得`：`W1`
- `关联窗口`：`W1–W2`
- `来源接口`：`SQ01`、`JR-SQ01`、`CDX-REG-02`

`物件 / 文献正文`

> 救急牌并不贵重，却总在主聚落最紧的时候变得像刀一样锋利。它代表的不是资源本身，而是谁被允许先从“还没轮到你”里暂时拉出来。牌子拿在谁手里，主聚落就会记成谁替别人排过一次次序；牌子最终给到谁手里，则会慢慢变成以后还愿不愿意再开口求援的理由。

`使用 / 调用边界`

- 可用于救急排序、分配争执与聚落口风变化。
- 不得写成简单治疗券或万能资源牌。
- 不得单独解决主聚落长期承压问题。

`尾声留存`

> 到尾声时，救急牌的意义不在于还剩几枚，而在于白鹿原是不是终于肯把“谁先被看见”这笔账明着记。

### ITM-OPS-02：资格改列签

- `首次获得`：`W1`
- `关联窗口`：`W1–W3`
- `来源接口`：`SQ02`、`JR-SQ02`、`CDX-STR-08`

`物件 / 文献正文`

> 改列签是一种极容易被误看成行政小物的东西。实际上，它每一次出现都意味着旧资格线、现实贡献与即时危险中至少有一条被临时压了下去。签面越轻，后果越重，因为它总会逼出那句真正难听的话：今天谁之所以排上来，是不是意味着另一个人又被往后挤了一格。

`使用 / 调用边界`

- 可用于资格争议、例外开口、临时放宽或临时收紧。
- 不得写成万能平等凭据。
- 不得在没有区域与人物反应的情况下独立生效。

`尾声留存`

> 尾声里若仍保留改列签，只能说明白鹿原还没找到不会继续制造新伤口的排序方式；但它至少不再假装所有排序都天然正确。

### ITM-OPS-03：风险票据

- `首次获得`：`W2`
- `关联窗口`：`W2–W4`
- `来源接口`：`SQ03`、`JR-SQ03`、`CDX-STR-05`

`物件 / 文献正文`

> 风险票据从来都不只是写货物、路程或损耗。它真正登记的是“这一趟若出事，先由谁垫账”。交换带越乱，这张票越值钱；可它之所以值钱，恰恰因为总有人被更熟练地写成可以先被递出去的风险。看懂风险票，就是看懂交换带究竟是怎么把活路和伤一起卖出去的。

`使用 / 调用边界`

- 可用于交换带议价、风险转嫁、货流权限与暗债牵引。
- 不得被简化成普通商业凭证。
- 不得单凭票据就判断道德优劣。

`尾声留存`

> 到尾声时，风险票据仍会存在，但应更难再被轻易说成中性工具。白鹿原至少要知道，价目表里一直埋着谁的伤。

### ITM-OPS-04：暗债折记绳

- `首次获得`：`W2`
- `关联窗口`：`W2–W4`
- `来源接口`：`SQ04`、`JR-SQ04`、`CDX-STR-06`

`物件 / 文献正文`

> 暗债折记绳看起来不起眼，常被混在货捆、担具或临时押条里。它真正的可怕之处，是把“今天先活下去”与“明天更难脱身”绑成同一件事。人一旦被这类绳记住，往后再想把命和账分开，往往已经太迟。它是交换带最常见、也最不肯被叫出名字的活法证据。

`使用 / 调用边界`

- 可用于暗债链调查、绑定关系、折记追踪与换链尝试。
- 不得写成一剪就断的爽点道具。
- 不得单独承担灰线全部吸附逻辑。

`尾声留存`

> 尾声里若仍有人保留这类折记绳，最重要的不是它有没有被剪断，而是白鹿原有没有终于开始学着不再靠它活。

### ITM-OPS-05：入口放行牌

- `首次获得`：`W1`
- `关联窗口`：`W1–EP`
- `来源接口`：`CL03`、`SQ05`、`JR-REG-03`

`物件 / 文献正文`

> 放行牌最刺人的地方，在于它从不回答“有没有路”，只回答“今天先轮到谁走”。它既是入口残带还能运行的证据，也是资格裂口被压在边线上的证物。每次牌面更新，都会让一部分人更安心，也让另一部分人更清楚地知道：后路并不是为所有人等宽地开着。

`使用 / 调用边界`

- 可用于入口放行、折返、边界盘查与路线版本切换。
- 不得写成通行无阻的万用令牌。
- 不得让它替代区域口风与失败回流。

`尾声留存`

> 尾声里保留下来的放行牌，应该更像一张窄路证明，而不是开放承诺。它证明路还在，也证明路仍很窄。

### ITM-OPS-06：折返回执片

- `首次获得`：`W2`
- `关联窗口`：`W2–EP`
- `来源接口`：`SQ05`、`SQ06`、`CDX-STR-09`

`物件 / 文献正文`

> 折返回执片通常会被攥得很紧，因为它写的不是说明，而是一句被迫接受的判断：今天先轮不到你。它既能保护更靠里的稳定，也会把同一句“以后再说”一次次盖在边缘者头上。白鹿原之所以难承认后路变窄，就是因为太多人的路，是先被写成了折返。

`使用 / 调用边界`

- 可用于折返现场、节点回撤、后路收窄与情绪余波。
- 不得写成单纯失败道具。
- 不得拿它当白鹿原完全封死的证据。

`尾声留存`

> 尾声中这类回执若还频繁出现，说明白鹿原争回来的未来确实存在，但仍不是每个人都能轻松挤进去的未来。

### ITM-OPS-07：节点护送绳标

- `首次获得`：`W2`
- `关联窗口`：`W2–W4`
- `来源接口`：`SQ06`、`JR-SQ06`、`CDX-ACT-03`

`物件 / 文献正文`

> 护送绳标没有荣耀感，它只负责证明一件事：还有人愿意替“后来者还有路”这句话继续站在风口上。绳标一旦断掉，不只意味着人没送到，还意味着某条本来就很窄的线，可能从此再也不敢被谁称作“还在”。

`使用 / 调用边界`

- 可用于节点护送、路线保留、回撤比较与后路判断。
- 不得写成普通护送任务标记。
- 不得把它与出口保证混同。

`尾声留存`

> 尾声里剩下的绳标应该少，但应更重。每一根都代表有人把“还留一条路”这件事真正写进了自己的承担里。

### ITM-OPS-08：回撤记号钉

- `首次获得`：`W2`
- `关联窗口`：`W2–EP`
- `来源接口`：`SQ06`、`JR-END-02`、`END-ASYNC-02`

`物件 / 文献正文`

> 回撤记号钉打在哪一处，往往就等于替哪一段以后宣判了先缩、先窄、先让。它不吵，也不显眼，却最能在地图之外让人明白：白鹿原没有同时保住所有路。很多以后并不是突然没的，而是从某一枚钉子开始，慢慢不再敢被谁提起。

`使用 / 调用边界`

- 可用于路线缩线、节点放弃、区域状态落版与尾声回看。
- 不得写成单一关闭地图的机械开关。
- 不得让它掩盖“谁决定回撤、谁承担后果”的问题。

`尾声留存`

> 尾声里若还能看见这些钉子，就该让玩家明白：白鹿原的受限未来，是带着被正式承认的退让一起成立的。

## 5. 外缘 / 灰线 / 回引物件

### ITM-GRY-01：外缘回收残片

### 8.3 风险票据现场块

> 票据被翻到最厚的一页时，往往也是说话声最小的时候。
>
> 因为谁都清楚，那些看起来只是价差、损耗和延误的字眼，最后总会落回同一句：今天先让谁替这条流动交一轮过路费。

可接对话：

`流动节点者`：
> 货没长腿。
>
> 真正在走的，是别人的以后。

### 8.4 入口放行牌现场块

适用于 `CL03`、`SQ05` 与尾声入口回看。

> 放行牌被递出去时，最先退后的往往不是没轮上的人，而是那些已经习惯了自己可能轮不上的人。
>
> 他们会先看别人的脸色，再看牌面，最后才决定要不要把那一点还想挤出去的念头继续留着。

可接对话：

`流动节点者`：
> 这牌不是路。
>
> 它只是今天暂时允许谁别先被路关在外面。

### 8.5 外缘回收残片现场块

适用于 `SQ07` 回收成功或失败后的第一轮复盘。

> 残片刚从包裹里拿出来时，空气通常会先安静一下。
>
> 因为所有人都明白，能从裂伤外缘带回来的东西，几乎不会只带着一种意义回来。
>
> 它可能是补命，也可能是下一轮更难说清的风险。

可接对话：

`剑器飞升者`：
> 先别问值不值。
>
> 先问它准备先咬谁。

### 8.6 回引隔离带现场块

适用于 `SQ08` 回引成功后的人群反应。

> 带子被系好以后，真正难看的不是它把谁隔开。
>
> 真正难看的是每个人下意识后退那半步时，谁都知道自己并不完全错。
>
> 白鹿原就是靠这种不肯轻易说出口、又没法假装不存在的迟疑继续活。

可接对话：

`愿力飞升者`：
> 这不是欢迎。
>
> 但至少也还不是宣判。

### 8.7 反证封套现场块

适用于 `SQ09` 的公开、分层释放与营地复盘。

> 封套一旦打开，屋里最先出现的不是争执，而是一种更危险的停顿。
>
> 那停顿在问每个人：如果这些页上的东西都是真的，那你之前靠着继续往下做事的那套解释，还剩多少能站住。

可接对话：

`灰线历史位`：
> 最重的从来不是证据。
>
> 是证据逼你承认自己以前到底把谁压在了账外。

### 8.8 余压事故共监抄记现场块

适用于 `SQ11`、`MQ39`、`MQ40` 与法宝相关区共监节点。

> 抄记被逐一签名的时候，现场不会显得更安全。
>
> 它只会显得更难逃。
>
> 因为从这一刻起，“谁知道了、谁还同意继续用、谁愿意把自己的名字留在这份风险旁边”都不再能躲在门里做完。

可接对话：

`中心执行者`：
> 这不是共识。
>
> 这是认账。

### 8.9 受限未来缺口记录现场块

适用于 `SQ12`、`MQ47`、`MQ48` 与尾声回看。

> 缺口被一条条写下时，最让人松口气的往往不是“终于有办法了”。
>
> 恰恰相反，真正让人松口气的是终于不用再替这些洞假装它们已经被补平。
>
> 白鹿原能往后走，很多时候靠的不是圆满，而是先把该背的东西分开写清。

可接对话：

`地方承接者`：
> 能写下来，不等于能立刻补上。
>
> 但总比继续拿假话顶着强。

## 9. 文献归档与尾声留存模板

本节只定义文书类条目在后段与尾声中的归档语气，供 `43a` 后续调用，不替代 `43a` 的系统提示层。

### 9.1 主聚落归档语气

适用于 `DOC-01`、`DOC-02`：

> 主聚落留下来的纸页通常会更皱，也更短。
>
> 不是因为问题变少了，而是因为留下来的人越来越知道，哪些账必须被一眼看见，哪些话再绕半圈就会又把人推回后面去。

### 9.2 交换带归档语气

适用于 `DOC-03`、`DOC-04`：

> 交换带的归档从不干净。
>
> 它最诚实的时候，也只是终于开始把价、债、转押和谁先垫账放在同一页上，而不是继续让它们分散在人人看得懂、却谁都不肯认全的角落里。

### 9.3 边界归档语气

适用于 `DOC-05`、`DOC-06`、`ITM-OPS-05`、`ITM-OPS-08`：

> 边界类文书最沉的地方，不是写了多少规则，而是每多一笔放行、多一枚回撤记号，白鹿原就多一分“谁以后还有路、谁以后得先学会不把自己算进去”的现实重量。

### 9.4 外缘与灰线归档语气

适用于 `DOC-07`、`DOC-08`、`DOC-09`、`DOC-10`、`ITM-GRY-01` 至 `ITM-GRY-04`：

> 这类文书不会给人整齐感。
>
> 它们更像一摞彼此顶撞的证词：有人说这是活路，有人说这是污染，有人说这是旧账，有人说这是将来不得不继续背着走的裂口。
>
> 白鹿原真正诚实的归档，只会把这些彼此顶撞都一起留下。

### 9.5 法宝归档语气

适用于 `ITM-REL-01` 至 `ITM-REL-06`、`DOC-11`：

> 法宝相关归档不应给人“终于弄明白了”的安稳感。
>
> 它最好的版本，也只是让人更清楚：白鹿原至今还没彻底断掉，不是因为找到了答案，而是因为仍有人在替继续使用的代价认账。

### 9.6 尾声缺口归档语气

适用于 `DOC-12` 及所有带 `尾声留存` 的条目：

> 尾声档案最该避免的，是把“被接住的一部分”和“还得继续背的一部分”重新混成一句好听的话。
>
> 受限未来之所以可信，恰恰因为它把两者分开了。

## 10. 失败回流与版本保留矩阵

本节补足物件在失败、拖延、版本分化与尾声不整齐时的实现口径，确保 `42a` 不是只在“成功拾取 / 成功交付”场景下成立。

### 10.1 主聚落物件回流

适用于 `ITM-OPS-01`、`ITM-OPS-02`、`DOC-01`、`DOC-02`。

若排序失当、例外过轻或拖延过久，可调用以下物件语气：

> 牌和签还在。
>
> 可它们已经不再像帮扶凭据，而更像提醒：主聚落又一次知道了，谁会替别人排次序，谁又会先被排到后面。

若主聚落接受度较高，可调用：

> 这些纸签没有变轻。
>
> 它们只是第一次不再全靠沉默决定谁先被看见。

### 10.2 交换带物件回流

适用于 `ITM-OPS-03`、`ITM-OPS-04`、`DOC-03`、`DOC-04`。

若稳盘优先而翻账不足，可调用：

> 票还在流，账也还在流。
>
> 真正被改熟练的，只是白鹿原继续把同一层人写成可先垫账对象的手法。

若拆链过猛、换链失稳，可调用：

> 绳被剪开了，路却没有立刻变轻。
>
> 许多人第一次发现，原来自己早就把“被绑着活”当成了还能继续活的默认版本。

### 10.3 边界物件回流

适用于 `ITM-OPS-05`、`ITM-OPS-06`、`ITM-OPS-07`、`ITM-OPS-08`、`DOC-05`、`DOC-06`。

若放行偏紧、回撤偏早，可调用：

> 牌、钉、会记和回执都还在。
>
> 它们共同说明的却只有一件事：白鹿原的路没有一下子断掉，而是先被一枚枚更不敢回看的判断慢慢磨窄了。

若后路保留较高，可调用：

> 留下来的绳标和抄件不会让人安心。
>
> 但至少它们证明，还有几条路不是靠假装宽阔才继续存在。

### 10.4 外缘与回引物件回流

适用于 `ITM-GRY-01`、`ITM-GRY-02`、`DOC-07`、`DOC-08`。

若回收失衡、回引受阻，可调用：

> 带回来的和没带回来的，都开始在更靠里的地方继续生效。
>
> 白鹿原最怕的从来不是外缘东西回来，而是回来以后，没有人愿意先替它们站住位置。

若版本较平衡，可调用：

> 有些东西终于不是只能被一起推回去了。
>
> 可这不等于它们已经被安稳接住，只意味着白鹿原开始承认自己必须和这些东西一起继续活。

### 10.5 反证与借道物件回流

适用于 `ITM-GRY-03`、`ITM-GRY-04`、`DOC-09`、`DOC-10`。

若反证被压住、借道被粗暴封堵，可调用：

> 套页没被公开，封签却更常被加盖。
>
> 这种版本看起来更稳，实际只是把旧账和新路一起压进更难追认的暗处。

若反证公开较足、借道转为受控窄路，可调用：

> 白鹿原没有因此变干净。
>
> 它只是更难继续说自己什么都没看见，也更难继续让所有脏路都不必认账。

### 10.6 法宝与尾声物件回流

适用于 `ITM-REL-01` 至 `ITM-REL-06`、`DOC-11`、`DOC-12`。

若事故处理偏封存、尾声又试图写轻，可调用：

> 封存条、抄记和缺口记录都还在。
>
> 它们像是在同时反驳同一句话：事情并没有因为大家不想再看，就真的过去。

若承担较足、缺口保留较诚实，可调用：

> 这些物件没有变成奖章。
>
> 它们更像尾声留下来的硬边，提醒所有人：白鹿原被接住了一部分，也有一部分仍要继续背着走。

## 11. 多版本保留语气模板

本节为 `42a` 中高频物件提供尾声后的多版本保留语气，供 `43a` 以后选择调用。

### 11.1 “被接住一部分”版本

适用于 `ITM-REL-01`、`ITM-GRY-02`、`DOC-12`：

> 这件东西之所以还能被留下，不是因为问题已经结束。
>
> 是因为白鹿原至少接住了最先会掉下去的一部分，却还没有本事把剩下的那部分也一起接稳。

### 11.2 “后路仍窄”版本

适用于 `ITM-OPS-05`、`ITM-OPS-06`、`ITM-OPS-07`、`DOC-05`、`DOC-06`：

> 它证明的从来不是“路已经好了”。
>
> 它证明的只是：路还没彻底断给后来的人看，但也远没宽到可以让任何人放心。

### 11.3 “旧账未平”版本

适用于 `DOC-01`、`DOC-02`、`DOC-09`、`DOC-10`：

> 纸页还留着，不是为了反复翻旧账取乐。
>
> 恰恰相反，它们被留下来，是因为白鹿原再也没有资格把同一种旧账继续压成“以后自然会过去”。

### 11.4 “法宝仍在负”版本

适用于 `ITM-REL-02`、`ITM-REL-03`、`ITM-REL-05`、`ITM-REL-06`、`DOC-11`：

> 这些留痕不会因为尾声到来就变成纪念。
>
> 它们最重要的意义，是继续提醒所有人：还在用，就还在负；还在负，就还得有人继续盯着最先会裂开的地方。

### 11.5 “灰线未清零”版本

适用于 `ITM-GRY-03`、`ITM-GRY-04`、`DOC-09`、`DOC-10`：

> 它们被保留下来，不是为了证明阴影还占上风。
>
> 而是为了防止白鹿原再次靠“看不见”来假装自己已经更稳了。

## 12. 查看时短段落模板

本节补足玩家在物品栏、调查页或营地复盘中二次查看这些物件时可调用的短段落，确保条目在重复查看时仍有叙事重量，而不是只剩标题与字段。

适用于 `ITM-OPS-01`、`ITM-OPS-02`、`DOC-01`、`DOC-02`：

### 12.3 后路类短段落

适用于 `ITM-OPS-05`、`ITM-OPS-06`、`ITM-OPS-07`、`DOC-05`、`DOC-06`：

> 真正把人压住的，不是“有没有路”。

### 12.4 灰线类短段落

### 12.5 尾声类短段落

### 12.6 交换与外缘类短段落

>
> 它们共同指向的，其实一直都是同一个白鹿原难题：眼下还能活的那一层，究竟是拿什么垫出来的；而那些被垫出去的人，又准备多久以后才会重新回来要账。

### 12.7 归档后回望类短段落

适用于全部 `DOC-*` 条目：

> 真正难看的档案通常不会被写得很满。

>
