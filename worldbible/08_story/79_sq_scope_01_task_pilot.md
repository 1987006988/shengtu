# SQ-SCOPE-01 任务试点

## 1. 文件定位

本文件只服务 `SQ-SCOPE-01《借来的承接位》`。  
它只允许停在 `CH02-S02–S03`，只服务 `A1` 窗口。

## 2. 任务定义

| 字段 | 内容 |
| --- | --- |
| `sq_id` | `SQ-A1-01《借来的承接位》` |
| `chapter_window` | `CH02-S02–CH02-S03` |
| `attach_chain` | `照护棚 -> 账本室` |
| `lead_roles` | `沈行策 / 柳照禾` |
| `entry_condition` | 完成 `MQ-A1-02A` 的照护棚补手起段，且主角已被允许进入账本分账链 |
| `core_function` | 放大“主角被临时收下，意味着别人今天要往后挪一格” |
| `objective_chain` | 协助照护棚稳住两名被延后者 -> 跟随柳照禾去账本室看见一条被借出的承接位 -> 主动承担一段劳作，换来把主角暂时记在可观察栏而非立即移交栏 |
| `completion_condition` | 主角理解“自己今天被留下，是借来的位置”，并接受“留下 = 别人更难”的现实后果 |
| `allowed_reveals` | 承接位紧张、基础补给额度紧张、延后名单真实存在、柳照禾的人命账视角 |
| `forbidden_reveals` | 守衡司全貌、白鹿解释权、古战前史、后段未来比较、任何新增地方权威 |

## 3. 与主线接口

1. 它只能附着在 `MQ-A1-02A《鹿门的价钱》` 内部。
2. 它只能放大 `CH02-S02–S03` 已落库承压，不得改写 `CH02-S04` 的移交接口。
3. 它不能单独生成新的 `Codex` 或新的主线状态终点。

## 4. 当前明确禁止

1. 不得把本试点扩到 `CH04-S01–S02`。
2. 不得拿 `SQ-SCOPE-02《断路前的空名册》` 一并解冻。
3. 不得让本试点反向改写 `A1` 主线任务链。

## 5. 一句话口径

> 这条支线试点不是开支线层，而只是证明“照护棚 -> 账本室”这一条仍停在 `A1` 窗口里的附着型任务也能受控落地。
