# A2 主线 MQ

## 1. 文件定位

本文件只服务 `V1 / A2 / CH04–CH06` 的现役主线任务链。  
它只把已落库正文翻译成 `MQ` 结构，不新增并列现役主线。

| `mq_id` | 名称 | 章节窗口 | 任务功能 | 完成条件 |
| --- | --- | --- | --- | --- |
| `MQ-A2-04A` | 《谁给这条路起名》 | `CH04-S01–S02` | 让玩家理解“借道资格 = 活路排序” | 前哨谈判完成，旧接引节点被临时补稳。 |

## 3. 单任务展开

### `MQ-A2-04A《谁给这条路起名》`

- `entry_state`：`A1-CH03-DEPART-LOCK`
- `lead_roles`：`沈行策 / 顾砺衡 / 祁渡川`
- `objective_chain`：
  - 在断引栈道前哨完成借道谈判
  - 理解“命名权 / 维护权 / 通行资格”属于同一张路价账
  - 在旧接引节点残桥完成一次临时补缝
- `allowed_reveals`：道路政治、窗口稀缺、主角会被白鹿原调用
- `forbidden_reveals`：灰缄线全貌、外部世界全图、主角终局身份

### `MQ-A2-04B《空掉的路名册》`

- `entry_state`：`A2-CH04-NODE-STABILIZED`
- `lead_roles`：`沈行策 / 祁渡川 / 桑沉岫`
- `objective_chain`：
  - 在残桥后段遭遇精准伏击
  - 抢回关键修补件并确认“有人卖路”
  - 在夜议中把问题从“谁袭击”推进到“谁替这条路继续签字”
- `allowed_reveals`：卖路、空名册、窗口被交易
- `forbidden_reveals`：宁折羽、碑林残忆、灰缄网络全貌

### `MQ-A2-05A《谁替这条缓冲带分账》`

- `entry_state`：`A2-CH04-SELLROAD-LOCK`
- `lead_roles`：`顾砺衡 / 柳照禾 / 岑持律`
- `objective_chain`：
  - 参与征调审议
  - 看见“整体可保”与“局部可弃”的算法冲突
  - 在急报抵达后转入风窖台现场
- `allowed_reveals`：守衡司结构逻辑、地方人命账、公开裂缝
- `forbidden_reveals`：顾砺衡最终转向、未来比较方案

### `MQ-A2-05B《风窖台前夜》`

- `entry_state`：`A2-CH05-LEVY-OPEN`
- `lead_roles`：`沈行策 / 柳照禾 / 顾砺衡 / 桑沉岫`
- `objective_chain`：
  - 进入风窖台外围
  - 对照结构优先与抢人优先的两条线
  - 用主角的临时接入争出最后窗口
- `allowed_reveals`：结构线、人命线、无第三路线
- `forbidden_reveals`：白鹿高位真身、完整回返代价

### `MQ-A2-06A《先救谁，先坏哪一边》`

- `entry_state`：`A2-CH05-WINDCELL-LOCK`
- `lead_roles`：`沈行策 / 柳照禾 / 顾砺衡`
- `objective_chain`：
  - 看完结构优先线
  - 看完抢人优先线
  - 在白鹿比较前完成第一次地区级签字
- `allowed_reveals`：白鹿只摊代价、地区级排序、共同担责前针
- `forbidden_reveals`：终局三方案、历史翻面、灰缄线全貌

### `MQ-A2-06B《废墟里的人不会一起谢你》`

- `entry_state`：`A2-CH06-CHOICE-SIGNED`
- `lead_roles`：`沈行策 / 柳照禾 / 桑沉岫 / 顾砺衡`
- `objective_chain`：
  - 接住灾后骂名与幸存名单
  - 锁定部分缓冲带长期受损的后果
  - 找到灰缄封记并把灾难从纯天灾改写为内部裂口问题
- `allowed_reveals`：骂名兑现、内部裂口证据、队伍共同挨骂
- `forbidden_reveals`：宁折羽直接出场、碑林、古战全貌

## 4. 一句话口径

> `91` 只把 `CH04–CH06` 的道路政治、征调分账与风窖台代价排序压成 `MQ`，不提前越入 `CH07+`。
