# 27 第一款游戏主线脊柱

## 1. 文档定位

本文档对应 `B02：重建游戏级主线架构 B` 的主输出之一，用于把 `25_volume1_core_conflict.md` 与 `26_volume1_revelation_ladder.md` 落成可供后续任务设计、场景卡设计与主线文本扩写调用的主线脊柱。

本文档不是小说章节复述，不是完整任务正文，也不是场景卡。它只负责建立：

- 第一款游戏从 Act 1 到 Act 4 再到 Epilogue 的任务节点骨架。
- 每个任务节点对应的揭示层级、状态输入、状态输出与压力承载面。
- 每个任务节点如何把现实托底、残秩序、灰线、法宝承压、同伴立场与白鹿之灵接口绑在一起。
- `28_volume1_chapter_outline.md` 与 B03/B04 场景卡批次可以直接继承的调用接口。

本文档服务 `白鹿原` 第一款游戏尺度。`白鹿原` 是 `古裂残天` 内部隔绝地域，不等于整个 `古裂残天`。本文档所有节点都只能收束到白鹿原未来与代价承担问题，不得上浮成主世界总命运线。

## 2. 继承口径与执行边界

### 2.1 继承自 B01 的固定口径

- 第一款游戏主问题是：白鹿原要如何继续存在，以及由谁承担它继续存在、重连、封闭、重启或重组的代价。
- 四幕推进顺序固定为：坠落与封绝、残存秩序与局部生存、大战真相与法宝绑定、地域命运的决断。
- 揭示顺序固定为：先现实，后结构，再历史，最终回到代价。
- 必须持续调用的状态维度为：`地方信任`、`残秩序合法性`、`灰线接触`、`入口带稳定`、`法宝承压`、`同伴立场压力`。
- 白鹿之灵只能作为接引法宝之灵的白鹿显化接口出现，不得承担完整讲解、万能支援或无代价复生。

### 2.2 本文档必须满足的任务化要求

- 主线节点必须具备可执行目标，不得只写成故事摘要。
- 主线节点必须有状态输入与输出，不得只有单向播放式揭示。
- 主线节点必须写出失败或代价反馈，不得把失败收缩成“重来即可”。
- 主线节点必须说明对白鹿之灵的触发条件与限制，不得把白鹿之灵写成自动推进器。
- 主线节点必须能映射到后续场景卡，不得停留在纯概念层。

### 2.3 本文档不处理的事项

- 不新增正式势力名、地名、人物名、任务名。
- 不扩写完整对白、Journal、Codex、Item 或 System 文本。
- 不创建 `29` 场景卡。
- 不改写宇宙通则、九主世界总层或 `古裂残天` 长期正典。

## 3. 主线总结构

### 3.1 节点总量与分布

本作第一轮主线脊柱暂定为 `48` 个主线任务节点候选。

- Act 1：`MQ01–MQ10`
- Act 2：`MQ11–MQ24`
- Act 3：`MQ25–MQ38`
- Act 4：`MQ39–MQ46`
- Epilogue：`MQ47–MQ48`

这 `48` 个节点不是最终任务列表，而是可供 B03/B04 继续拆成场景卡、再供 B05 以后压成任务 ID 体系的稳定骨架。

### 3.2 节点字段解释

每个节点统一记录以下接口：

- `所属章节束`：对应 `15_chapter_cluster_skeleton.md` 的中层归并。
- `对应揭示层级`：对应 `R0–R8` 的主揭示位置。
- `进入条件`：节点开始时玩家最低应掌握的事实或状态。
- `核心问题`：该节点必须解决的现实问题。
- `节点目标`：玩家在该节点内的可执行目标。
- `主要承压对象`：哪一层现实或结构首先承担后果。
- `状态输出`：至少推动一项状态维度。
- `白鹿之灵接口`：出现时的触发条件、提供内容与边界。
- `失败 / 代价反馈`：不可省略的后果。
- `后续去向`：指向下一个节点群，而非单线小说顺承。

## 4. Act 1：坠落与封绝

### 4.1 Act 目标

- 让玩家确认这里是高位隔绝地域，而不是普通废墟。
- 让玩家理解白鹿原之所以未崩，是法宝、残秩序与地方承接共同托底。
- 让玩家确认主角团已经与法宝存续机制发生绑定。
- 让玩家接受第一款游戏尺度被锁定在白鹿原内部。

### 4.2 节点列表

#### MQ01：坠落失衡

- 所属章节束：章节束 01 `坠落适配束`
- 对应揭示层级：`R0`
- 进入条件：主角团飞升完成但未正常接入主世界秩序带。
- 核心问题：为什么落点不是正常接引地点，而是一片高位失衡区域。
- 节点目标：在高压环境中先站稳、确认失衡不是幻觉。
- 主要承压对象：主角团自身适配能力与最初复生风险。
- 状态输出：记录 `法宝承压` 初值；开启 `同伴立场压力` 的低阶观察。
- 白鹿之灵接口：仅在坠落冲击触发时给出“识别异常 / 接引偏移”级反馈，不解释成因。
- 失败 / 代价反馈：强行闯压会留下痛感残留与认知裂痕，后续对白要反映失衡后遗。
- 后续去向：`MQ02`、`MQ03`

#### MQ02：残秩序试压

- 所属章节束：章节束 01 `坠落适配束`
- 对应揭示层级：`R0`
- 进入条件：玩家已确认这里仍保有高位环境压力。
- 核心问题：为何这里没塌，却处处不稳定。
- 节点目标：识别残秩序对移动、救援与探索的限制形式。
- 主要承压对象：`残秩序合法性` 的初步判断与中心执行接口。
- 状态输出：玩家对残秩序的判断从“单纯危险”改为“仍在运转但失衡”。
- 白鹿之灵接口：无主动显化，只允许通过法宝接口发出结构逼近失衡的示警。
- 失败 / 代价反馈：误判残秩序为纯背景，会导致后续救援窗口变窄。
- 后续去向：`MQ03`、`MQ04`

#### MQ03：初始绑定

- 所属章节束：章节束 01 `坠落适配束`
- 对应揭示层级：`R0 -> R2`
- 进入条件：主角团遭遇首次致命或准致命异常。
- 核心问题：主角团为何会出现异于正常飞升者的绑定与回返迹象。
- 节点目标：让玩家体验绑定后果，而不是获知绑定机制。
- 主要承压对象：`法宝承压` 与主角团自身身心成本。
- 状态输出：正式开启“复生是代价型能力”的底线认知。
- 白鹿之灵接口：只允许短促显化，反馈“识别 / 拒绝脱离 / 代价回返”三种信息。
- 失败 / 代价反馈：复生后出现延迟、疼痛、错认或记忆缺口，不允许无痕恢复。
- 后续去向：`MQ04`、`MQ05`

#### MQ04：聚落拒识

- 所属章节束：章节束 02 `生存托底束`
- 对应揭示层级：`R1`
- 进入条件：玩家已摆脱坠落即死风险，开始接触人类活动痕迹。
- 核心问题：这里为何有人活着，但不愿轻易接纳外来飞升者。
- 节点目标：进入主聚落承接面之前，先面对地方生存秩序的防御性。
- 主要承压对象：地方共同体与 `地方信任` 初始值。
- 状态输出：`地方信任` 进入可增可减状态；玩家获得第一个现实锚点。
- 白鹿之灵接口：不主动介入地方判断，避免替玩家赢得信任。
- 失败 / 代价反馈：被拒识会改用旁证、事故或第三方通路推进，导致信息更碎。
- 后续去向：`MQ05`、`MQ06`

#### MQ05：承压托底

- 所属章节束：章节束 02 `生存托底束`
- 对应揭示层级：`R1`
- 进入条件：玩家已确认白鹿原并非无人废地。
- 核心问题：这些人能继续活着的成本由谁承担。
- 节点目标：让玩家直面本地共同体、小势力与承接层的现实负担。
- 主要承压对象：地方承接层、凡俗群体、低阶风险处理者。
- 状态输出：`地方信任` 与 `同伴立场压力` 同步启动，不同席位开始评价“该先救谁、先稳哪层”。
- 白鹿之灵接口：禁止出现，避免削弱现实托底的重量。
- 失败 / 代价反馈：若优先追异象而忽视承压者，后续地方证词会减少，入口资源也会收紧。
- 后续去向：`MQ06`、`MQ07`

#### MQ06：残秩序缓冲圈

- 所属章节束：章节束 02 `生存托底束`
- 对应揭示层级：`R2`
- 进入条件：玩家已知聚落依附某种更高位的结构稳定。
- 核心问题：是什么在把高位压力削成白鹿原还能承受的秩序外壳。
- 节点目标：接触残秩序缓冲区，理解残秩序不是纯背景效果。
- 主要承压对象：`残秩序合法性` 与中心执行接口。
- 状态输出：玩家开始承认残秩序“有用但并不无害”。
- 白鹿之灵接口：仅在法宝与缓冲区共振时提示“结构仍在接续”，不讲旧制全貌。
- 失败 / 代价反馈：误触缓冲区会抬升 `法宝承压` 或造成聚落外围失稳。
- 后续去向：`MQ07`、`MQ08`

#### MQ07：法宝托底证实

- 所属章节束：章节束 02 `生存托底束`
- 对应揭示层级：`R2`
- 进入条件：玩家已感知残秩序与绑定异常之间存在联系。
- 核心问题：接引法宝到底只是异常源，还是白鹿原未崩的结构支点之一。
- 节点目标：收集法宝托底的多源证据，包括环境、反应与现实后果。
- 主要承压对象：法宝核心影响区与地方现实平衡。
- 状态输出：`法宝承压` 从个人风险维度扩展为地域结构维度。
- 白鹿之灵接口：允许出现一次片段化回应，只确认“这里仍被托住”，不确认“还能托多久”。
- 失败 / 代价反馈：若过度调用法宝证据，会提前抬升复生负担并触发同伴分歧。
- 后续去向：`MQ08`、`MQ09`

#### MQ08：入口封锁

- 所属章节束：章节束 03 `封绝定性束`
- 对应揭示层级：`R3` 的前置边界
- 进入条件：玩家已知白鹿原与法宝、残秩序存在结构绑定。
- 核心问题：为什么外界强力秩序没有直接把这里接回或清空。
- 节点目标：验证入口残带的失真、封锁与高风险流通现实。
- 主要承压对象：`入口带稳定` 与流动节点接口。
- 状态输出：`入口带稳定` 首次进入显性记录；主线规模被锁在白鹿原内部。
- 白鹿之灵接口：不解释外界布局，仅对白鹿原与接引接口断续状态给出反馈。
- 失败 / 代价反馈：强行冲口会让后续通路更难、消息更少、灰线更早渗入。
- 后续去向：`MQ09`、`MQ10`

#### MQ09：封绝定性

- 所属章节束：章节束 03 `封绝定性束`
- 对应揭示层级：`R0–R2` 收束
- 进入条件：玩家已同时掌握现实承压、残秩序与入口封锁证据。
- 核心问题：白鹿原到底是什么性质的地域。
- 节点目标：把“高位失衡区”收束为“大战后被推离常规秩序带的隔绝地域”。
- 主要承压对象：守序飞升者、中心执行接口与地方承接者三方解释框架。
- 状态输出：统一第一幕认知终点，为第二幕转向“谁在托底、谁在付代价”铺路。
- 白鹿之灵接口：可作为旁证触发，但不能给出完整历史说明。
- 失败 / 代价反馈：若定性过浅，第二幕前段会以更多现实摩擦补课。
- 后续去向：`MQ10`、`MQ11`

#### MQ10：外界隔断确认

- 所属章节束：章节束 03 `封绝定性束`
- 对应揭示层级：`R3` 的尺度锁定
- 进入条件：第一幕核心认知已建立。
- 核心问题：为什么第一款游戏不能依靠外界力量直接收束。
- 节点目标：把玩家目标从“发出求援”改成“理解并介入本地存续结构”。
- 主要承压对象：主角团自身期待与同伴判断。
- 状态输出：第一幕正式结束；开放第二幕的地方承接、流动网络与风险处置线。
- 白鹿之灵接口：无。
- 失败 / 代价反馈：若仍执着把外界当万能解法，会在后续节点遭遇更高代价的现实反证。
- 后续去向：`MQ11`、`MQ13`

## 5. Act 2：残存秩序与局部生存

### 5.1 Act 目标

- 让玩家看到白鹿原能活下去依赖哪些现实结构。
- 让玩家理解地方承接层、地方势力、流动节点与入口带如何共同托底。
- 让飞升者差异、异质接口与灰线风闻开始介入。
- 把主线自然推向“深因追问”，而不是中段跑图。

### 5.2 节点列表

#### MQ11：地方运转

- 所属章节束：章节束 04 `地方承接束`
- 对应揭示层级：`R1 -> R3`
- 进入条件：第一幕已完成尺度锁定。
- 核心问题：地方共同体如何在不稳定环境里维持日常秩序。
- 节点目标：进入主聚落承接面，看见日常运转不是背景板。
- 主要承压对象：地方承接层与凡俗承重面。
- 状态输出：`地方信任` 与 `同伴立场压力` 同步拉开差异。
- 白鹿之灵接口：无。
- 失败 / 代价反馈：若只把聚落当补给站，后续地方合作权会受限。
- 后续去向：`MQ12`、`MQ13`

#### MQ12：凡俗承重

- 所属章节束：章节束 04 `地方承接束`
- 对应揭示层级：`R1`
- 进入条件：玩家已接入主聚落承接面。
- 核心问题：凡人与低阶超凡如何被迫共同承担高位地域的存续成本。
- 节点目标：让“现实托底”从口头说明变成亲历事件。
- 主要承压对象：地方共同体、资源分配链、风险处理链。
- 状态输出：`地方信任` 可明显升降；若忽视凡俗承重，后续地方将更防御。
- 白鹿之灵接口：禁止出现。
- 失败 / 代价反馈：承重事故会把代价直接压回聚落与营地关系。
- 后续去向：`MQ13`、`MQ14`

#### MQ13：资源风险交换

- 所属章节束：章节束 04 `地方承接束`
- 对应揭示层级：`R3`
- 进入条件：玩家已理解地方不只靠意志活着。
- 核心问题：为什么资源与风险处理能力会转化成地方权力。
- 节点目标：进入资源与风险交换带，理解“谁能处置风险，谁就会坐大”。
- 主要承压对象：地方势力与流动节点间的交换秩序。
- 状态输出：`地方信任` 与 `入口带稳定` 产生联动记录。
- 白鹿之灵接口：无。
- 失败 / 代价反馈：若处理方式过偏，会激化地方依附或灰线交易需求。
- 后续去向：`MQ14`、`MQ15`

#### MQ14：风险处置权

- 所属章节束：章节束 04 `地方承接束`
- 对应揭示层级：`R4` 前置
- 进入条件：玩家已看见交换带逻辑。
- 核心问题：风险处置为何不是纯技术问题，而是权力来源。
- 节点目标：让玩家面对地方坐大与守护残余默许之间的现实交易。
- 主要承压对象：地方势力、中心执行接口、资源带边缘群体。
- 状态输出：`残秩序合法性` 开始被地方现实重新评估。
- 白鹿之灵接口：不介入结构裁定。
- 失败 / 代价反馈：若只做单边站队，会提前推高 `同伴立场压力`。
- 后续去向：`MQ15`、`MQ16`

#### MQ15：节点通路

- 所属章节束：章节束 05 `流动托底束`
- 对应揭示层级：`R3`
- 进入条件：玩家已知地方现实不能闭门维持。
- 核心问题：残存通路为何仍是白鹿原续命骨架。
- 节点目标：进入通路 / 节点 / 入口残带，确认流通不是便利而是生存命脉。
- 主要承压对象：流动节点者与 `入口带稳定`。
- 状态输出：开启通路状态判断；玩家开始能比较“保通”和“封险”的代价。
- 白鹿之灵接口：只对白鹿接口受阻时发出边界异常警示。
- 失败 / 代价反馈：通路失守会直接压缩资源、消息与撤离选择。
- 后续去向：`MQ16`、`MQ17`

#### MQ16：入口资格

- 所属章节束：章节束 05 `流动托底束`
- 对应揭示层级：`R3`
- 进入条件：玩家已接触入口残带。
- 核心问题：入口带为什么是资格、交换与风险并存的复合边界。
- 节点目标：理解入口带不是“去外界的门”，而是白鹿原最脆弱的结构阀门。
- 主要承压对象：流动节点者、地方共同体边缘、灰线潜伏面。
- 状态输出：`入口带稳定` 可以显著升降；灰线触边的窗口开始出现。
- 白鹿之灵接口：仅反馈接口失真或接引残带撕裂。
- 失败 / 代价反馈：资格处理失衡会带来黑市流通或封口式报复。
- 后续去向：`MQ17`、`MQ18`

#### MQ17：流通失稳

- 所属章节束：章节束 05 `流动托底束`
- 对应揭示层级：`R3 -> R4`
- 进入条件：玩家已接触入口带与通路资格问题。
- 核心问题：当流通与封锁失衡时，白鹿原先崩的是哪一层。
- 节点目标：让玩家亲历入口带不稳对聚落、资源带和裂伤外缘的连锁冲击。
- 主要承压对象：`入口带稳定`、地方承接面与灰线潜伏面。
- 状态输出：玩家必须在“保命式封锁”与“续命式放行”间做阶段性选择。
- 白鹿之灵接口：只提示法宝接口承压上升，不提供解法。
- 失败 / 代价反馈：入口失稳会提前打开灰线与裂伤线的后续入口。
- 后续去向：`MQ18`、`MQ19`

#### MQ18：单线极致对照

- 所属章节束：章节束 06 `飞升差异束`
- 对应揭示层级：`R4`
- 进入条件：玩家已意识到同一套地方秩序对所有飞升者并不等价。
- 核心问题：守序、剑器、愿力三类飞升者为何不能被当作同一种新人处理。
- 节点目标：通过同一压力场下的不同应对，建立三类飞升路径差异。
- 主要承压对象：三位飞升席位角色与 `同伴立场压力`。
- 状态输出：同伴开始围绕“先稳结构还是先破局”形成明确立场倾向。
- 白鹿之灵接口：无。
- 失败 / 代价反馈：若强行抹平路径差异，后续异质秩序线会显得失真。
- 后续去向：`MQ19`、`MQ20`

#### MQ19：异质接口预置

- 所属章节束：章节束 06 `飞升差异束`
- 对应揭示层级：`R4` 向 `R5` 过渡
- 进入条件：玩家已看见三类飞升者的立足方式差异。
- 核心问题：白鹿原内部是否存在不完全服从守护残余逻辑的承认方式。
- 节点目标：为第三幕“异质承认”预埋现实接口。
- 主要承压对象：愿力飞升者、中心执行接口与地方承接层之间的认知张力。
- 状态输出：`残秩序合法性` 不再是唯一评判框架。
- 白鹿之灵接口：只能在接引残忆碎片里折射“承认链并非单一”。
- 失败 / 代价反馈：若将异质接口简单视为灰线阴影，后续 R5/R6 揭示会更陡更险。
- 后续去向：`MQ20`、`MQ21`

#### MQ20：地方依附

- 所属章节束：章节束 04 `地方承接束`
- 对应揭示层级：`R4`

- 核心问题：地方势力为什么依附残秩序而不是直接脱离。
- 节点目标：把“地方坐大”从道德判断压回结构选择。
- 主要承压对象：地方势力、中心执行接口与现实安全需求。
- 状态输出：玩家开始理解守护残余为何能继续有现实抓手。
- 白鹿之灵接口：无。
- 失败 / 代价反馈：若只用善恶判断地方依附，后续灰线线索会被误读。
- 后续去向：`MQ21`、`MQ22`

#### MQ21：灰线风闻

- 所属章节束：章节束 07 `托底翻转束`
- 对应揭示层级：`R4 -> R5`

- 节点目标：让灰线从传闻进入可追索线索，但仍保持危险性。
- 主要承压对象：`灰线接触` 与入口带边缘群体。
- 状态输出：`灰线接触` 由无到有；玩家开始可从灰线视角反推旧制盲区。
- 白鹿之灵接口：禁止直接认证灰线真假。
- 失败 / 代价反馈：灰线接触过早过深会让入口带更不稳、同伴更分裂。
- 后续去向：`MQ22`、`MQ23`

#### MQ22：托底确认

- 进入条件：地方承接、流动托底与灰线风闻均已建立。
- 核心问题：白鹿原到底靠什么苟续。
- 节点目标：把“法宝唯一托底”修正为“法宝、地方承接、流动网络共同托底”。
- 主要承压对象：三类托底结构的平衡关系。
- 状态输出：第二幕的现实结构认知收束，后续可以进入深因追问。
- 白鹿之灵接口：可以作为一条边界回响佐证法宝承压，不可独自定论。
- 失败 / 代价反馈：若仍把法宝当万能底盘，第三幕历史翻面会更痛、更危险。

#### MQ23：深因追问

- 核心问题：为什么白鹿原必须以这种代价结构维持下去。
- 节点目标：把玩家问题从“怎么活”抬到“为什么会沦为这样”。
- 主要承压对象：守序飞升者、中心执行接口与地方解释框架。
- 状态输出：第三幕入口开启，灰线与历史线不再是旁枝。
- 白鹿之灵接口：只在追问接近接引残忆触发点时留下残片指向。
- 失败 / 代价反馈：若绕开深因，第三幕会以更高损失强制补课。
- 后续去向：`MQ24`、`MQ25`

#### MQ24：第二幕收束

- 对应揭示层级：`R5` 的前夜
- 进入条件：深因追问已展开。
- 核心问题：是否继续把白鹿原问题理解为地方治理问题。
- 节点目标：结束“中段跑图”错觉，把主线正式推入历史与法宝绑定线。
- 主要承压对象：主角团整体判断。
- 状态输出：第二幕结束；第三幕灰线、裂伤外缘、法宝翻面线全部开放。

- 失败 / 代价反馈：若玩家仍将核心问题压回地方小冲突，后续同伴会明确指出认知不足。
- 后续去向：`MQ25`、`MQ27`

## 6. Act 3：大战真相与法宝绑定

### 6.1 Act 目标

- 让灰线从危险传闻变成结构证据。
- 让异质承认、法宝承认链、大战推离成因与复生来源互相扣合。
- 让主角团从旁观干预者转为已被卷入的承压变量。
- 让“清剿某一方即可解决”彻底失效。

### 6.2 节点列表

#### MQ25：灰线触边

- 所属章节束：章节束 08 `灰线显形束`
- 对应揭示层级：`R5`
- 进入条件：第二幕已留下灰线风闻与入口残带裂口。
- 核心问题：灰线为何总沿裂缝、节点与入口边界活动。
- 节点目标：在入口残带与裂伤外缘交界处捕捉灰线的结构性轨迹。
- 主要承压对象：`灰线接触`、`入口带稳定`
- 状态输出：灰线从“风闻对象”转为“可追索结构面”。
- 白鹿之灵接口：仅对白鹿接口受裂缝污染发出示警。
- 失败 / 代价反馈：追缉过猛会把灰线压回隐匿层，同时抬高地方恐慌。
- 后续去向：`MQ26`、`MQ27`

#### MQ26：裂伤外缘

- 所属章节束：章节束 08 `灰线显形束`
- 对应揭示层级：`R5`
- 进入条件：灰线触边已成立。
- 核心问题：大战残伤为何仍在为当前冲突持续供血。
- 节点目标：进入裂伤外缘，看到风险、资源、高位残响与灰线生长如何捆在一起。
- 主要承压对象：探索队、流动节点者、地方外围。
- 状态输出：`灰线接触` 与 `法宝承压` 同时上升的可能性被建立。
- 白鹿之灵接口：只在裂伤直接勾连接引残忆时给出破碎记忆影像。
- 失败 / 代价反馈：裂伤探索失败会造成认知缺口与后续证据不完整。
- 后续去向：`MQ27`、`MQ28`

#### MQ27：灰线渗透

- 所属章节束：章节束 08 `灰线显形束`
- 对应揭示层级：`R5`
- 进入条件：玩家已掌握灰线外部活动与裂伤资源逻辑。
- 核心问题：反抗残余为何能长期潜伏，而不只是一群逃亡者。
- 节点目标：确认灰线依附承认盲区、入口残带与地方裂口生长。
- 主要承压对象：地方共同体的盲区、守护残余的边界治理。
- 状态输出：灰线不再等于单纯敌对势力，反派框架开始松动。
- 白鹿之灵接口：无。
- 失败 / 代价反馈：若粗暴处理灰线，会把证据链压碎并提前拉高对抗。
- 后续去向：`MQ28`、`MQ29`

#### MQ28：异质承认

- 所属章节束：章节束 09 `法宝翻面束`
- 对应揭示层级：`R5 -> R6`
- 进入条件：飞升差异与灰线结构都已被建立。
- 核心问题：除了守护残余逻辑外，此地是否还存在另一种组织人的秩序方式。
- 节点目标：让异质秩序从抽象猜想变成现实接口。
- 主要承压对象：愿力飞升者、地方承接层、部分灰线接口。
- 状态输出：`残秩序合法性` 不再独占正当性；同伴内部开始出现路线预判。
- 白鹿之灵接口：只可折射承认链并非单线，不允许替异质秩序背书。
- 失败 / 代价反馈：若把异质承认视作纯叛乱，后续法宝承认链将更难理解。
- 后续去向：`MQ29`、`MQ30`

#### MQ29：法宝承认链

- 所属章节束：章节束 09 `法宝翻面束`
- 对应揭示层级：`R6`
- 进入条件：玩家已理解异质承认的存在可能。
- 核心问题：接引法宝为何不只是工具，而是白鹿原承认链的核心接口。
- 节点目标：通过多方证据确认法宝决定的是“谁被留下、谁被接回、谁被牺牲”的结构逻辑。
- 主要承压对象：法宝核心影响区、守护残余与灰线解释框架。
- 状态输出：`法宝承压` 进入主线中心层；后续所有终局路线都必须回到法宝。
- 白鹿之灵接口：允许显化一次，但只能确认“承认链正在撕裂”，不能交代完整机制。
- 失败 / 代价反馈：误调法宝会导致白鹿之灵反馈更碎、复生代价更重。
- 后续去向：`MQ30`、`MQ31`

#### MQ30：接引残忆

- 所属章节束：章节束 09 `法宝翻面束`
- 对应揭示层级：`R6`
- 进入条件：法宝承认链已成为显性问题。
- 核心问题：过去的接引秩序是如何被拖入白鹿原当前状态的。
- 节点目标：拼合接引残忆、环境证据与当下失衡之间的逻辑关系。
- 主要承压对象：白鹿接口、主角团认知负荷、法宝承压。
- 状态输出：玩家确认接引残忆只能被拼合，而不能被完整回放。
- 白鹿之灵接口：只给残片，不给全片；只给方向，不给裁决。
- 失败 / 代价反馈：残忆读取过深会在后续复生、营地对话与判断里留下错位。
- 后续去向：`MQ31`、`MQ32`

#### MQ31：推离成因

- 所属章节束：章节束 09 `法宝翻面束`
- 对应揭示层级：`R6`
- 进入条件：残忆与灰线结构证据已形成拼合基础。
- 核心问题：白鹿原为何会连同法宝一起被推离主世界常规秩序带。
- 节点目标：把大战余波从背景设定转成对白鹿原现状的直接解释。
- 主要承压对象：裂史携行者、中心执行接口与守序飞升者判断。
- 状态输出：主线历史解释从地方灾变抬升为大战后结构推离。
- 白鹿之灵接口：只回响与接引相关的断裂，不讲大战全貌。
- 失败 / 代价反馈：若证据拼合不足，后续残余布局会更像阴谋清单而非结构后果。
- 后续去向：`MQ32`、`MQ33`

#### MQ32：残余布局

- 所属章节束：章节束 09 `法宝翻面束`
- 对应揭示层级：`R6`
- 进入条件：推离成因已成立。
- 核心问题：为什么守护残余与反抗残余都未彻底放弃此地。
- 节点目标：确认白鹿原是大战后双方都留有抓手的残存支点。
- 主要承压对象：守护残余、灰线网络、地方被裹挟面。
- 状态输出：反派框架继续崩解，阵营判断开始转向结构判断。
- 白鹿之灵接口：无。
- 失败 / 代价反馈：若处理成“某方幕后黑手”，会伤害后续代价比较的复杂度。
- 后续去向：`MQ33`、`MQ34`

#### MQ33：复生来源

- 所属章节束：章节束 09 `法宝翻面束`
- 对应揭示层级：`R6 -> R7`
- 进入条件：法宝承认链与大战推离因果已被建立。
- 核心问题：主角团为何能复生，以及复生到底在消耗什么。
- 节点目标：确认复生是法宝绑定带来的代价型结构回返。
- 主要承压对象：`法宝承压`、主角团身心损耗、白鹿接口稳定度。
- 状态输出：复生不再是机制设定，而是剧情核心代价。
- 白鹿之灵接口：允许指出“回返不是免费”，不允许说明全部损耗模型。
- 失败 / 代价反馈：每次错误理解都会在后续回返时加重痛感、延迟或残忆错乱。
- 后续去向：`MQ34`、`MQ35`

#### MQ34：绑定代价

- 所属章节束：章节束 09 `法宝翻面束`
- 对应揭示层级：`R7`
- 进入条件：复生来源已确认。
- 核心问题：主角团与法宝绑定给白鹿原和自身都带来了什么新代价。
- 节点目标：把“主角团被卷入”从认知层推进到现实承压层。
- 主要承压对象：主角团内部、法宝核心影响区、地方承接层。
- 状态输出：`同伴立场压力` 明显上升；是否继续调用法宝成为路线判断问题。
- 白鹿之灵接口：只能作为边界回响，提示承压上限逼近。
- 失败 / 代价反馈：若无视绑定代价，后续某些路线将自动转入高风险版本。
- 后续去向：`MQ35`、`MQ36`

#### MQ35：命运卷入

- 所属章节束：章节束 09 `法宝翻面束`
- 对应揭示层级：`R7`
- 进入条件：主角团与法宝代价关系已明确。
- 核心问题：为何主角团已不能把自己视作外来调停者。
- 节点目标：让玩家接受“离开”本身也会是一种对白鹿原的改写。
- 主要承压对象：主角团自我定位、同伴路线判断。
- 状态输出：第三幕后半转入“必须承担”的结构位置。
- 白鹿之灵接口：允许提示“你们已被写入”，但不说明终局必须怎么选。
- 失败 / 代价反馈：若仍试图把自己置外，第四幕前会出现更强烈的现实反证。

- 节点目标：彻底打碎“守护是正、反抗是邪”或相反的单向判断。
- 主要承压对象：裂史携行者、愿力飞升者、中心执行接口。

#### MQ37：修复代价预示

- 所属章节束：章节束 10 `认知崩解束`
- 对应揭示层级：`R8` 前置
- 进入条件：反派框架已崩解。
- 核心问题：为什么“修复白鹿原”本身也会制造新的承压者。
- 节点目标：把终局问题从“恢复正常”改成“如何分配代价”。
- 主要承压对象：地方共同体、入口带、法宝承压面、守护残余执行链。
- 状态输出：第四幕的代价聚焦线被明确激活。
- 白鹿之灵接口：只允许示警“任何调取都会留痕”，不作路径建议。
- 失败 / 代价反馈：若玩家仍期待零代价修复，第四幕会以更剧烈分歧回击。
- 后续去向：`MQ38`、`MQ39`

#### MQ38：第三幕收束

- 所属章节束：章节束 10 `认知崩解束`
- 对应揭示层级：`R7 -> R8`
- 进入条件：第三幕核心揭示已完成。
- 核心问题：主角团是否准备好进入真正的代价比较。
- 节点目标：把第三幕收束成“知道更多，不等于答案更轻”。
- 主要承压对象：主角团整体立场与营地气氛。
- 状态输出：第四幕正式开启；同伴分歧与法宝路径比较进入强显性。
- 白鹿之灵接口：无。
- 失败 / 代价反馈：若此处仍回避承担问题，同伴会在第四幕前段直接逼问。
- 后续去向：`MQ39`、`MQ40`

## 7. Act 4：地域命运的决断

### 7.1 Act 目标

- 把所有前段冲突压回“谁承担何种代价”。
- 把重连、继续封闭、重启、重组四类方向做成可比较路径，而非口号。
- 让七席位分歧与现实承压同步上桌。
- 让终局收束于白鹿原的受限未来，而不是绝对胜利。

### 7.2 节点列表

#### MQ39：法宝回收

- 所属章节束：章节束 11 `代价决断束`
- 对应揭示层级：`R8`
- 进入条件：第三幕已明确法宝是承认链与复生接口核心。
- 核心问题：为什么所有路径最终都绕不过法宝核心牵引面。
- 节点目标：把终局舞台重新压回法宝核心影响区与相关承压面。
- 主要承压对象：法宝核心影响区、中心执行接口、主角团绑定关系。
- 状态输出：所有终局比较以法宝路径作为公共前提。
- 白鹿之灵接口：可以显化为“边界过载警示”，不得说明最优路线。
- 失败 / 代价反馈：错误回收会直接提升 `法宝承压` 并削减后续可选空间。
- 后续去向：`MQ40`、`MQ41`

#### MQ40：承压面聚拢

- 所属章节束：章节束 11 `代价决断束`
- 对应揭示层级：`R8`
- 进入条件：法宝核心回收线已开始。
- 核心问题：谁正在承受白鹿原延续、变更或失稳的第一重现实代价。
- 节点目标：把地方共同体、入口带、守护残余执行链、灰线与法宝承压面聚到同一张桌上。
- 主要承压对象：地方承接层、流动节点者、中心执行者。
- 状态输出：终局比较不再抽象；每条路线都绑定承压对象。
- 白鹿之灵接口：仅反馈法宝承压上限变化。
- 失败 / 代价反馈：若忽略承压面，后续路线比较会丢失现实合法性。
- 后续去向：`MQ41`、`MQ42`

#### MQ41：代价镜像

- 所属章节束：章节束 11 `代价决断束`
- 对应揭示层级：`R8`
- 进入条件：承压面已经聚拢。
- 核心问题：为什么任何路线都不是零代价方案。
- 节点目标：以镜像结构展示重连、封闭、重启、重组的首批代价差异。
- 主要承压对象：四条路线各自对应的第一承压者。
- 状态输出：玩家不再寻找“正确答案”，而开始计算“谁付出什么”。
- 白鹿之灵接口：只可提示承压极限，不可替路线背书。
- 失败 / 代价反馈：若只看抽象价值而不看承压者，部分同伴会直接失去认同。
- 后续去向：`MQ42`、`MQ43`

#### MQ42：团队分歧

- 所属章节束：章节束 11 `代价决断束`
- 对应揭示层级：`R8`
- 进入条件：路线代价镜像已经摆上桌。
- 核心问题：为什么看见同样的现实后，七席位仍会给出不同判断。
- 节点目标：把同伴立场压力从潜流转为必须处理的终局条件。
- 主要承压对象：`同伴立场压力`
- 状态输出：同伴不再只是评论者，而成为路线合法性、风险与代价的共同承载面。
- 白鹿之灵接口：不得介入团队裁决。
- 失败 / 代价反馈：若压制分歧而不是处理分歧，终局落地会更残缺且营地反馈更尖锐。
- 后续去向：`MQ43`、`MQ44`

#### MQ43：重连代价

- 所属章节束：章节束 11 `代价决断束`
- 对应揭示层级：`R8`
- 进入条件：终局路径比较正式开始。
- 核心问题：让白鹿原重新接回主世界常规秩序带意味着什么。
- 节点目标：具体化重连的收益、代价、承压者与被重排者。
- 主要承压对象：地方共同体、守护残余、入口带、外界承认链。
- 状态输出：重连路线从“求援幻想”转成“高代价制度重排”。
- 白鹿之灵接口：只提示接引链强制重接的危险，不支持方案宣传。
- 失败 / 代价反馈：若轻视重连后果，会丢失部分地方支持或同伴认同。
- 后续去向：`MQ45`、`MQ46`

#### MQ44：封闭代价

- 所属章节束：章节束 11 `代价决断束`
- 对应揭示层级：`R8`
- 进入条件：玩家已理解重连代价。
- 核心问题：继续封闭白鹿原究竟是在保命，还是在延长消耗。
- 节点目标：具体化继续封闭的稳定收益与长期损耗。
- 主要承压对象：地方承接层、流动节点者、灰线生长面。
- 状态输出：封闭路线从“保守选项”转成“慢性消耗方案”。
- 白鹿之灵接口：可示警结构尚可维持，但不能宣称“继续封闭就安全”。
- 失败 / 代价反馈：若把封闭写成无脑保守，会导致路线失真并削弱后续选择重量。
- 后续去向：`MQ45`、`MQ46`

#### MQ45：重启 / 重组比较

- 所属章节束：章节束 11 `代价决断束`
- 对应揭示层级：`R8`
- 进入条件：重连与继续封闭的代价已被具体化。
- 核心问题：重启旧结构与重组新结构各自会牺牲什么、保留什么。
- 节点目标：把“重启”与“重组”从抽象理想转成具体承认链改写方案。
- 主要承压对象：法宝核心、地方共同体、灰线网络、同伴关系。
- 状态输出：四类路线比较闭环建立。
- 白鹿之灵接口：只反馈承认链撕裂风险与法宝承压上限。
- 失败 / 代价反馈：若将重组浪漫化或将重启机械化，都会让终局变得单薄。
- 后续去向：`MQ46`、`MQ47`

#### MQ46：代价承担者比较

- 所属章节束：章节束 11 `代价决断束`
- 对应揭示层级：`R8`
- 进入条件：四类路径均已具体化。
- 核心问题：究竟谁来承担白鹿原未来的第一责任与首轮伤害。
- 节点目标：让玩家在真实承压者之间做判断，而不是在口号之间投票。
- 主要承压对象：地方承接层、守护残余、灰线网络、主角团、入口带与法宝核心。
- 状态输出：终局选择条件被锁定；Epilogue 开始只负责落地，不再重新辩论世界观。
- 白鹿之灵接口：只能显示代价接近阈值，不能代为决定。
- 失败 / 代价反馈：若玩家试图逃避比较，系统必须把后果以更坏版本落在现实层。
- 后续去向：`MQ47`

## 9. 横向接口要求

### 9.1 状态维度与节点绑定要求

后续场景卡与任务层必须沿用以下最低绑定：

- `地方信任`：至少在 `MQ04–MQ05`、`MQ11–MQ14`、`MQ40–MQ47` 出现显性变化。
- `残秩序合法性`：至少在 `MQ02`、`MQ06`、`MQ14`、`MQ28–MQ32`、`MQ43–MQ45` 出现重估。
- `灰线接触`：至少在 `MQ21`、`MQ25–MQ27`、`MQ32`、`MQ44–MQ45` 出现升级。
- `入口带稳定`：至少在 `MQ08`、`MQ15–MQ17`、`MQ21`、`MQ40`、`MQ43–MQ47` 出现联动。
- `法宝承压`：至少在 `MQ03`、`MQ07`、`MQ29–MQ35`、`MQ39–MQ48` 持续可见。
- `同伴立场压力`：至少在 `MQ05`、`MQ18–MQ19`、`MQ34–MQ42`、`MQ47–MQ48` 形成曲线。

### 9.2 白鹿之灵使用硬限制

- 白鹿之灵只能在绑定、复生、接引残忆、承认链撕裂或结构过载时被触发。
- 白鹿之灵输出只允许是识别、示警、残片、回响与边界反馈。
- 白鹿之灵不得替地方承接层讲现实，不得替守护残余讲结构，不得替灰线讲历史，不得替玩家做路线判断。
- 若某个节点无法在没有白鹿之灵完整说明的情况下成立，则该节点设计不合格。
