# Book1 支线范围矩阵

## 1. 矩阵目的

本矩阵不是“所有支线都开放”的许可表。  
它只定义：在 `Book1` 当前已完成的审计链下，哪些支线范围已经有受控落点，哪些仍冻结。

## 2. 范围矩阵

| 卷 / 幕 | 已审支线范围 | 当前状态 | 说明 |
| --- | --- | --- | --- |
| `V1 / A1 / CH01–CH03` | `SQ-SCOPE-01《借来的承接位》` | `OPEN` | 只允许附着 `CH02-S02–S03` 的照护棚 -> 账本室链。 |
| `V1 / A2 / CH04–CH06` | `SQ-SCOPE-02《断路前的空名册》` | `OPEN` | 只允许附着 `CH04-S01–S02` 的前哨 -> 残桥窗口链。 |
| `V2 / A3 / CH07–CH09` | 未命名新支线 | `FROZEN` | 当前未做支线验证，不得私开。 |
| `V2 / A4 / CH10–CH12` | 未命名新支线 | `FROZEN` | 当前未做支线验证，不得私开。 |

## 3. 调用规则

1. `OPEN` 只表示该范围内已有受控支线接口，不表示可无限扩写。
2. `FROZEN` 不是空白，而是明确禁止在未补同等级验证前新增支线正文或任务层。
3. 任何新增支线都必须先进入矩阵，再进入正文或任务层。
