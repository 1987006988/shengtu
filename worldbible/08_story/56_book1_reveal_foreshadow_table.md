# 第一部详细揭示与伏笔表

## 1. 文件定位

本文件用于锁死信息顺序。  
它回答：

- 什么信息第几章可以出现；
- 什么信息必须后置；
- 哪些伏笔先埋，什么时候回收；
- 哪些真相只能由哪类角色 / 事件带出。

## 2. 使用原则

1. **先现实，后结构，再历史，最后代价比较。**
2. 任一揭示若不改变行动条件，应降级为环境文本或 Codex。
3. 任一伏笔若没有明确回收点，不得硬埋。
4. 任一真相若提前说满，必须视为蓝图事故。
