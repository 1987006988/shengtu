# 第一部详细世界状态变化表

## 1. 文件定位

本文件负责记录白鹿原在每章后的状态变化。  
它回答：

- 哪些区域开放、封锁、失稳或重构；
- 哪些势力关系发生变化；
- 哪些变化来自角色推动，哪些来自剧情大事件；
- 哪些变化绝对不能提前发生。

## 2. 状态控制原则

1. 白鹿原的变化必须是逐层发生的，不能突然地图大开；
2. 每章都至少改动一项：
   - 区域可达性
   - 势力关系
   - 公共认知
   - 主角与白鹿关系
3. 世界状态变化必须与角色与揭示同步；
4. 世界状态变化不能被纯文本量取代。

## 5. 世界状态调用规则

1. `58` 只负责压实章末世界状态增量、锁死项、不可回退项与下一章唯一合法起手状态，不得反向改写 `54 / 55` 已锁死的章场承压链。
