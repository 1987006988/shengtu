# 第一部详细世界状态变化表

## 1. 文件定位

本文件负责记录白鹿原在每章后的状态变化。  
它回答：

- 哪些区域开放、封锁、失稳或重构；
- 哪些势力关系发生变化；
- 哪些变化来自角色推动，哪些来自剧情大事件；
- 哪些变化绝对不能提前发生。

## 2. 状态控制原则

1. 白鹿原的变化必须是逐层发生的，不能突然地图大开；
2. 每章都至少改动一项：
   - 区域可达性
   - 势力关系
   - 公共认知
   - 主角与白鹿关系
3. 世界状态变化必须与角色与揭示同步；
4. 世界状态变化不能被纯文本量取代。

## 4. 终局后白鹿原默认状态

### 4.1 成立的状态
- 白鹿原进入 `有限重开、共享承责、保留缓冲` 的受限未来；
- 守衡司不再拥有绝对单边签字权；
- 共保会从被动承压主体升级为共责主体；
- 行栈纳入白鹿原未来的共同责任结构；
- 灰缄线失去一次性改写未来的窗口，但没有彻底消失；
- 主角与白鹿原形成长期回引关系。

### 4.2 未成立、必须留到后续的状态
- 古裂残天总体重构；
- 白鹿原完全稳定；
- 所有失地恢复；
- 各派仇怨清账；
- 九主世界视野全面打开。

## 5. 一句话口径

> 世界状态表的意义，在于确保白鹿原不是一个永远不变的舞台，而是一块会被选择真实改写的土地。
