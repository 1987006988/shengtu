# 159 Book1 Change Control

## 1. Purpose

This file governs what can be edited directly during `Book1` production mode and what must be rechecked upstream.

## 2. Direct Edit Zone

The following can be revised in place, as long as the revision stays inside the approved scope:

- prose bridge quality
- scene flow
- point-of-view stability
- chapter-end hooks
- task-layer anchoring
- reveal wording
- state-trigger wording
- formal sidequest wording inside the approved windows
- the final production review text

## 3. Upstream Review Zone

Any explanation-level change that affects one of these tables or files must be checked upstream before it lands:

- `15 / 16`
- `21 / 22`
- `54 / 55 / 56 / 57 / 58`

## 4. Hard Prohibitions

- Do not create a second `CH01–CH18` truth.
- Do not create a second mainline task-layer truth.
- Do not reopen `A3–A6` sidequests.
- Do not reopen `Item / System feedback`.
- Do not revive old batch or narrative-lab assets as current truth.
- Do not turn open-scope production into "全面开放" language.

## 5. Gate Rule

If a change might alter what `56` reveals, how `57` advances roles, or how `58` resolves state, then the change is not a local downstream edit. It must be reviewed against the upstream mother tables first.
