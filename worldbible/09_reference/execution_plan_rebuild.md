# 执行计划（重构后）

## 1. 计划性质

本文件不是抽象项目口号，  
而是**下一轮工作实际该怎么做**的施工计划。

## 2. 当前执行目标

用最少的继续偏航成本，把项目切换到下面这条正确链路：

> 当前状态纠正 → 古裂残天总层 → 白鹿原地区圣经 → 七席位总控 → 第一部蓝图闭环 → 任务化试点

## 4. 完成以上后才允许做的事

在 `50–58` 系列闭环后，才允许：

1. Act I 的主线任务化试点；
2. Act I 的 Journal / Codex / 白鹿反馈最小样例；
3. 1–2 条支线试点；
4. 第一卷少量正文试写用于验证蓝图。

## 5. 仍然禁止做的事

即使蓝图已成，在试点验证前仍禁止：

- 恢复旧式大规模 batch schedule；
- 一口气写完整部所有 MQ / SQ 正文包；
- 大批量铺同伴对白包与系统反馈包。

## 6. 每轮工作都要做的自检

每轮工作结束前必须回答：

1. 本轮是在补上游还是补下游？
2. 本轮是否让下游反向定义了上游？
3. 本轮是否制造了新的“完成错觉”？
4. 本轮输出是否能被后续 Codex 直接调用，而不是只会读完点头？

## 7. 一句话执行口径

> 下一步不是“继续写包”，而是“先把承重结构和施工蓝图补齐”。
