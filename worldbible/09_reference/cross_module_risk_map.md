# 跨模块风险图

## 1. 文档用途

本文档用于标记当前 `01–09` 之间最值得优先关注的跨模块高风险接口。

## 2. 高风险接口清单

### 风险 1：`01_cosmology` <-> `08_story`

- 风险类型：尺度冲突 / 因果断裂
- 分层归类：属于“宇宙长期正典被第一部强绑定设定误读或误回写”的高风险接口
- 当前表现：
  - 宇宙层已经确认飞升、接引、世界群与高低位关系。
  - 故事层已把接引法宝、隔绝地域、复生机制收束回主线中心。
  - 但 `01_cosmology` 关键文件有乱码损坏，导致上层口径难以直接核验。
- 建议处理方式：
  - 优先修复 `02_ascension.md` 与 `04_macro_cosmic_layout.md`。
- 是否需要人工拍板：否，优先属于文档修复问题。

### 风险 2：`04_main_world` <-> `08_story`

- 风险类型：尺度冲突
- 分层归类：属于“`古裂残天` 长期正典被第一部舞台误写”的高风险接口
- 当前表现：
  - `04_main_world` 写的是整个 `古裂残天` 总层。
  - `08_story` 写的是内部隔绝地域局部舞台。
  - 当前两者总体兼容，但后续若继续写故事层，容易再次把局部舞台膨胀回总层尺度。
- 建议处理方式：
  - 后续任何故事层扩写，都应显式注明“隔绝地域内部”。
- 是否需要人工拍板：否，属于持续执行护栏。

### 风险 3：`06_society` <-> `08_story`

- 风险类型：因果断裂 / 故事越界
- 分层归类：属于“`古裂残天` 总层社会被第一部局部社会混写”的高风险接口
- 当前表现：
  - 地方承接层、流动网络、空间入口带已经对齐隔绝地域的生存逻辑。
  - 但若进入任务层后不继续保持区分，容易把“古裂残天总层社会”与“隔绝地域局部社会”混写。
- 建议处理方式：
  - 后续任务层前先补一轮“社会层到故事层”的桥梁说明。
- 是否需要人工拍板：否。

### 风险 4：`07_characters` <-> `08_story`

- 风险类型：角色越界 / 尺度冲突
- 分层归类：属于“第一部强绑定设定容易借角色接口无声上浮”的高风险接口
- 当前表现：
  - 角色框架总体成立。
  - 但部分早期角色接口文件仍偏主世界总层，且关键角色原型文件存在乱码损坏。
- 建议处理方式：
  - 优先修复 `08_party_archetype_cards.md` 与 `11_character_concept_cards.md`。
  - 再检查角色接口是否都已稳定收束到隔绝地域内部。
- 是否需要人工拍板：部分需要，尤其是“角色接口最终收多窄”。

### 风险 5：`09_reference` <-> 全仓库

- 风险类型：术语冲突 / 协作摩擦
- 分层归类：属于“三层分层已存在，但未必总被执行层直接承接”的护栏风险
- 当前表现：
  - `09_reference` 记录基本正确。
  - 但当源文件损坏时，参考层虽然能提示正确口径，却不能代替正典正文。
- 建议处理方式：
  - 继续把 `09_reference` 当作协作护栏。
  - 但不要把它当成损坏正文的永久替代品。
- 是否需要人工拍板：否。
