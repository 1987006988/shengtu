# 新总路线图（重构版）

## 1. 总原则

项目的新默认路线不再是：

> 项目总纲 → 大量主线 / 支线 / 同伴 / 系统文本包 → review → acceptance

而应改为：

> 项目总纲 → 主世界承载层 → 地区圣经 → 角色接口总控 → 第一部蓝图闭环 → 小规模任务化试点 → 文本生产

## 2. 新阶段顺序

### 阶段 0：治理与入口重置
**目标**：让所有协作者先看到真实状态，而不是旧批次工程的完成幻觉。  
**交付**：

- 新 `current_state`
- 新 `session_handoff`
- 新 `reading_order`
- 新 `execution_plan`
- 总审判决书
- 文件操作清单

**完成标志**：

- 旧 `END / final acceptance` 不再被作为当前真实状态引用；
- 后续协作者一进入仓库就能读到新的阶段口径。

---

### 阶段 1：宇宙长期总纲与九主世界调用层稳固
**目标**：不再只靠差异卡，而是给未来系列扩写留下可调用的世界矩阵。  
**交付**：

- `07_universe_longline_master.md`
- `17_nine_main_world_longterm_usage_matrix.md`

**必须完成的点**：

- 明确九主世界各自承担什么长期叙事功能；
- 明确哪些主世界后期才进入主线，避免第一部抢位；
- 明确古裂残天在九主世界中的独特长期地位。

**当前最短板**：不是数量，而是调用逻辑与长期用途清晰度。

---

### 阶段 2：古裂残天总层重建
**目标**：把古裂残天从“概念成立”推进到“能长期承载多地区、多伤区、多制度冲突的主世界”。  
**交付**：

- `14_gulie_cantian_rebuild_bible.md`
- `09_bailuyuan_prestory_timeline_rebuild.md`

**必须完成的点**：

- 宏观地理与伤区分布
- 治理结构与主要力量
- 修复 / 改写 / 迁转三条长期轴
- 地方共同体、守衡司、灰线、路权网络如何在主世界级继续成立

**当前最短板**：古裂残天还不够“世界”，更像“很好用的设定定位”。

---

### 阶段 3：白鹿原地区圣经重建
**目标**：把白鹿原从“功能地图”升级成“地区圣经”。  
**交付**：

- `15_bailuyuan_region_bible.md`
- `16_book1_binding_boundary.md`

**必须完成的点**：

- 聚落、通路、危险区、价格体系、配额体系、记忆区、核心区全部具体化；
- 至少形成 5–6 个可持续调用的区域；
- 至少形成 4–5 个真正能推动剧情的势力；
- 明确白鹿之灵显化边界与接引法宝功能边界；
- 明确第一部能揭示什么，不能揭示什么。

**当前最短板**：这是当前项目的第一短板。

---

### 阶段 4：七席位角色总控重建
**目标**：把角色从“概念卡 / 任务插话容器”重建为真正承担世界接口与命题功能的长期角色。  
**交付**：

- `21_party_story_function_bible.md`
- `22_character_progression_master.md`

**必须完成的点**：

- 每位角色承担的世界接口不重叠；
- 每位角色的前 / 中 / 后段抬权曲线明确；
- 哪些角色负责真相，哪些角色负责代价，哪些角色负责现实，哪些角色负责历史，全部锁定；
- 不再让同伴工程反向定义世界层。

**当前最短板**：角色已太多，真正稳的角色总控反而不够。

---

### 阶段 5：第一部蓝图闭环
**目标**：在**不进入任务层**的前提下，把第一部写到后续写作者几乎不用再猜。  
**交付**：

- `50_series_master_outline.md`
- `51_book1_master_outline.md`
- `52_book1_volume_structure.md`
- `53_book1_act_structure.md`
- `54_book1_chapter_blueprint.md`
- `55_book1_scene_blueprint.md`
- `56_book1_reveal_foreshadow_table.md`
- `57_book1_character_progression_table.md`
- `58_book1_world_state_table.md`

**必须完成的点**：

- 分卷成立；
- 分幕成立；
- 分章成立；
- 分场成立；
- 揭示顺序成立；
- 角色推进表成立；
- 世界状态变化表成立。

**完成标志**：

- Codex 可以直接按章 / 按场继续写；
- 但还没有进入 MQ / SQ / Journal / Codex / System feedback 生产层。

---

### 阶段 6：任务化试点（只允许小范围）
**目标**：不是全面恢复旧 batch，而是在蓝图闭环后做小范围转译试点。  
**只允许做**：

- 第一卷 / Act I 的主线任务映射试点；
- 第一卷 1–2 条支线试点；
- 白鹿之灵、Journal、Codex 的最小调用样例。

**不允许做**：

- 直接恢复 33a–43a 式全面文本包生产；
- 直接开第二轮 batch schedule；
- 在没有验证蓝图有效前重建 massive text warehouse。

---

### 阶段 7：文本生产与系统扩容
**目标**：等蓝图与试点都稳定后，再进入真正的大体量生产。  
**这时才允许**：

- Act 1 → Act 6 的任务化生产；
- 同伴对白包；
- Codex / Journal / Item / System feedback 扩容；
- 失败回流与多分支生产。

## 3. 哪些层必须先完成

必须优先完成的顺序是：

1. **当前状态与入口纠正**
2. **古裂残天总层**
3. **白鹿原地区圣经**
4. **七席位总控**
5. **第一部蓝图闭环**
6. **任务化试点**
7. **文本大生产**

## 4. 哪些层不该过早做

在阶段 5 完成前，以下内容都不该再成为默认工作重心：

- 任务正文包
- 大规模对白包
- Journal / Codex / Item / System feedback 大包
- 批次化 review / acceptance 工程
- “整部故事已完成”的口径维护

## 5. 当前最短板排序

1. **白鹿原地区圣经缺失**
2. **古裂残天总层厚度不足**
3. **项目状态口径失真**
4. **角色接口与舞台具体度不匹配**
5. **第一部蓝图没有真正闭环**
6. **旧批次工程占据主入口**
7. **任务层过早默认开放**

## 6. 新路线的一句话版本

> **先把世界和舞台做实，再把角色做稳，再把第一部蓝图闭环，然后才允许任务化与文本化。**
