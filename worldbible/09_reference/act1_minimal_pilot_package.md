# Act I 最小任务化试点包

## 1. 文件定位

本文件不是旧 batch，也不是全面任务层入口。  
它只服务一个目标：

> 在蓝图一致性审计通过后，把《封绝之地》允许进入任务层的**最小试点**正式压成可执行包。

当前试点只允许服务：

- `Act I`
- `CH01–CH02`
- 一条最小主线试点
- 最小 `Journal / Codex` 样例

不默认放开支线正文包、系统反馈大产线或旧 narrative-lab 节奏。

## 2. 试点范围锁定

### 2.1 当前唯一试点范围

- 试点章节：`CH01《坠落不死》` 与 `CH02《鹿门的价钱》`
- 试点主线：`主角坠入白鹿原外缘 -> 被护撤 -> 被押送鹿门集 -> 进入风险登记与结构审查前夜`
- 试点目标：
  - 验证 `15 / 16 / 21 / 22 / 54 / 55 / 56 / 57 / 58` 是否能稳定落到最小任务层
  - 验证“现实承压先于解释”“主角不是救世主而是风险变量”的任务化表达是否成立
  - 验证最小 `Journal / Codex` 样例不会越权吞掉蓝图层

### 2.2 当前明确不进入的范围

- `CH03` 及以后内容
- 支线正文
- 大规模 `MQ / SQ / Journal / Codex / System feedback`
- 旧 `game_narrative_* / game_batch* / game_phase* / final_acceptance*`

## 3. 上游继承边界

本试点只能继承以下已经落库的上游结论：

- `15_bailuyuan_region_bible.md`
  - 鹿门集、黑砂外缘、配额、护撤、延后名单、人命账
- `16_book1_binding_boundary.md`
  - 前段只允许先写“活着很贵”的现实压力
  - 不得提前讲满白鹿之灵、古战前史、远端主世界或终局未来
- `21_party_story_function_bible.md`
  - 沈行策、柳照禾、顾砺衡、桑沉岫承担前段主接口

  - 前段只抬 `沈 / 柳 / 顾 / 桑`，祁渡川仅在 `CH04` 后进入主轴
- `54 / 55 / 56 / 57 / 58`
  - `CH01–CH02` 的章场承压链、揭示顺序、角色推进与世界状态锁定

## 4. 最小任务 Schema

| 字段 | 当前试点要求 | 当前禁止 |
| --- | --- | --- |
| `quest_id` | 必须稳定，且只服务 `CH01–CH02` | 不得沿用旧 batch 编号体系 |
| `chapter_window` | 必须明确写出 `CH01` 或 `CH02` | 不得跨到 `CH03+` |
| `start_state` | 必须回指 `58` 已锁死的章首现实状态 | 不得凭空开场 |
| `objective_chain` | 每个目标都必须先服务现实承压，再服务信息释放 | 不得先讲设定、后补任务 |
| `allowed_reveals` | 只能落到 `56` 允许的前段信息 | 不得提前讲满白鹿、古战、法宝或终局 |
| `journal_hooks` | 只能记录玩家当前知道、且会改变行动判断的信息 | 不得把未公开真相提前写成公共记录 |
| `codex_hooks` | 只能写样本级、现象级、前段可知级条目 | 不得写宇宙级总说明或后段答案 |
| `forbidden_escalation` | 必须写明“不能救世、不能总解释、不能无代价回返” | 不得省略禁止项 |

## 5. 最小主线试点

### 5.1 试点任务链

| 任务节点 | 章节窗口 | 任务目标 | 任务完成条件 | 只允许释放的信息 | 不得越权释放什么 |
| --- | --- | --- | --- | --- | --- |
| `MQ-ACT1-01A《裂风带活下来》` | `CH01-S01–S03` | 让玩家先理解“这里不是新手村，而是会主动追人的外缘” | 完成坠落求生、跟随桑沉岫撤离、见证第一次异常回返 | 白鹿原外缘仍有人活着；主角会异常回返；桑沉岫代表第一层身体代价 | 不得解释回返原因、白鹿高位性质或主角终局身份 |
| `MQ-ACT1-01B《被押送的人》` | `CH01-S04` | 把主角从“幸存者”压成“待审风险人” | 完成巡守队押送、触发回引印被识别、进入鹿门集 | 白鹿原第一反应是风险管理而不是欢迎 | 不得让任何人把主角当救世主 |
| `MQ-ACT1-02A《鹿门的价钱》` | `CH02-S01–S03` | 让玩家先看见病床、配给、工分和安置排序如何决定今天谁先活 | 完成医棚协助、账本分账、劳作换取临时停留 | 鹿门集靠延后名单与配额苟活；柳照禾是地方现实锚点 | 不得把共保会写成纯温情组织 |
| `MQ-ACT1-02B《移交前夜》` | `CH02-S04` | 把主角推进守衡司结构审查线 | 完成议棚对峙、确认本地陪同移交、锁定进入 `CH03` 的结构审查前状态 | 主角是结构异常变量；柳照禾争取到最小地方看护权 | 不得提前讲守衡司全貌、白鹿之灵解释权或后段未来比较 |

### 5.2 任务链硬规则

1. 任务目标必须先落在“活下来、被收下、被登记、被分账、被移交”这些现实动作上。
2. 任何对白、条目或说明只要不改变玩家当前行动判断，就不应进入本试点。
3. 这条最小主线的终点不是“理解世界”，而是“被白鹿原纳入风险登记并被迫进入结构审查线”。

## 6. 最小 Journal 样例

### 6.1 `Journal / Active`

**条目名**：`《坠落不死》`  
**当前目标**：
- 跟着外缘猎手活下来
- 搞清楚为什么自己死后会被拉回来
- 在鹿门集证明自己不是会立刻害死所有人的麻烦

**当前认知**：
- 白鹿原不是死地，这里还有很多人靠配额和护撤活着
- 自己的异常回返已经被人看见
- 被留下，不代表被接纳

### 6.2 `Journal / Update`

**更新条件**：进入医棚与账本室后  
**更新文本**：

> 鹿门集没有多余的病床，也没有多余的粮药。  
> 我被留下，不是因为他们相信我，而是因为把我直接丢出去可能更危险。  
> 这里每一个被留下的人，都是从别人的配额里挤出来的。

## 7. 最小 Codex 样例

### 7.1 条目：`白鹿原`

一处被推离主世界常规秩序带的伤区共同体。  
这里还活着的人，不是因为奇迹，而是因为有人每天继续签字、继续分账、继续决定谁先被延后。

### 7.2 条目：`鹿门集`

白鹿原的主聚落与托底面。  
病床、配给、工分、安置与移交流程，决定了这里“今天还能活着”到底要付什么价。

### 7.3 条目：`回引印`

主角胸口出现的鹿蹄状异常印记。  
当前只知道它与主角的异常回返有关，也让白鹿原的巡守者把他视为结构相关者。  
当前不知道它的完整来历。

## 8. 当前明确禁止事项

1. 不得把试点扩成 `CH03+`。
2. 不得把 `Journal / Codex` 写成世界观百科。
3. 不得让任务文本反向改写 `15 / 16 / 21 / 22 / 54 / 55 / 56 / 57 / 58`。
4. 不得恢复旧 `game_narrative_* / game_batch* / game_phase* / final_acceptance*`。
5. 不得把主角写成前段救世主、白鹿之灵写成总解释器、回返写成无代价复活。

## 9. 一句话口径

> 这个最小试点不是为了“开始大量写任务”，而是为了验证：`CH01–CH02` 的前段现实承压，能不能被稳定翻译成一条不越权的可玩主线。
