# 正典规则

## 1. 文档用途

本文档用于定义哪些内容可视为本仓库的正式正典，以及哪些内容必须经过确认后才能写入。

## 2. 属于正典的内容

以下内容在写入正式文档后，可视为正典：

- 已在对应模块正式文档中明确拍板，且不与参考规范冲突的内容。
- 已写入 `glossary.md` 的正式术语定义。
- 已写入 `decision_log.md` 且状态为“已确认”的重大决策。
- 已由当前维护流程明确接受，并同步更新索引与交叉引用的正式内容。

## 3. 必须经过确认才能写入的内容

以下内容在确认前不得写入为正式设定：

- 新的核心底层术语。
- 对既有核心术语定义的重大改写。
- 新的世界级结构、世界层级关系或底层机制。
- 新的关键势力、角色、地名、小世界名、历史事件名。
- 会影响多个模块的制度、路径、历史前提与剧情前提。
- 会改变主世界、飞升、秩序、规则等核心概念边界的内容。

## 4. 非正典内容

以下内容默认不属于正典：

- 讨论中的提案。
- 未确认草稿。
- 会话中的临时想法。
- 没有记录确认状态的补充内容。
- 与参考文件冲突但尚未解决的文本。

## 5. 写入规则

1. 先确认边界，再写入模块文档。
2. 涉及核心设定时，先更新参考规范，再写正文。
3. 涉及重大变更时，必须补记决策日志。
4. 未确认内容必须显式标注为“待定”或“待人工拍板”。

## 6. 冲突处理

当不同文档出现冲突时：

1. 先以本文件定义的正典边界为准。
2. 对术语冲突，回查 `glossary.md`。
3. 对变更来源，回查 `decision_log.md`。
4. 冲突未解决前，不继续扩写受影响模块。
