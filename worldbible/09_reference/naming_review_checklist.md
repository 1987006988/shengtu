# 命名审核清单

## 1. 文件用途

本文档用于在新增正式命名时提供一份可重复执行的审核清单。

它服务于 `worldbible/09_reference/naming_rules.md`，负责把“命名原则”转成“命名前必须过的检查项”。

## 2. 使用方式

- 先判断本次命名属于哪一类：势力、地域、人物、世界、术语，或其他待归类对象。
- 再逐项核查本清单。
- 若任一关键项答案为“否”或“存疑”，应暂停写入正典，并回查对应护栏文件。

## 3. 基础核查

1. 这个名称是否已经明确所属层级。
2. 这个名称是否已经明确所属类别。
3. 这个名称是否只是工作代号、功能代号或会话占位词。
4. 若不是正式命名，现在是否真的到了必须拍板的时点。

## 4. 术语与正典核查

1. 是否与 `glossary.md` 中既有术语重名、近义冲突或容易混淆。
2. 是否与 `canon_rules.md` 所规定的“需先确认内容”发生越界。
3. 是否会引入新的底层术语、世界级结构或关键历史前提。
4. 是否需要同步登记 `decision_log.md`。

## 5. 层级与尺度核查

1. 是否与所属层级相符。
2. 是否会让局部对象听起来像更高一层对象。
3. 是否会把第一部舞台写得比宇宙总仓库更大。
4. 是否与当前故事尺度相符。
5. 是否会让 `古裂残天` 或隔绝地域反向吞没宇宙级命名系统。

## 6. 风格与气质核查

1. 是否符合中式玄幻、高位秩序、上升感、裂伤感、历史沉积感。
2. 是否过于网文自动生成感。
3. 是否过于现代口语、网络梗或混入英文 / 拼音。
4. 是否过于西幻、日式、科幻或现代都市风。
5. 是否只是为了酷炫而牺牲辨识度。

## 7. 类别边界核查

1. 是否与角色 / 势力 / 地域 / 世界类别混层。
2. 是否把术语名写成剧情名或地方名。
3. 是否把地方名写成势力名。
4. 是否把势力名写成人物尊号。
5. 是否把世界名写成法宝名或组织名。

## 8. 语言形态核查

1. 长度是否与同层级名称大体均衡。
2. 是否过长、过满、过多限定语。
3. 是否过度使用万能字，如 `神`、`圣`、`帝`、`尊`、`皇`、`天`、`道`、`玄`。
4. 是否过度依赖生僻字制造古意。
5. 是否读写成本过高，不利于后续高频引用。

## 9. 功能性核查

1. 是否方便长期扩写与复用。
2. 是否便于在状态页、索引、剧情结构文档和交接文档中稳定引用。
3. 是否会让后续相关对象难以命名，因为它已经把同层语义空间抢得太满。
4. 是否需要同时准备简称、全称、俗称或外界称呼。

## 10. 处理建议

- 若 1-2 项存在轻微问题：先在草稿层调整，不写入正典。
- 若 3 项以上存在问题：退回重命。
- 若涉及术语、世界级对象、主世界级对象或跨模块命名：优先人工拍板后再写入正式文档。
