# 命名规范

## 1. 文件用途

本文件用于定义 `升途` / `Ascendant Path` / `shengtu` 仓库的正式命名规则。

它属于项目级护栏 / 执行规范，不属于世界观正典正文；其作用是约束后续如何为势力、地域、人物、世界与术语命名，而不是直接生成大量正式专名。

## 2. 使用边界

- 本文件只规定“如何命名”，不负责拍板具体名称。
- 本文件不能替代 `glossary.md`、`canon_rules.md` 与 `decision_log.md`。
- 新增正式命名时，应先检查本文件，再回查术语表、正典规则与相关模块文档。
- 若某个命名会影响多个模块、改变既有层级口径，必须同步登记 `decision_log.md`。
- 若某个命名仍在讨论中，应显式标注为“待定”或“待人工拍板”，不得因名称看起来完整就默认视为正典。

## 3. 总体命名原则

### 3.1 世界观整体命名气质

本项目的正式命名应优先体现以下气质：

- 中式玄幻的文化质感，而不是西幻、日式奇幻或科幻词汇系统。
- 高位秩序感，而不是纯热血口号感。
- 上升感、位格差与世界层级感，而不是单纯“大”“强”“无敌”。
- 裂伤感、残存感与修复期历史压力，而不是纯盛世仙境或纯废土末世。
- 历史沉积感与传承感，而不是即时生成、只为眼前剧情服务的即兴名号。

### 3.2 层级辨识优先原则

- 名称首先要服务于层级辨识，其次才是华丽感。
- 项目总名、宇宙层名、主世界名、局部舞台名、地域名、势力名、人物名不得混层。
- 第一部标题与当前舞台名称不得反向代替宇宙总名，也不得写成比宇宙总名更像总标题。
- `古裂残天` 属于故事主世界层命名；隔绝地域属于第一部当前舞台层命名；二者都不得反向定义九主世界或宇宙总层命名风格。

### 3.3 长度与复杂度原则

- 层级越高，名称越应稳、凝、可复用，不宜拖长。
- 层级越低，名称可以适度增加地方性、行业性与生存现实感，但仍不得变成说明书。
- 同一层级的名称长度应大体均衡，避免某个名称明显比同层其他名称长一倍、复杂一倍或花哨一倍。
- 正式名称优先控制在 2-4 个汉字；若需更长，应有明确层级理由，如称号、全称或制度名。
- 附加限定语应尽量作为描述使用，而不是一律塞进正式名称本体。

### 3.4 语义聚焦原则

- 名称应优先表达结构、位格、历史或气质中的一个核心点，不应试图把所有信息一次塞满。
- 若一个名字同时承担世界属性、阵营立场、剧情暗示、功能说明与情绪口号，通常说明命名过载。
- 名称不应只追求“好听”，还应便于后续扩写、比较、引用与归类。

### 3.5 风格一致原则

- 同一层级命名应共享一定的语义重心与音节密度，避免有的极古雅、有的像游戏系统词、有的像现代网名。
- 同类对象可允许有风格差，但差异应来自世界内在结构差异，而不是作者随机换风格。
- 若一个新名称放进现有目录后显得像“别的项目来的词”，应优先视为不合格。

## 4. 层级命名长度建议

| 第一部标题 | 2+4 或 2+3 结构 | 可带副标题 | 不得压过项目总名 |

| 更高层界域名 | 2-4 字 | 稀少、抽象、克制 | 只适合少量传闻式命名 |

## 5. 类别化命名规则

### 5.1 势力命名规则

#### 5.1.1 层级区分

- 宇宙级势力命名应体现高位秩序、古老法统、统摄能力或长期结构位置，不应像地方帮会升级版。
- 主世界级势力命名应体现主世界内部中心性、制度性、法统性或长期历史负担。
- 地域级势力命名应体现与某片区域、入口带、群陆、外海、裂伤带或地方治理的绑定关系。
- 地方级势力命名应体现现实生存、地缘依托、行业属性、地方共同体或小规模秩序维护功能。
- 灰线级势力命名应体现潜伏、渗透、残余、断裂、旧网或隐流感，但不得写成低幼的“暗黑组织名”。

#### 5.1.2 适合使用的字词风格

- 高位正式势力宜偏结构词、制度词、承认词、秩序词。
- 主世界中心势力宜偏 `庭`、`阙`、`府`、`司`、`宗`、`门`、`宫`、`朝`、`殿`、`台` 一类，可按具体制度气质使用。
- 地域治理或地方共同体宜偏 `盟`、`会`、`社`、`坊`、`寨`、`坞`、`镇`、`关`、`营`、`行` 等更贴近现实承接层的字。
- 灰线、残余、地下网络宜偏“旧网、隐脉、残部、潜流、余系”这一类关系感和历史感表达，而不是纯威慑词。

#### 5.1.3 不适合使用的字词风格

- 不宜大量堆叠“神、圣、帝、尊、皇、天、道、玄”等万能字来制造虚高感。
- 不宜使用明显像现代企业、公司、学院社团、互联网组织的词气。
- 不宜使用强烈西幻感、蒸汽感、赛博感、机甲感命名。
- 不宜把势力名写成技能名、装备名、任务名或副本名。
- 不宜直接写成价值判断口号，例如纯“救世”“灭世”“无上”“终极”式命名。

#### 5.1.4 长度建议

- 宇宙级 / 主世界级正式势力名：2-5 字为主。
- 地域级 / 地方级势力名：2-4 字为主，必要时可加一字限定。
- 灰线级势力名：2-4 字为主，宜克制，不宜过长。
- 若需正式全称，可保留简称 / 俗称 / 外界称呼，但不要把全称直接当日常高频称呼。

#### 5.1.5 特殊气质约束

- 守护残余类命名应体现残存正统、旧制留守、艰难维系，而不是绝对正义宣言。
- 反抗残余类命名应体现裂伤时代产物、潜伏汇流或旧敌残脉，而不是现代革命组织口号。
- 地方共同体类命名应体现现实承重层，不宜过分高位化。
- 小宗门 / 小势力类命名应体现具体传承、地方依托或生存逻辑，不应人人都像顶层势力。

#### 5.1.6 冲突回避原则

- 势力名不得与已确认世界名、核心术语名、人物尊号高频重叠。
- 若某个名字已经像世界层名，就不应再拿来做势力名。
- 若某个名字已经像正式术语，就不应改成某家势力专属名。

### 5.2 地域命名规则

#### 5.2.1 总体要求

- 地域命名要首先说明“这是什么层级的地方”，再说明“它有什么气质”。
- 主世界总名应体现高位结构、历史沉积、位格差与世界整体性。
- 局部舞台命名应体现区域命运、空间状态、裂伤结构或功能位置，不应冒充整个世界。

#### 5.2.2 常见字的使用建议

- `界`：更适合世界层、层级边界、结构性空间，不宜在普通地方区滥用。
- `天`：更适合高位世界、上层结构、承天法统或极高位地域；慎用，避免所有高层对象都带 `天`。
- `域`：适合中高层区域、统摄范围、特殊场域。
- `地`：适合泛称或说明性指代，正式命名时通常比 `域`、`界` 更弱。
- `海`、`陆`：适合大地理结构、群陆外海区、宏观空间分区。
- `庭`、`阙`：更适合制度中心、高位建制区、礼制或法统意味强的地点。
- `荒`、`渊`、`墟`：适合裂伤、失序、危险、残留、古战后遗之地，但应区分语义，不宜三者乱换。
- `狱`：只适合极端压迫、封禁、囚禁、镇压语义的高压区，不宜泛化成任何危险区。
- `原`、`州`：更适合可居住、可治理、历史地缘较厚的地方层。
- `城`：只适合聚居中心、节点城市或明确建成秩序区，不宜拿来命名大世界或大区域。

#### 5.2.3 哪些字词适合高位结构

- 更适合主世界总层或高位区域的字：`界`、`天`、`域`、`庭`、`阙`、`海`、`陆`。
- 这类字使用时应控制频率，避免所有主世界、大区、核心区都共享同一批万能字。

#### 5.2.4 哪些字词适合局部舞台

- 更适合地方区、入口带、危险区、资源区或隔绝地域内部节点的字：`谷`、`岭`、`川`、`原`、`泽`、`湾`、`关`、`城`、`坊`、`港`、`口`、`营`、`渊`、`墟`。
- 局部舞台命名可带更多生存现实感、历史损坏感与地方承接感。

#### 5.2.5 慎用与少用

- 高重复网文化字词应慎用，如过度堆叠 `神`、`魔`、`仙`、`圣`、`帝`、`尊`、`狱`、`渊`。
- 过于自动生成感的组合应少用，如纯靠“颜色 / 方位 + 高能字 + 地貌字”机械拼装。
- 纯情绪词、纯审判词、纯毁灭词不宜直接作为大量正式地名。

#### 5.2.6 第一部隔绝地域特殊约束

- 隔绝地域正式名称应体现“被推离”“高位残存”“与外界难直连”“仍被托底”中的部分特征。
- 不得写成比 `古裂残天` 更像主世界总名。
- 不得写成比 `升途` 更像系列总标题。
- 不得写成一听就像任务区域、关卡副本或玩法模式名。

### 5.3 人物命名规则

#### 5.3.1 正式人物名

- 正式人物名优先 2-3 字。
- 主角团与重要角色的人名应便于记忆、呼叫与反复书写。
- 不宜人人都使用极端生僻字、极端古奥字或过长复名。
- 不宜过于接近现代现实高频姓名，也不宜写成一眼模板化“仙侠网文名”。

#### 5.3.2 尊号 / 道号 / 法号 / 职位称呼 / 民间称呼

- 这些名称应与人物身份、道路来源、社会位置和他人视角有关，而不是随机升级版本名。
- 尊号 / 道号 / 法号宜控制在 2-4 字，重点体现修行道路、法统归属、社会承认或历史经历。
- 职位称呼应像制度位置，而不是传奇称号。
- 民间称呼应体现地方感、口耳相传感或简化倾向，不宜比正式称号更华丽。
- 同一角色可以拥有多个称呼，但必须分清哪个是正式名、哪个是尊号、哪个是俗称、哪个是外号。

#### 5.3.3 主世界本土人物 vs 飞升者人物

- 主世界本土人物命名应更能体现其出身环境、法统传统、地方社会和长期秩序背景。
- 飞升者人物命名可保留小世界来源的地方性或道路偏科痕迹，但不应因此与主世界整体风格完全脱节。
- 若飞升者来自差异很大的小世界模板，其名字差异应主要体现在语义与气质上，而不是突然换成别的语系。

#### 5.3.4 避免同质化

- 不要把所有高位人物都命名成“清、玄、尘、霄、衍、无、虚”一类模板名。
- 不要把所有女性人物都命名得极柔极冷，或把所有男性人物都命名得极硬极威。
- 不要通过堆叠冷僻字来制造人物高贵感。
- 不要把人物名直接写成角色功能说明，例如“执法者”“守门人”“断罪人”这类只能做代称的命名。

### 5.4 世界命名规则

#### 5.4.1 小世界命名

- 小世界名可更偏意象、地方性、道路偏科或文明气质。
- 可以比主世界名更灵活，但仍需避免太轻飘、太网文、太像副本名。
- 小世界名不宜过于庞大宏阔，否则会与主世界层抢位格。

#### 5.4.2 主世界命名

- 主世界名应更偏结构、历史、裂伤、法统或高位秩序感。
- 应给人“这是一个可以统摄世界群的高位世界”的感受。
- 主世界名不宜过分地方化、村镇化、器物化或人物化。

#### 5.4.3 更高层界域命名

- 更高层界域命名应保持稀少、抽象、传闻化。
- 不宜过早具体化、地表化、地图化。
- 命名应更像终极方向、古老称谓或不可轻证之名，而不是常规冒险地点。

#### 5.4.4 边界要求

- 世界名不应过于接近势力名、人名、法宝名。
- 若一个名称更像宗门、法宝或人物尊号，通常不适合作为世界名。
- 同层世界名应避免语义完全重复，例如全都围绕“天、玄、圣、神”轮换堆叠。

### 5.5 术语命名规则

#### 5.5.1 新增门槛

- 底层术语不得轻易新增。
- 新术语只有在现有术语无法承载、且确有跨模块稳定需求时，才可进入正式讨论。
- 新术语写入前必须同步核对 `glossary.md`、`canon_rules.md` 与 `decision_log.md`。

#### 5.5.2 命名要求

- 术语名应尽量短、稳、边界清楚。
- 术语要优先支持定义、引用和长期复用，而不是追求神秘感。
- 术语名不得与剧情名、地域名、人物名、势力名高频冲突。

#### 5.5.3 近义滥造限制

- 对 `规则`、`法`、`锚点`、`飞升`、`接引`、`位格`、`世界群`、`灰色小世界`、`隔绝地域`、`残秩序`、`修复期` 等已确认术语，不得为“更酷”而频繁制造近义替代词。
- 若只是表述润色，不应把同义表述升级成新术语。
- 若某个新词会让人误以为它与已确认术语平级或不同义，应优先禁用。

## 6. 禁用规则

### 6.1 来源既视感禁用

- 禁止直接套用既视感过强的成熟玄幻 IP 词组。
- 禁止只换一个字就沿用知名作品的核心命名结构。

### 6.2 说明书化禁用

- 禁止把正式名称直接写成职业、技能、装备、任务说明或系统条目。
- 禁止把“功能说明”伪装成名称本体。

### 6.3 同层堆字禁用

- 禁止同层级名称都反复使用同一批万能字。
- 禁止所有高层名字都靠 `神、圣、帝、尊、皇、天、道、玄` 堆叠。

### 6.4 语体混杂禁用

- 禁止现代口语、网络梗、拼音缩写、英文夹杂式正式命名。
- 禁止明显西幻、日式、科幻或现代都市感名称直接混入正式命名系统。

### 6.5 尺度误导禁用

- 禁止为了酷炫牺牲层级辨识度。
- 禁止第一部舞台名称听起来比宇宙总名、主世界总名还像总标题。
- 禁止局部舞台名听起来像统摄九主世界的终极结构名。

### 6.6 过度模板化禁用

- 禁止名称过于像网文自动生成词。
- 禁止大量使用“无上、终极、永恒、灭世、诸天、万古”等纯拉满词汇而无结构区分。
- 禁止靠堆砌生僻字制造深度假象。

## 7. 命名审核流程

新增正式命名时，建议按以下顺序执行：

1. 先判断该名称属于哪个层级、哪个类别。
2. 回查本文件对应类别规则。
3. 回查 `glossary.md`，确认不与既有术语冲突。
4. 回查 `canon_rules.md`，确认该名称是否已触及必须先确认的范围。
5. 若影响跨模块或长期口径，登记 `decision_log.md`。
6. 再决定是否写入正式模块文档。

## 8. 审核清单

正式审核清单已拆分至：

- `worldbible/09_reference/naming_review_checklist.md`

在未单独打开清单时，至少应先确认以下九项：

1. 是否与既有术语冲突。
2. 是否与所属层级相符。
3. 是否与世界气质相符。
4. 是否会把第一部写得比宇宙总仓库更大。
5. 是否与当前故事尺度相符。
6. 是否与角色 / 势力 / 地域 / 世界类别混层。
7. 是否过于网文自动生成感。
8. 是否过于说明书 / 游戏系统词。
9. 是否利于长期扩写与复用。

### 9.1 推荐字词池

以下字词可作为方向参考，不是专名库，也不是组合模板：

- 高位结构向：`界`、`域`、`天`、`庭`、`阙`、`府`、`台`、`海`、`陆`
- 裂伤 / 修复向：`裂`、`残`、`断`、`缺`、`墟`、`渊`、`遗`、`封`
- 历史 / 法统向：`旧`、`古`、`承`、`统`、`序`、`法`、`册`、`脉`
- 地方 / 生存向：`原`、`州`、`川`、`岭`、`泽`、`关`、`城`、`坊`
- 灰线 / 潜流向：`隐`、`潜`、`余`、`脉`、`流`、`网`、`影`

### 9.2 慎用字词池

以下字词不是绝对禁用，但若高频滥用，极易造成风格漂移或层级失真：

- 万能拉高字：`神`、`圣`、`帝`、`尊`、`皇`、`道`、`玄`
- 纯情绪拉满字：`灭`、`绝`、`终`、`劫`、`罚`
- 过度网文化字：`魔`、`神`、`狱`、`渊` 的机械叠用
- 现代系统化字：`模块`、`终端`、`协议`、`系统`
