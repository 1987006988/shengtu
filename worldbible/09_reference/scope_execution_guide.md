# 分层执行指南

## 1. 文档用途

本文档用于把“三层分层”从说明口径压成可执行护栏，帮助后续协作者在写作、修桥、快审和进入更细层开发前，先判断一条内容究竟属于哪一层。

本文件不是世界观正典正文，而是执行指南文件。

## 2. 三层分层的执行定义

### 2.1 宇宙长期正典

指适用于整个宇宙、多个主世界或底层通则的长期内容，例如：

- 三层宇宙结构
- `3000` 小世界与 `9` 主世界框架
- `气 / 规则 / 法 / 锚点`
- 九主世界比较框架

这类内容一旦成立，应优先写入宇宙总层与参考层文件，不得由局部故事舞台反写。

### 2.2 `古裂残天` 长期正典

指只属于故事主世界 `古裂残天` 总层、但不等于其他主世界共性的长期内容，例如：

- 古老残缺修复型主世界定位
- 双极旧秩序与残胜格局
- 大势力主导型格局
- 四级社会模型

这类内容应优先落在 `04_main_world`、`05_history`、`06_society` 的故事主世界总层文件中。

### 2.3 第一部 / 当前故事强绑定设定

指只服务当前第一部舞台、主角团当前尺度与主线结构的绑定内容，例如：

- 隔绝地域舞台
- 主角团当前只能决定隔绝地域命运
- 接引法宝是当前主线核心装置
- 当前主线围绕地域命运与代价承担展开

这类内容应优先落在 `07_characters`、`08_story` 与相关桥梁说明中，并保留“当前舞台 / 当前尺度”提示。

## 3. 后续写作时如何判断属于哪一层

可按以下顺序判断：

1. 这条内容是否对整个宇宙、多个主世界或底层通则都成立。
2. 这条内容是否只对 `古裂残天` 主世界总层成立。
3. 这条内容是否只对第一部当前舞台、主角团当前尺度或接引法宝主线成立。
4. 若仍不确定，优先按更低层处理，并标注“待确认”，不要直接上浮。

## 4. 发现混层时怎么处理

- 若第一部强绑定设定被写成 `古裂残天` 长期正典：先补提示语，再把具体绑定内容收回 `07_characters`、`08_story` 或桥梁说明。
- 若 `古裂残天` 长期正典被写成宇宙通则：先回查 `05_scope_map.md` 与 `first_arc_vs_universe_scope.md`，再把内容收回主世界层。
- 若宇宙长期正典被局部经验改口：暂停扩写，优先回查 `glossary.md`、`canon_rules.md` 与 `decision_log.md`。
- 若同一文件必须同时提及多层内容，应显式写明“哪一段属于哪一层”，不要让默认语气替代层级判断。

## 5. 最容易误上浮的内容类型

- 隔绝地域舞台设定被误写成整个 `古裂残天` 常态。
- 接引法宝、复生机制被误写成主世界普遍结构。
- 主角团当前能力尺度被误写成能影响整个主世界命运。
- `古裂残天` 的残胜格局、四级社会模型被误写成其他主世界通则。
- 角色接口、故事骨架、章节束结构被误当作长期正典。

## 6. 进入更细层开发前如何自检

在继续下沉到更细层开发前，至少先问：

1. 这条内容属于宇宙长期正典、`古裂残天` 长期正典，还是第一部强绑定设定。
2. 这条内容是否已经写在正确层级的文件里。
3. 这条内容是否因为高频出现而被误当作更高一层常态。
4. 这条内容是否需要在文件开头或使用边界处加一句最小提示语。
5. 若继续细化，是否会把第一部舞台重新膨胀回整个主世界。

若以上任一问题答案不清楚，应先补桥、补提示或回查参考层，而不是直接进入更细层开发。
