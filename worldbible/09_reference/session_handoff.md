# 会话交接（重构后）

## 1. 你现在接手的项目到底是什么状态

你接手的不是一个“已经完成首轮大型 CRPG 叙事资产并通过最终验收、只需继续扩容”的项目。  
你接手的是一个：

- 上层世界观基础可用；
- 古裂残天方向正确；
- 白鹿原尺度收束正确；
- 但中下层工程顺序失真；
- 项目状态页和交接页曾长期失真；
- `08_story` 与 `09_reference` 出现过早任务化 / 文本化 / 批次化膨胀；

的**重构中项目**。

## 2. 旧结论里哪些别再沿用

以下旧结论不得再默认沿用：

- `END 已完成，所以项目当前已在扩容 / 生产期`
- `B00–B15 与 END 已完成，所以可直接继续 MQ / SQ / Journal / Codex / System feedback`
- `第一部已具备大型 CRPG 叙事底盘`
- `当前 story 层已经足够稳定，不必再补白鹿原承载层`
- `大量文本包 = 大量可用真实资产`

这些结论只说明：  
过去做过一轮高强度 narrative lab。  
它们**不代表当前真实阶段**。

## 3. 当前必须遵守的新结论

你必须遵守以下新结论：

1. 项目当前真实阶段是：`主世界承载层与第一部施工蓝图重建阶段`。
2. 白鹿原仍需从功能地图升级为地区圣经。
3. 古裂残天仍需从概念主世界升级为长期承重主世界。
4. 七席位角色需重新锁定各自接口与权重曲线。
5. 第一部必须先完成卷 / 幕 / 章 / 场 / 揭示 / 角色 / 世界状态的蓝图闭环。
6. 在上述工作完成前，不默认进入任务层。
7. 旧批次文本全部视为历史资产，不视为当前主入口真相。

默认只允许做以下工作：

- 看见 `game_narrative_*`、`game_batch*`、`game_phase*`、`final_acceptance` 相关文件时：
  - 一律先当**历史记录**而不是当前状态；

  - 可以回收具体片段、术语、局部桥段；
  - 不能再把它们当成第一部主控蓝图本身。
