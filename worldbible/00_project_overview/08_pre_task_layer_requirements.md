# 08 进入任务层前置条件清单

## 1. 文档用途

本文档用于明确：在默认进入任务层之前，项目至少还需要满足哪些前置条件，以及当前哪些条件已满足、哪些仍未满足。

本文件是进入任务层前的执行清单，不承载任务设计本身。

## 2. 当前总口径

- 当前仍不默认进入任务层。
- 当前下一步是“白鹿原势力与关键节点落位阶段”。
- 在前置条件未满足前，不写任务设计、对白、关卡、战斗或玩法映射。

## 3. 进入任务层前必须完成的前置条件

### 3.1 项目级护栏稳定

- 项目总纲、范围图、状态页、交接页、阶段门槛表、目录职责表与本清单已建立并可执行。

### 3.2 白鹿原舞台结构稳定

- `白鹿原` 正式地域名、六层结构、三带总括、托底逻辑、长期张力已稳定。
- 接引法宝之灵白鹿显化及功能边界已稳定。

### 3.3 白鹿原势力与关键节点完成落位

- 白鹿原内部主要力量关系已完成第一轮落位。
- 关键结构节点已不再只是占位词，而具备明确承重功能。

### 3.4 主角团接口完成落位

- 主角团七席位已与白鹿原势力、关键节点、结构层形成稳定接口。
- 角色接口不会在任务设计时临时补洞。

### 3.5 第一卷主线设计稳定

- 第一卷主线结构已能稳定承接舞台、势力、节点与角色接口。
- 主线推进顺序、揭示顺序与命题压力已可解释。

### 3.6 章节大纲稳定

- 至少已有可用的章节大纲或等价章节级结构承接。
- 每章的结构目标、冲突目标与揭示目标已清楚。

### 3.7 任务层边界清楚

- 已明确哪些内容是任务层输出，哪些内容仍属于结构层。
- 不会把章节束、节点骨架直接误当任务清单。

## 4. 哪些条件现在已经满足

- 项目级护栏稳定：已基本满足。
- 白鹿原舞台结构稳定：已满足。
- 命名、尺度、接引法宝之灵边界：已满足。
- 当前阶段不是任务层：已明确成立。

## 5. 哪些条件现在还没满足

- 白鹿原势力与关键节点完成落位：未满足。
- 主角团接口完成落位：未满足。
- 第一卷主线设计稳定：未满足。
- 章节大纲稳定：未满足。
- 任务层边界进一步细化：未满足。

## 6. 为什么现在不能直接进入任务层

- 白鹿原内部主要势力与关键节点还未完成第一轮功能落位。
- 主角团接口尚未和势力、节点形成稳定闭环。
- 当前 `08_story` 只有结构骨架与章节束，尚未到可以安全映射任务层的程度。
- 若现在直接进入任务层，极容易让任务设计反向定义势力结构、角色接口与白鹿原内部秩序。
- 任务层需要的“谁在推动、在哪里发生、为什么现在发生”仍缺少上游结构承重。

## 7. 后续什么时候才适合谈任务层

至少要等到以下条件同时成立后，才适合正式讨论任务层：

- 白鹿原势力与关键节点落位完成。
- 主角团接口落位完成。
- 第一卷主线设计完成。
- 章节大纲已可稳定承接结构推进。
- 状态页与交接页明确允许下沉。

## 8. 当前阶段结论

- 当前仍不默认进入任务层。
- 当前真正该做的是：进入“白鹿原势力与关键节点落位阶段”。
- 在此之前，任何任务、对白、关卡、战斗或玩法设计都应视为越界。
