# 06 阶段门槛表

## 1. 文档用途

本文档用于把项目当前推荐开发顺序、各阶段输入前提、允许产物、禁止越界项与进入下一阶段的门槛压成可执行表。

它是项目级执行护栏，不属于世界观正典正文，也不用于偷渡具体剧情、任务或对白。

## 2. 当前推荐开发阶段顺序

当前推荐按以下顺序推进：

1. 目录治理 / 阶段门槛
2. 白鹿原势力与关键节点落位
3. 主角团接口落位
4. 第一卷主线设计
5. 章节大纲
6. 任务层 / 玩法映射（如以后需要）

若上游阶段未稳定，不建议跳级进入下游阶段。

## 3. 各阶段门槛

### 3.1 目录治理 / 阶段门槛

#### 输入前提

- 项目总纲、范围地图、状态页与交接页可读。
- 三层执行分层已存在基础护栏。
- 当前阶段目标已明确不是任务层。

#### 允许产物

- 阶段门槛文件
- 目录职责文件
- 进入任务层前置条件清单
- 最小必要的入口文件同步

#### 禁止越界项

- 不扩写新宇宙通则。
- 不扩写角色传记。
- 不写正文、任务、对白、关卡、战斗或玩法。
- 不借治理文件偷渡新势力、新节点、新剧情。

#### 进入下一阶段条件

- 阶段顺序、目录职责与任务层前置条件已整理完毕。
- 项目总入口、状态页与交接页已同步反映当前门槛。
- 当前协作者能明确回答“下一步为什么是势力与关键节点落位，而不是任务层”。

### 3.2 白鹿原势力与关键节点落位

#### 输入前提

- `白鹿原` 的舞台结构、托底逻辑、张力图已稳定。
- 接引法宝之灵白鹿显化与功能边界已稳定。
- 当前仍不默认进入任务层的口径已被总入口文件压实。

#### 允许产物

- 白鹿原内部主要势力类别、功能位置与层级关系
- 白鹿原关键结构节点的功能落位
- 势力与节点之间的最小桥梁说明
- 仍停留在设定层 / 承载层的结构文件

#### 禁止越界项

- 不写详细阵营剧情。
- 不写任务清单、关卡流程、对白草稿。
- 不把节点文件写成地图打卡表或副本列表。
- 不让白鹿原势力结构反向改写宇宙总层或 `古裂残天` 总层。

#### 进入下一阶段条件

- 白鹿原内部主要力量已能按“谁托底、谁控结构、谁改变量、谁管地方、谁卡节点”稳定说明。
- 关键节点已具备结构功能，而不只是名称占位。
- 势力与节点之间的承接关系已足以支撑主角团接口落位。

### 3.3 主角团接口落位

#### 输入前提

- 白鹿原势力与关键节点已完成第一轮落位。
- `07_characters` 当前仍保持角色原型 / 概念卡深度，未滑入人物小传。
- 主角团七席位与白鹿原结构层之间已存在稳定接口。

#### 允许产物

- 主角团与白鹿原各力量、节点、社会层的接口关系
- 七席位在第一部中的结构职责重新校准
- 角色接口与舞台结构的对照关系

#### 禁止越界项

- 不写完整人物传记。
- 不写角色个人任务线、恋爱线、结局线。
- 不把角色接口文件写成剧情桥段表。

#### 进入下一阶段条件

- 每个关键席位都能回答“进入白鹿原后到底打开哪一层世界问题”。
- 主角团接口与势力 / 节点落位不再打架。
- 角色结构足以支撑第一卷主线设计，而无需靠临时补角色功能。

### 3.4 第一卷主线设计

#### 输入前提

- 白鹿原势力、关键节点、主角团接口三者已形成闭环。
- 第一部主线尺度仍稳定停留在白鹿原。
- 结构性张力与代价问题已能映射到主要力量与节点。

#### 允许产物

- 第一卷主线结构设计
- 主线冲突推进框架
- 揭示顺序、代价结构与命题推进设计

#### 禁止越界项

- 不直接下沉成详细任务流。
- 不写对白稿、关卡脚本与玩法说明。
- 不把章节设计提前写成逐节点任务说明书。

#### 进入下一阶段条件

- 第一卷主线已有稳定的起承转合与结构承重。
- 主线推进已能解释主要势力、节点与角色接口为何依次上桌。
- 仍能在不写任务层的情况下讲清楚主线如何成立。

### 3.5 章节大纲

#### 输入前提

- 第一卷主线设计已稳定。
- 势力、节点、角色接口、结构张力都已能被章节承接。
- 当前章节拆分不会改变主线尺度或命题边界。

#### 允许产物

- 章节大纲
- 章节间推进关系
- 每章的结构目标、冲突目标与揭示目标

#### 禁止越界项

- 不写详细任务步骤。
- 不写对白、演出脚本、战斗编排。
- 不把章节大纲伪装成可直接执行的任务流。

#### 进入下一阶段条件

- 章节大纲已经稳定到可映射任务层。
- 每章都能明确回答“这一章承接什么结构问题”，而不只是“去哪里做什么”。
- 章节拆分与主线设计、角色接口、势力节点落位保持一致。

### 3.6 任务层 / 玩法映射（如以后需要）

#### 输入前提

- 前述阶段均已完成并通过自检。
- `08_pre_task_layer_requirements.md` 中的前置条件已满足。
- 项目状态页明确允许下沉。

#### 允许产物

- 任务层映射
- 玩法承接关系
- 任务与章节之间的结构对照

#### 禁止越界项

- 不得绕过章节大纲直接生成碎片化任务。
- 不得用玩法需求反向改写世界观底层规则。
- 不得在前置条件未满足时偷跑任务设计。

#### 进入下一阶段条件

- 当前阶段已经是最细层开发，不再由本文件继续往下拆。

## 4. 当前阶段结论

- 当前轮次应停在“目录治理 / 阶段门槛”收口。
- 下一步是“白鹿原势力与关键节点落位阶段”。
- 当前仍不默认进入任务层。

## 5. 使用方式

- 每次准备切换阶段前，先回查本文件。
- 若某一文件产出看起来已经像下游阶段内容，应先按本文件判断是否越界。
- 若想进入任务层，必须同时回查 `04_current_state.md` 与 `08_pre_task_layer_requirements.md`。
