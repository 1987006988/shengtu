# 06 阶段门槛表

## 1. 文档用途

本文档用于把项目当前推荐开发顺序、阶段输入前提、允许产物、禁止越界项与阶段切换门槛压成可执行表。

它是项目级执行护栏，不属于世界观正典正文，也不用于偷渡具体剧情、任务、对白或文本包产线。

## 3. 各阶段准入与禁止

### 3.1 控制层同步

- 准入条件：
  - 控制层文件落后于仓库现实；
  - `50–58` 已落库，但状态页、交接页、执行状态页或护栏页仍未同步。
- 允许产物：
  - 控制层改写；
  - run / decision 补记；
  - 阅读入口与冲突规则修正。
- 明确禁止：
  - 借控制层同步之名重开大规模任务包；
  - 借同步之名推翻 `execution_plan_rebuild.md` 已成立的试点口径。

### 3.2 蓝图一致性审计

- 准入条件：
  - 控制层同步已完成当前轮收口；
  - `15 / 16 / 21 / 22 / 51–58` 已可作为被审计对象。
- 允许产物：
  - 跨文件冲突清单；
  - 最小必要修订文件清单；
  - 试点准入结论；
  - 禁止事项清单。
- 明确禁止：
  - 默认进入全面任务层；
  - 恢复旧 `game_narrative_* / game_batch* / game_phase* / final_acceptance*`；
  - 直接生成整批 MQ / SQ / Journal / Codex / System feedback。

### 3.3 Act I 任务化试点

- 准入条件：
  - 蓝图一致性审计 `PASS`；
  - 试点范围已锁定；
  - `rebuild_execution_state.md` 已明确试点交付件与禁止事项。
- 允许产物：
  - `Act I` 一条最小主线任务化试点；
  - `CH01–CH02` 级别的最小试点范围定义；
  - 最小 `Journal / Codex` 样例；
  - `1–2` 条支线试点的范围定义。
- 明确禁止：
  - 大规模任务文本工程；
  - 旧式 batch schedule；
  - 让试点反向改写 `14 / 15 / 16 / 21 / 22 / 50–58` 的上游边界。

### 3.4 试点复盘

- 准入条件：
  - 已有最小试点输出；
  - 已能对照蓝图、护栏与状态页进行复核。
- 允许产物：
  - 试点问题清单；
  - 护栏补丁；
  - 蓝图回修建议；
  - 是否允许扩容的阶段结论。
- 明确禁止：
  - 在未复盘前直接扩大试点规模；
  - 用单次试点成功替代长期门槛。

### 3.5 扩容评估

- 准入条件：
  - 试点复盘通过；
  - 风险、禁区与扩容边界已被正式记录。
- 允许产物：
  - 后续任务层扩容顺序；
  - 新阶段目标文件；
  - 扩容前提与继续禁止项。
- 明确禁止：
  - 把扩容评估等同于自动开放全量生产；
  - 恢复旧 narrative-lab 大产线。
