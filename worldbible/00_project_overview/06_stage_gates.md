本文档用于把项目当前推荐开发顺序、阶段输入前提、允许产物、禁止越界项与阶段切换门槛压成可执行表。

## 3. 各阶段准入与禁止

### 3.1 控制层同步

- 准入条件：
  - 控制层文件落后于仓库现实；
  - `50–58` 已落库，但状态页、交接页、执行状态页或护栏页仍未同步。
- 允许产物：
  - 控制层改写；
  - run / decision 补记；
  - 阅读入口与冲突规则修正。
- 明确禁止：
  - 借控制层同步之名重开大规模任务包；
  - 借同步之名推翻 `execution_plan_rebuild.md` 已成立的试点口径。

### 3.2 蓝图一致性审计

- 准入条件：
  - 控制层同步已完成当前轮收口；
  - `15 / 16 / 21 / 22 / 51–58` 已可作为被审计对象。
- 允许产物：
  - 跨文件冲突清单；
  - 最小必要修订文件清单；
  - 试点准入结论；
  - 禁止事项清单。
- 明确禁止：
  - 默认进入全面任务层；
  - 恢复旧 `game_narrative_* / game_batch* / game_phase* / final_acceptance*`；
  - 直接生成整批 MQ / SQ / Journal / Codex / System feedback。

### 3.3 Act I 任务化试点

- 准入条件：
  - 蓝图一致性审计 `PASS`；
  - 试点范围已锁定；
  - `rebuild_execution_state.md` 已明确试点交付件与禁止事项。
- 允许产物：
  - `Act I` 一条最小主线任务化试点；
  - `CH01–CH02` 级别的最小试点范围定义；
  - 最小 `Journal / Codex` 样例；
- 明确禁止：

  - 旧式 batch schedule；
  - 让试点反向改写 `14 / 15 / 16 / 21 / 22 / 50–58` 的上游边界。

### 3.4 试点复盘

- 准入条件：
  - 已有最小试点输出；
  - 已能对照蓝图、护栏与状态页进行复核。
- 允许产物：
  - 试点问题清单；
  - 护栏补丁；
  - 蓝图回修建议；
  - 是否允许扩容的阶段结论。
- 明确禁止：
  - 在未复盘前直接扩大试点规模；
  - 用单次试点成功替代长期门槛。

### 3.5 极小扩容评估 + 直线正文试点

- 准入条件：
  - 试点复盘通过；
  - 风险、禁区与扩容边界已被正式记录；
  - 已明确“先定义范围、再测一段正文”的最小策略。
- 允许产物：
  - `1–2` 条极小支线候选的范围定义；
  - `CH01-S01–S02` 或最多 `CH01-S01–S04` 的一段极小直线正文样本；
  - 对两者的分开审计与合并阶段结论；
  - 试点术语降噪与护栏补丁。
- 明确禁止：
  - 把支线范围定义直接偷换成支线正文；
  - 把一段正文样本偷换成主线正文已开放；
  - 把本阶段结果等同于自动开放全量生产；
  - 恢复旧 narrative-lab 大产线。

### 3.6 Act I 主线正文开放评估

- 准入条件：
  - 极小支线范围定义审计通过；
  - 直线正文试点审计通过；
  - 两项通过结论已被正式记录。
- 允许产物：
  - 是否开放 `Act I` 主线正文的建议结论；
  - 主线正文仍需继续遵守的窗口、层级与禁止事项；
  - 继续禁止支线正文试点与全面任务层的护栏。
- 明确禁止：
  - 把“允许讨论主线正文”写成“支线正文已开放”；
  - 把“主线正文开放评估”写成“全面任务层已开放”；
  - 跳过主线正文评估，直接恢复全量任务文本生产。
