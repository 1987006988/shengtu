# 03 范围与约束

## 1. 文件用途

本文件用于明确当前阶段允许处理的内容、暂不处理的内容、禁止事项与跨模块约束。

它是项目级执行护栏，负责防止“明明在做修桥或骨架设计，却被写成新设定 / 新剧情 / 新任务”。

## 2. 当前阶段允许处理的内容

当前阶段允许优先处理以下内容：

- `升途` / `Ascendant Path` / `shengtu` / `升途：封绝之地` 的命名统一与入口文件口径维护。
- 已确认正典的结构整理、口径修正与最小必要补充。
- 索引、范围说明、护栏文件、交接文件、状态页、决策日志与术语表维护。
- 跨模块桥梁说明、接口收束、尺度纠偏、一致性修复。
- 已确认设定下的故事结构、角色骨架、社会模型、历史框架等结构级文档。
- 关键损坏文件的恢复与可读性修复。

## 3. 当前阶段暂不处理的内容

当前阶段不默认处理以下内容：

- 新的大型宇宙级设定扩写。
- 具体人物正式传记。
- 具体剧情桥段、详细任务层设计、对白层设计、关卡层设计、玩法层设计。
- 具体地名、势力名、人物名、法宝名、历史事件名的大规模发明。
- 面向实现层的数值、职业、技能、任务流与结局树。

## 4. 输出边界

- 允许输出：结构骨架、范围文件、护栏文件、正式设定文档、修桥记录、审计报告。
- 不允许输出：把结构文档伪装成剧情文案、把角色框架伪装成人物小传、把章节束伪装成任务清单。
- 若必须为后续保留空间，应明确标注“待定”或“待人工拍板”。
- 若写到项目名称，应使用 `升途`；英文正式说明使用 `Ascendant Path`；仓库 / 目录上下文使用 `shengtu`；第一部标题使用 `升途：封绝之地`。

## 5. 禁止事项

- 禁止擅自新增核心世界结构、底层规则、关键历史锚点与主干阵营框架。
- 禁止让 `07_characters` 与 `08_story` 反向改写宇宙通则。
- 禁止把 `古裂残天` 或第一部隔绝地域写成整个仓库的默认尺度。
- 禁止在未补齐桥梁层时，直接推进任务层、对白层、玩法层。
- 禁止把讨论中的提案、模糊记忆或口头总结直接写成正典。

## 6. 跨模块约束

在执行层面，以下三层分层必须始终成立：

- 宇宙长期正典：只要适用于整个宇宙、多个主世界或底层通则，就不得按 `古裂残天` 局部经验回写。
- `古裂残天` 长期正典：只要只属于故事主世界总层，就不得自动外推到其他主世界。
- 第一部强绑定设定：只要只服务当前隔绝地域舞台、主角团当前尺度或接引法宝主线，就不得默认升级为长期正典。

### 6.1 对术语与正典的依赖

- 术语统一以 `glossary.md` 为准。
- 正典优先级以 `canon_rules.md` 定义为准。
- 重大口径变化必须登记 `decision_log.md`。

### 6.2 对 `07_characters` 的特殊约束

- 角色模块属于游戏聚焦层，但必须承接 `01–06`。
- 在当前故事已收束为隔绝地域后，角色接口必须优先落到隔绝地域内部接口，而不是继续默认站在整个主世界尺度。
- 不得把角色席位、角色原型或角色概念卡写成职业槽位、剧情小传或任务引导器。

### 6.3 对 `08_story` 的特殊约束

- 故事模块当前主线舞台是隔绝地域，不是整个 `古裂残天`。
- 主角团只能对隔绝地域命运起决定性作用，不得继续上浮到整个主世界命运。
- 接引法宝必须保持为主线核心装置，不得退化为背景道具。
- 在进入任务层前，必须先保证 premise、四幕、分段、节点与章节束的尺度完全一致。
- 属于第一部强绑定设定的舞台、命题、法宝绑定与代价结构，不得回写成 `古裂残天` 长期正典或宇宙长期正典。

## 7. 哪些变更必须登记

以下变更必须同步登记到 `decision_log.md`：

- 核心口径变化。
- 故事舞台尺度变化。
- 角色来源结构变化。
- 模块方法论变化。
- 任何会影响多个模块的桥梁修复或纠偏。

## 8. 哪些变更必须先确认

以下变更必须先人工确认后才能写入正式文档：

- 新增世界级机制、核心设定或关键装置。
- 放大故事舞台或角色影响范围。
- 改写既有主世界格局、力量体系或历史主轴。
- 让第一部故事强绑定内容升级为宇宙普遍规则。

## 9. 进入任务层前的前置条件

在默认进入任务层前，至少应满足：

- 项目级护栏文件已补齐并可执行。
- 关键正典文件无乱码、可直接核查。
- 角色接口与故事接口已严格收束到当前舞台尺度。
- premise、四幕、节点与章节束口径一致。
- 当前状态页与交接页反映的是最新真实推进点。
