# 05 范围地图

## 1. 文档用途

本文档用于明确 `worldbible/` 各目录分别服务于哪一层范围，防止后续协作中目录用途逐步漂移。

## 2. 仓库的总范围

本仓库是 `升途` / `Ascendant Path` 的宇宙级设定仓库，仓库 / 目录英文名统一为 `shengtu`。

它的覆盖范围包括：

- 整个宇宙的底层结构与通则
- 九主世界与主世界层的比较与展开
- 当前故事主世界 `古裂残天` 的重点细化
- 游戏聚焦层的角色、剧情与叙事内容

必须明确：

- 古裂残天是当前重点展开对象，不是整个仓库的唯一对象。
- 第一部标题为 `升途：封绝之地`，其当前舞台是古裂残天内部的隔绝地域，不等于整个主世界。
- 游戏内容只是宇宙设定中的一个聚焦层，不应反向吞没整个仓库。

## 3. 四层范围划分

### 宇宙通则层

负责全宇宙适用的底层结构、通则、关系与模板。

### 九主世界 / 主世界层

负责主世界共性、九主世界比较、主世界差异与主世界层模型。

### 故事主世界层

负责当前故事主世界 `古裂残天` 的重点展开。

### 游戏聚焦层

负责主角团、重要角色、主线与分支等具体游戏内容。

## 4. 各目录对应范围表

| 目录 | 主要服务层级 | 说明 |
| --- | --- | --- |
| `00_project_overview` | 项目级护栏层 | 负责总纲、范围、状态、协作规范 |
| `01_cosmology` | 宇宙通则层 | 写宇宙结构、飞升、接引、宏观格局，不只服务古裂残天 |
| `02_power_system` | 宇宙通则层 | 写气、规则、法、锚点等底层力量逻辑，不只服务单一主世界 |

| `05_history` | 宇宙历史层 / 主世界历史层 / 故事主世界层 | 先写宏观与主世界关系，再写古裂残天历史 |

## 5. 后续写作时如何判断内容放哪一层

可按以下顺序判断：

1. 这条内容是否对整个宇宙都成立？若是，放宇宙通则层。
2. 这条内容是否描述九主世界或主世界的一般差异？若是，放主世界层。
3. 这条内容是否只属于 `古裂残天`？若是，放故事主世界层。
4. 这条内容是否只服务主角、重要角色、具体主线或玩法叙事？若是，放游戏聚焦层。

## 5.1 三层执行分层对照

在当前执行层中，还应同步判断这条内容属于哪一种“确认层级”：

- 宇宙长期正典：优先落在 `01–06` 与 `09_reference` 的总层文件中。
- `古裂残天` 长期正典：优先落在 `04_main_world`、`05_history`、`06_society` 的故事主世界总层文件中。
- 第一部强绑定设定：优先落在 `07_characters`、`08_story` 及相关桥梁说明中，并显式保留“当前舞台 / 当前尺度”提示。

若一条内容只属于第一部强绑定设定，不得因为写在高频文件里就默认升格为 `古裂残天` 长期正典或宇宙长期正典。

## 6. 常见跑偏风险

- 把 `01_cosmology` 写成只解释古裂残天的宇宙学背景。
- 把 `02_power_system` 写成只为故事主世界或主角服务的力量说明。
- 把 `03_worlds_and_paths` 写成单纯的职业来源仓库。
- 把 `04_main_world` 直接等同于古裂残天专属目录。
- 把 `05_history` 一开始就缩成游戏前史。
- 把 `06_society` 一开始就缩成古裂残天地方社会。
- 让 `07_characters` 或 `08_story` 的局部需求反向改写宇宙通则。

## 7. 本项目当前重点对象与整体范围的关系

当前重点对象是 `古裂残天`。

但它的地位应明确理解为：

- 它是当前阶段最重点展开的故事主世界。
- 它属于整个宇宙级设定仓库中的一个重点对象。
- 后续 `04_main_world`、`05_history`、`06_society` 的推进，都应先处理主世界总层，再处理故事主世界层，避免范围写窄。
