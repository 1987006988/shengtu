# 02 设计原则

## 1. 文件用途

本文件用于记录本仓库在长期建设中必须持续遵守的设计原则、取舍标准与冲突处理方式。

它不承载具体设定结论，而负责说明：当不同模块、不同尺度、不同创作冲动同时出现时，项目应如何稳定判断。

## 2. 长期设计原则

### 2.1 命名稳定原则

- 项目中文总名统一为 `升途`。
- 英文正式名统一为 `Ascendant Path`。
- 仓库 / 目录英文名统一为 `shengtu`。
- 第一部标题统一为 `升途：封绝之地`。
- 若本地物理路径仍显示旧工程名，不得反向改写文档口径；物理目录重命名需人工执行。

### 2.2 宇宙级优先原则

- 本仓库首先是宇宙级设定仓库。
- 任何局部舞台、单一主世界、第一部故事、主角团或当前剧情，都不得反向吞没宇宙总范围。

### 2.3 总层优先原则

- 先建立宇宙总层、主世界总层、历史总层、社会总层，再写故事主世界与游戏聚焦层。
- 若总层尚未稳定，局部展开必须保守，不能倒逼总层改口。

### 2.4 尺度控制原则

- 人物尺度、故事尺度、舞台尺度与力量尺度必须彼此匹配。
- 若角色仍只是高含金量新人，就不能把其主线舞台默写成整个主世界命运。
- 若故事舞台已经收束为局部隔绝地域，就不得再悄悄膨胀回整个主世界。

### 2.5 先修桥再扩写原则

- 一旦发现模块间尺度冲突、术语冲突、因果断裂、文件损坏或桥梁层过薄，应优先修桥、修文件、查决策日志。
- 不得在桥梁没补齐时，继续往下推进任务层、对白层或玩法层。

### 2.6 第一部不绑死宇宙原则

- 第一部故事可以有强绑定装置、强绑定舞台与强绑定人物结构。
- 但这些内容必须被明确识别为“第一部强绑定设定”，不能无声覆盖整个宇宙长期正典。
- 未来其他主世界、小世界与其他故事入口必须仍然可写、可扩、可独立成立。

### 2.7 角色 / 故事不得反写宇宙通则原则

- `07_characters` 与 `08_story` 必须承接 `01–06`，不能反过来改写宇宙通则。
- 若角色或故事需要特殊机制，优先解释为局部条件，而不是直接升级为宇宙普遍规则。

### 2.8 接引法宝 / 隔绝地域使用原则

- 接引法宝、隔绝地域、复生机制等当前故事关键装置可以作为第一部主线核心。
- 但它们的使用必须服务于“局部舞台成立”，而不是扩张为整个宇宙都依赖同一装置。
- 任何关于这些装置的推进，都必须保住“局部命运可被影响、宇宙总格局不被轻易改写”的尺度边界。

### 2.9 三层分层执行原则

- 宇宙长期正典、`古裂残天` 长期正典、第一部强绑定设定必须分层书写、分层引用。
- 宇宙长期正典不得由 `古裂残天` 局部经验反写。
- `古裂残天` 长期正典不得被第一部隔绝地域舞台直接替代。
- 第一部强绑定设定可以强绑定舞台、法宝、主角团与命题，但不得无声升级为长期正典。
- 若某条内容只在隔绝地域尺度成立，应优先标记为第一部强绑定设定，而不是默认写成主世界共性。

## 3. 取舍标准

当多个方案同时成立时，优先级如下：

1. 是否符合已确认正典。
2. 是否符合范围层级与尺度控制。
3. 是否有利于长期扩写，而不是只服务当前一部故事。
4. 是否减少跨模块重复、冲突与后续维护成本。

若某方案短期看更“精彩”，但会破坏上述四项中的任一项，应优先放弃该方案。

## 4. 冲突处理方式

- 先回查 `canon_rules.md`、`glossary.md`、`decision_log.md`。
- 再回查对应模块正式文件。
- 若冲突仍无法消解，先停写并记录到审阅 / 修桥文件或待人工拍板项。
- 不允许用新文本把旧冲突覆盖掉，而不留下可追踪记录。

## 5. 何时必须回查本文件

- 当故事层开始逼近任务层时。
- 当角色层开始影响世界观口径时。
- 当局部舞台开始有扩张到整个主世界的倾向时。
- 当第一部强绑定设定开始侵占宇宙级术语与规则时。
- 当发现“继续写”比“先修桥”更诱人时。

## 6. 与其他文件的关系

- `01_project_brief.md`：说明项目是什么、当前在干什么。
- `03_scope_and_constraints.md`：说明当前允许做什么、不允许做什么。
- `05_scope_map.md`：说明内容该落在哪一层。
- `decision_log.md`：记录原则被落实后的关键决策。

## 7. 当前阶段约束

- 当前阶段仍应优先保证护栏稳定、接口闭环与尺度一致。
- 若后续要进入更细层开发，必须先确认：项目级护栏、关键正典文件与故事尺度都已稳定。
- 后续新增执行层文件时，应优先回查 `worldbible/09_reference/first_arc_vs_universe_scope.md` 与 `worldbible/09_reference/scope_execution_guide.md`，确认内容属于哪一层。
