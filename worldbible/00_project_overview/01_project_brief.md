# 01 项目总纲

## 1. 文件用途

本文件用于作为项目级总入口，集中说明本仓库的定位、目标、当前阶段重点与协作边界。

它不是世界观正典正文文件，也不是剧情文档；其作用是帮助后续协作者在进入具体模块前，先确认“这个仓库是什么、当前做到哪里、暂时不该做什么”。

## 2. 项目定位

- 项目类型：中式玄幻 CRPG 世界观设定仓库。
- 项目中文总名：`升途`。
- 项目英文正式名：`Ascendant Path`。
- 仓库 / 目录英文名：`shengtu`。
- 第一部标题：`升途：封绝之地`。
- 仓库性质：宇宙级设定仓库。
- 使用目标：为长期世界观建设、模块协作、后续剧情开发与多部故事扩写提供稳定正典与协作护栏。

必须明确：

- 本仓库不是 `古裂残天` 单世界设定仓库。
- `古裂残天` 只是当前重点展开对象，不是整个仓库的唯一对象。
- 第一部 `升途：封绝之地` 的当前故事舞台也不是整个 `古裂残天`，而是其内部一片被大战推离主世界常规秩序带的隔绝地域。

## 3. 项目目标

本项目的长期目标，是构建一套能够支持“多个主世界、多个小世界、多个故事入口、多个时代层次”的中式玄幻宇宙。

当前已经建立的主干链条为：

- `01_cosmology`：宇宙结构与飞升 / 接引框架。
- `02_power_system`：气 / 规则 / 法 / 锚点等底层力量逻辑。
- `03_worlds_and_paths`：小世界道路模板与分化逻辑。
- `04_main_world`：九主世界与故事主世界 `古裂残天` 的主世界层框架。
- `05_history`：七大时代框架、宇宙总史、九主世界历史映射与古裂残天重点史。
- `06_society`：主世界社会模型总论与古裂残天社会落地。
- `07_characters`：角色来源模型、主角团框架与角色概念层。
- `08_story`：故事结构骨架，并已收束到隔绝地域尺度。

## 4. 当前阶段目标

当前阶段的重点，不是继续无上限扩写，而是：

- 保障 `01–09` 之间的层级边界稳定。
- 保障“宇宙长期正典 / 古裂残天长期正典 / 第一部强绑定设定”分层清楚。
- 保证 `07_characters` 与 `08_story` 不反向吞没宇宙级仓库。
- 在进入任务层之前，把关键护栏、接口与损坏文件先修稳。

因此，当前不默认直接进入任务层的原因是：

- 仍存在少数关键文件完整性问题与跨模块桥梁问题。
- 第一部故事虽然已经可以成立，但仍需继续避免其绑死整个宇宙仓库。
- 项目级护栏文件需要先补齐为可执行正式版，确保后续协作不靠会话记忆推进。

## 5. 当前重点对象与整体范围的关系

当前重点对象为：

- 主世界：`古裂残天`
- 当前故事舞台：`古裂残天` 内部被大战推离的隔绝地域

这意味着：

- `古裂残天` 可以被深描，但不能反向定义全部主世界通则。
- 隔绝地域可以承载第一部故事，但不能被误写成整个主世界等同物。
- 第一部故事可以强绑定接引法宝、复生机制与大战余波，但这些绑定应被标记为“当前故事重点”，而不是默认扩展到整个宇宙。

## 5.1 三层执行分层

为避免后续协作混层，当前执行层统一采用以下三层分层：

- 宇宙长期正典：指三层宇宙、`3000` 小世界与 `9` 主世界框架、底层力量逻辑、九主世界比较框架等长期适用于整个仓库的内容。
- `古裂残天` 长期正典：指其古老残缺修复型主世界定位、双极旧秩序与残胜格局、大势力主导型格局、四级社会模型等只属于该主世界总层的内容。
- 第一部 / 当前故事强绑定设定：指隔绝地域舞台、主角团当前能力尺度、接引法宝作为主线核心装置，以及当前主线围绕地域命运与代价承担展开的内容。

后续进入具体模块前，应先判断当前内容属于哪一层；若只属于第一部强绑定设定，不得无声上浮为 `古裂残天` 长期正典或宇宙长期正典。

## 6. 与其他项目级文件的关系

- `05_scope_map.md`：负责明确全仓库的层级划分与目录边界。
- `04_current_state.md`：负责记录当前推进到哪一步、下一步做什么。
- `session_handoff.md`：负责为下一轮会话快速恢复上下文。

使用顺序建议为：

1. 先读本文件，明确仓库定位与当前目标。
2. 再读 `05_scope_map.md`，明确内容应落在哪一层。
3. 再读 `04_current_state.md` 与 `session_handoff.md`，明确当前正在处理什么。

## 7. 使用约束

- 本文件只能总结、压实和约束已确认内容，不用于偷渡新设定。
- 若本文件与正式模块文档冲突，应优先回查 `canon_rules.md`、`glossary.md`、`decision_log.md` 与对应模块正式文件。
- 若当前阶段重点对象发生变化，应同步更新本文件、`04_current_state.md` 与 `session_handoff.md`。
