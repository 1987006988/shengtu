# 07 目录职责表

## 1. 文档用途

本文档用于明确 `worldbible/01_cosmology` 至 `worldbible/09_reference` 在当前阶段各自承担什么职责，以及哪些目录能写到什么深度。

它服务于目录治理与协作分工，不属于世界观正典正文。

## 2. 当前阶段总原则

- 当前大多数目录仍属于设定层或执行护栏层。
- `07_characters` 与 `08_story` 已开始承担第一部承载层，但仍未进入任务层。
- 任何目录当前都不得写对白、关卡、战斗、玩法与详细任务流。
- 第一部内容不得反向污染宇宙总层与 `古裂残天` 总层。

## 3. 目录职责表

### 3.1 `01_cosmology`

- 当前职责：维护宇宙结构、位格、飞升、接引、九主世界宏观格局。
- 当前性质：宇宙通则层。
- 当前阶段允许深度：结构、通则、宏观比较。
- 当前阶段禁止内容：按白鹿原局部经验改写宇宙通则；任务、对白、玩法。

### 3.2 `02_power_system`

- 当前职责：维护 `气 / 规则 / 法 / 锚点` 等底层力量逻辑。
- 当前性质：宇宙通则层。
- 当前阶段允许深度：底层定义、边界、例外规则说明。
- 当前阶段禁止内容：为第一部舞台量身新造宇宙级特殊规则；任务、对白、玩法。

### 3.3 `03_worlds_and_paths`

- 当前职责：维护小世界模板、道路源流与飞升者归类框架。
- 当前性质：宇宙通则层 + 小世界道路源流层。
- 当前阶段允许深度：模板、分类、来源比较。
- 当前阶段禁止内容：退化为职业来源表，或直接写成主角团技能树背景。

### 3.4 `04_main_world`

- 当前职责：维护九主世界比较框架与 `古裂残天` 主世界总层。
- 当前性质：主世界层 + 故事主世界总层。
- 当前阶段允许深度：主世界共性、主世界比较、`古裂残天` 长期正典。
- 当前阶段禁止内容：把白鹿原第一部绑定设定直接写成 `古裂残天` 总层常态。

### 3.5 `05_history`

- 当前职责：维护宇宙总史、主世界历史映射与 `古裂残天` 历史主轴。
- 当前性质：历史设定层。
- 当前阶段允许深度：时代框架、长期演变、结构性历史。
- 当前阶段禁止内容：写成逐年流水账、剧情前史串场文案、任务背景脚本。

### 3.6 `06_society`

- 当前职责：维护主世界社会模型与 `古裂残天` / 白鹿原的社会承接逻辑。
- 当前性质：设定层，已部分承担第一部承载层。
- 当前阶段允许深度：社会模型、生存逻辑、托底逻辑、结构承接关系。
- 当前阶段禁止内容：写具体人物生活正文、任务分区、对白、关卡。

### 3.7 `07_characters`

- 当前职责：维护主角团框架、角色原型卡、概念卡与角色接口。
- 当前性质：第一部承载层。
- 当前阶段允许深度：
  - 角色来源模型
  - 席位骨架
  - 角色原型卡
  - 角色概念卡
  - 与白鹿原结构、势力、节点的接口关系
- 当前阶段禁止内容：
  - 正式人物传记
  - 角色个人任务线
  - 对白
  - 恋爱线、结局线
  - 战斗职业槽位表

### 3.8 `08_story`

- 当前职责：维护第一部主线结构、白鹿原舞台结构、长期张力、章节束等。
- 当前性质：第一部承载层。
- 当前阶段允许深度：
  - premise
  - 四幕结构
  - 分段、节点、章节束骨架
  - 白鹿原结构、托底逻辑、张力图
- 当前阶段禁止内容：
  - 详细任务层
  - 对白稿
  - 关卡流程
  - 战斗脚本
  - 玩法说明

### 3.9 `09_reference`

- 当前职责：维护术语、决策、命名规则、执行护栏、交接与审阅记录。
- 当前性质：长期记忆与协作规范层。
- 当前阶段允许深度：规则、日志、门槛、清单、执行指南。
- 当前阶段禁止内容：借参考层文件偷渡世界观正文、剧情正文或角色正文。

## 4. 哪些目录当前仍属于设定层

- `01_cosmology`
- `02_power_system`
- `03_worlds_and_paths`
- `04_main_world`
- `05_history`
- `06_society`
- `09_reference`

这些目录当前主要处理规则、结构、模型、长期正典与执行护栏，不负责下沉任务层。

## 5. 哪些目录开始承担第一部承载层

- `07_characters`
- `08_story`
- `06_society` 的白鹿原相关桥梁文件

这里的“承载层”指开始承接第一部舞台、角色接口、主线结构，但仍停在结构层与设定层，不等于正文层或任务层。

## 6. 当前哪些目录不得写任务、对白、关卡、战斗、玩法

- 当前全部目录都不得直接写任务、对白、关卡、战斗或玩法。
- 若未来需要下沉，也应主要由 `08_story` 与后续专门文件承接，而不是反写到 `01–06`。

## 7. `07_characters` 与 `08_story` 当前阶段允许写到什么深度

### `07_characters`

- 允许写到：角色来源模型、席位、原型卡、概念卡、接口落位。
- 不允许写到：完整人物传记、具体人物故事、个人任务、对白、战斗职业定义。

### `08_story`

- 允许写到：舞台结构、托底逻辑、长期张力、主线骨架、章节束。
- 不允许写到：任务清单、对白、逐章节正文、关卡脚本、玩法映射。

## 8. 第一部内容如何避免反向污染宇宙总层

- 第一部专属内容优先落在 `07_characters`、`08_story` 与桥梁文件，不直接上浮到 `01–05`。
- 若 `06_society` 提及白鹿原，应显式说明这是第一部强绑定设定。
- `04_main_world`、`05_history` 中若需要引用白鹿原，只能把它作为 `古裂残天` 内部当前舞台，不得写成主世界共性。
- 任何第一部绑定内容若影响跨模块口径，先记入 `09_reference`，再决定是否写入更高层。

## 9. 当前阶段结论

- 当前仓库仍以设定层与承载层治理为主。
- 下一步应进入“白鹿原势力与关键节点落位阶段”。
- 当前仍不默认进入任务层。
